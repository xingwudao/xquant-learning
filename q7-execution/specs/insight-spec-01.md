## 一、设计意图复盘

这是 q7「实盘执行」第一份 spec，承担**从权重到订单的翻译器**角色。Q6 之前学员一直在浮点数权重的世界里——35% 是 0.35，机器算就行了。Q7 第一脚直接踩进现实：券商 APP 只接受整数股。这份 spec 的 9 步任务不是教 API，是用三档资金 × 两个市场（A 股 / 美股）让学员**亲眼看到取整偏差是系统性的、不可消除的**。【动机】【教学场景】

**为什么三个步骤层层递进**：第一步空仓+10万人民币只让 AI 跑通 generate_orders 接口（破冰）；第二步对比 A 股 vs 美股（暴露 lot_size=100 vs 1 的差异）；第三步五档资金量×两个市场（揭示资金量越小偏差越大）。这种「单点 → 横切 → 网格」的递进是 q7 整章的认知节奏——先看到现象、再看到对比、再看到规律。【结构】

**为什么用 Decimal 而不是 float**：spec 第 3 条要求 `from decimal import Decimal`，并在常量定义里坚持 `Decimal("0.35")`、`Decimal("4.27")`。这是 q7 实盘严肃性的第一条规矩——金额计算用浮点会累积舍入误差，10 万本金跑五档资金量算下来差几分钱不离谱，但学员要从第一行代码开始知道**钱的事用 Decimal**。【精确】【教学场景：金融严肃性】

**oxq 接口先读后写**：spec 第 2 条明确指示 AI 先读 `oxq.trade.order_generator.generate_orders` 和 `oxq.core.types.Position` 的源码，列出参数签名和 `PlannedOrder` 的属性清单。这是 q2 起项目自有规范的延续，避免 AI 按训练数据猜接口。【可学习】

**取整偏差的克制呈现**：第 8 步明确「figsize 14×5、两个子图并排、A 股红色、美股蓝色」——颜色对应承袭 Q4-Q6 的视觉惯例（A 股红、美股蓝），让学员跨章节看图不需要重新解码。【同步】

## 四、可改进点（带改写示例）

### 改进 1：把图片保存路径写进 spec【同步】【精确】

**现状**（spec 第 8 步）：
```
8. 画一张图（figsize 14×5，两个子图并排）：...
```
没有提保存路径，但 notebook 实际写 `plt.savefig("../book/images/01-rounding-deviation.png", dpi=150, bbox_inches='tight')`。book 章节正是引用这张图——一旦 AI 重跑保存到别处，book 图片断链。

**改写**：
```
8. 画一张图保存到 `../book/images/01-rounding-deviation.png`（dpi=150, bbox_inches='tight'），同时显示：
   - figsize 14×5，两个子图并排
   - bar_width=0.35
   - 左图：A 股 vs 美股的最大权重偏差随资金量变化（分组柱状图）
   - 右图：A 股 vs 美股的现金残余占比随资金量变化（分组柱状图）
   - A 股 #E74C3C（红）、美股 #3498DB（蓝）
```

把颜色 hex、bar_width、保存路径全显式——视觉契约零漂移。

### 改进 2：补机械 assert【验证】

**现状**：验证段是 5 条描述（"显示 3 只 ETF""有明确数值"），都需要人眼确认。

**改写**（在第 9 步分析后加）：
```
0. 数据合法性断言：
   assert len(orders) == 3, "应生成 3 笔订单"
   assert all(po.target_shares % 100 == 0 for po in orders), "A 股股数必须是 100 的整数倍"
   assert (CAPITAL - total_invested) >= 0, "投入金额不能超出本金"
```

### 改进 3：声明 oxq 版本【可复现】

**现状**：spec 未提 oxq 版本号，但 `generate_orders` 接口未来可能变。

**改写**（任务段最后加一句）：
```
本 spec 基于 open-xquant >= 0.7.0（generate_orders 在该版本引入 lot_size 参数）。
如使用更早版本，先升级：pip install -U open-xquant
```

### 改进 4：明确布局方式【同步】

**现状**：spec 没说用 `tight_layout()` 还是 `layout='constrained'`，notebook 此 cell 用 `tight_layout()`，但 spec-02 / spec-03 改用 `constrained`——章内不一致。

**改写**（第 8 步末尾加）：
```
- 使用 plt.tight_layout()（本 cell 子图简单，constrained 在 spec-02 起统一启用）
```

或者直接全章统一改用 `constrained`，spec-01 也改。

### 改进 5：把"为什么用 Decimal"写进 spec【动机】【教学场景】

**现状**：spec 只说 `from decimal import Decimal` 和 `Decimal("0.35")`，没解释。

**改写**（第 3 条加注）：
```
- from decimal import Decimal
  · 实盘金额计算必须用 Decimal 而非 float —— 浮点累积误差在大额账户上会到分到角的差异
  · 所有权重、价格、本金都用 Decimal("...") 字符串构造，避免 0.1 + 0.2 != 0.3 的陷阱
```

让学员单读 spec 时就能学到这条领域规矩。

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【结构】"单点 → 横切 → 网格"三段递进**：第一步跑通接口、第二步两市场对比、第三步五档资金 × 两市场——q7 整章的认知节奏可作为「参数空间探索型 spec」的模板
- **【精确】关键参数全显式**：`lot_size=100/1`、`Decimal("0.35")`、`COMMON_CAPITALS = [10_000, 50_000, ...]`、figsize 14×5、颜色对应——每个参数点名
- **【动机】用调仓日开场**："今天是调仓日……打开券商 APP——只能输入股数和价格"——强代入感，0 基础学员立即理解任务必要性
- **【可学习】先读后写指令**："阅读 oxq.trade.order_generator 模块的源码"——q2 起的项目规范在 q7 持续遵循
- **【教学场景】跨章颜色一致**：A 股红、美股蓝——Q4-Q7 视觉惯例延续，学员跨章节读图不需要重新解码

### B. 这份 spec 修补后可示范的原则

- **【同步】图片保存路径写进 spec**：当前缺位、book 引用断链风险高；修补后可作为「凡是 book 要用的图，spec 必须写明保存路径」的范例
- **【验证】机械 assert 替代描述性验证**：当前验证段全是描述；修补后可作为「validation oracle」原则示范
- **【可复现】oxq 版本声明**：当前未声明；q7 起接口稳定性要求高，应作为「实盘相关 spec 必须锁版本」的示范
- **【动机】Decimal 的"为什么"显式写进 spec**：当前只是 import，修补后可作为"领域规矩必须解释而非默认"的示范

### C. 领域 / 教学场景特殊考量

- **【教学场景：金融严肃性】Decimal 不是可选项**：q1-q6 都用 float 没事（回测内部计算），q7 起涉及真实金额——浮点误差在 100 万账户上可能到分到角的差异。这是从「实验」过渡到「执行」的第一条规矩，spec 必须显式教
- **【教学场景：lot_size 是中国 ETF 的硬约束】**：A 股 100 股整手是市场规则，美股 1 股是市场规则——这不是 oxq 的设计选择，是真实世界的制度差异。spec 用 lot_size 参数化让学员看到这种「制度差异如何在代码里体现」，是量化教学的领域知识点
- **【教学场景：取整偏差的渐进暴露】**：先单点（10万人民币）让学员看到偏差存在 → 两市场对比让学员看到差异程度 → 五档资金量让学员看到资金量与偏差的关系。这种「现象 → 对比 → 规律」的渐进，是 q7 整章避免一上来轰炸 0 基础学员的关键设计
