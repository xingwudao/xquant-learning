## 一、设计意图复盘

这份 spec 是 q7 章的**金融严肃性放大器**——七步任务里同时压缩了五种成交价模式 × 完整回测 × 双子图净值曲线 × 四层成本层叠 × 成本侵蚀分解。Step 1 让学员看到取整偏差，Step 2 直接告诉学员**回测的收盘价假设是最乐观的**——同样一个策略决策，仅仅因为执行时机不同，结果差距就这么大。【动机】【教学场景：金融严肃性】

**为什么要五种 FillPriceMode 全跑**：CLOSE / MID / NEXT_OPEN / NEXT_HIGH / NEXT_LOW 不是穷举，而是构造一个**带状区间**——HIGH 和 LOW 之间用 fill_between 填充的橙色区域就是"执行不确定性区间"。学员第一眼看到的不是数字，是带宽。这种「带宽即认知」的可视化设计是 q5/q6 已有的方法论，q7 在执行场景里再用一次。【结构】

**为什么用四层成本层叠而不是一次到位**：spec 第 6 条把成本拆成 4 层（无成本 → +佣金万一 → +印花税合计万六 → +滑点千一），每层一次完整回测、四条净值曲线对比。这是「积分式认知」——看到的不是「成本扣 X%」一个数字，而是「佣金扣多少、印花税扣多少、滑点扣多少」的归因分解。【动机】【教学场景】

**oxq 接口先读后写最丰富的一份**：spec 第 1 条列了 7 个 oxq 模块/接口（SimBroker、FillPriceMode、PercentageFee、PercentageSlippage、Engine、YFinanceDownloader、LocalMarketDataProvider、make_strategy / Threshold / RiskParityOptimizer / RebalanceFrequencyRule / StopLossRule），是 q7 四份 spec 里最长的接口清单。这反映 q7 spec 的特点——一份 spec 已经在调用前面整章建立的 oxq 全栈，先读后写不可省略。【可学习】

**印花税近似的真诚**：spec 第 6 条主动说"印花税实际只在卖出时收取 0.05%，这里用 PercentageFee 近似折算为双边费率"，并要求 notebook 打印这条说明。这是教学诚实——不是把模型当真理，而是承认精度边界。【动机】【教学场景：金融严肃性】

## 四、可改进点（带改写示例）

### 改进 1：金融术语在 spec 内最低限度解释【可学习】【教学场景】

**现状**：spec 直接用「印花税折算双边费率」「执行不确定性区间」「归一化净值」「夏普比率」等术语，0 基础学员单读 spec 难懂。

**改写**（在要求段开头加一段术语简释）：
```
**本 spec 涉及术语**（详见 book 章节）：
- 成交价：你的订单实际成交时使用的价格（≠ 你看到的报价）
- 滑点：实际成交价与下单时报价的差异
```

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【结构】"探索 → 归因"两段叙事**：第 4-5 条压力测试（看到差距）+ 第 6-7 条成本层叠（归因到佣金/印花税/滑点）——这种「先看到现象、再做归因」的结构是 q7 的核心方法
- **【动机】带宽即认知**：HIGH 和 LOW 之间的橙色填充——把「执行不确定性」从抽象概念变成可视化的带宽，是金融教学的常用招数
- **【精确】成本拆四层逐层加**：无成本 → 佣金 → 佣金+印花税 → 全成本——每层一次完整回测，结果可量化归因。可作为「积分式归因」的模板
- **【可学习】oxq 接口先读后写最丰富版**：列出 7 个 oxq 模块/类/枚举的签名清单——q2 起规范的实践示范
- **【教学场景】教学诚实**：主动说"印花税近似折算双边费率，实际略有差异"——不把模型当真理。这种坦白是金融教学避免误导的关键

### B. 这份 spec 修补后可示范的原则

- **【可学习】spec 内最低术语解释**：当前金融黑话密度过高；修补后可作为「面向 0 基础的 spec 应自包含基础术语」的示范
- **【同步】跨章参数引用必须显式**：BEST_FREQ=21、BEST_SL=0.10 来自 q4，当前未引用；修补后可作为「跨章 spec 必须引用上游产出」的示范
- **【可复现】end 日期锁死**：当前用 `today` 动态值，是时间炸弹；修补后是 q1/insight-spec-01 同款修补的延续证据
- **【验证】机械 assert 验单调性**：当前验证段是描述；修补后可作为「结构性断言」（不是数据值断言而是关系断言）的示范

### C. 领域 / 教学场景特殊考量

- **【教学场景：金融严肃性的可视化】**：q7 的核心张力是「让学员从舒适的回测世界进入残酷的执行世界」。这份 spec 用「五条净值曲线 + 橙色不确定性带宽 + 四层成本侵蚀曲线」三个视觉构造让学员**直接看到**而不是**听说**——比文字千倍有效
- **【教学场景：印花税教学诚实】**：A 股印花税是 0.05% 单边（卖出收）；用 PercentageFee 双边费率近似是工程妥协——spec 主动承认这点，避免学员把近似当真理。这是金融教学和通用编程教学的分野
- **【教学场景：oxq 接口集成测试】**：spec-02 一份 spec 同时调用 7 个 oxq 模块——是 q7 后段「全栈集成」的预演。前期 q1-q6 各章只动 oxq 一两个模块，q7 起 spec 写法必须教学员**如何用一份 spec 协调多个模块**，先读后写规范在这里价值最大
