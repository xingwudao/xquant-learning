## 一、设计意图复盘

这是 q1 章「过拟合教学的伏笔 spec」——刻意呈现"穷举找最优看起来很科学"的诱惑场景，为 spec-07 的"真相揭晓"埋伏笔。29 行的精简篇幅反映了一个有意识的写作选择：**spec 本身的措辞要"看起来无害"**，不能提前剧透"这是过拟合"，否则会破坏 spec-07 的反转张力。【教学场景：叙事张力】

**为什么扫描 MA5 到 MA120 步长 5（24 个参数）**：原 spec 没说，背后的教学张力——24 个参数刚好是"看起来全面但不算夸张"的数量。如果只扫 5 个参数，学员不会觉得"穷举科学"；如果扫 100 个，学员会觉得"那不就是数据挖掘吗"，提前看穿伎俩。**24 是个甜区——足够多到产生过拟合，又不至于让学员直接察觉**。这条精妙的教学设计**完全没写进 spec 主体**——是 spec-06 的最大薄弱处。【动机】

**为什么不在这一节做样本外验证**：spec 第 4 条只要求"找出最高收益率的参数"，没有任何"验证"步骤。这是有意为之——把"训练 / 测试集分割"完整留给 spec-07 作为反转。**spec-06 必须保持"看起来很美好"的纯净状态**，不能掺杂"但要小心"之类的提示。【教学场景：叙事张力】这是 q1 中最特殊的 spec——刻意"不写"才是好 spec。

**柱状图配色的教学意图**：spec 第 3 条结果呈现要求"正绿负红 + 最优金色 + 灰虚线标基准 + 零线"。这是**最强对比的视觉设计**——金色柱子在绿色丛中"高高跃起"，灰虚线在下方画出"简单定投基准"。这种视觉对比让学员产生"找到了！"的多巴胺反馈——而这正是 spec-07 反转所需的情绪铺垫。【精确】【教学场景】

**spec 第 3 条"对每个参数调用已有的均线回测函数"**：这是接续型 spec 链的复用范例——直接调 spec-05 已经定义的 `backtest_dca_with_ma`，不写新函数。spec 用一句话点明"已有的"，让 AI 知道这是接续而非重写。【结构】

**spec 第 4 条"找出收益率最高的参数和对应收益率"**：用了 `idxmax` 这个 pandas 的标准 idiom。这条措辞比代码片段抽象一层（不说 `idxmax`），给 AI 留实现自由度。但这种自由度对 0 基础学员是把双刃剑——读 spec 时不知道 `idxmax` 这个概念存在。【可学习】【抽象层次】

## 四、可改进点（带改写示例）

### 改进 1：把"为什么 5-120 步长 5"写进 spec【动机】

**现状**（要求 第 2 条）：
```
2. 扫描范围：MA5 到 MA120，步长为 5（共 24 个参数）
```

**改写**：
```
2. 扫描范围：MA5 到 MA120，步长为 5（共 24 个参数）
   · 选这个范围：覆盖从超短期（5 日）到中长期（120 日，约半年）的合理均线周期
   · 步长 5 而非 1：步长 1 会扫 116 个参数 — 过拟合的"数据挖掘嫌疑"过于明显，会破坏 spec-07 的反转张力；
     步长 10 又只扫 12 个 — 过拟合幅度不够明显
   · 24 个参数是教学甜区：足以产生过拟合（spec-07 会揭穿），又不会让学员现在就察觉
```

把"24 是教学甜区"显式化是必须的——这是 q1 章过拟合教学的核心设计，绝不能埋没。

### 改进 2：把"刻意不做样本外"写进 spec【动机】【教学场景】

**现状**：spec 没有任何"为什么不验证"的说明

**改写**（加在任务段后）：
```
## 任务

扫描所有均线参数（MA5 到 MA120），找到历史收益率最高的「最优参数」并可视化。

> **教学注**（给 AI 的提示，不写进 notebook）：本节刻意保持"参数扫描看起来很美好"的叙事张力。
> 不要主动加入"训练 / 测试集分割""样本外验证""谨慎使用最优参数"等保险措辞——
> 那是 spec-07 的工作。本节越美好，spec-07 反转越有冲击力。
```

把教学叙事的设计意图写进 spec，让 AI 知道"这是有意识的不写"而非"忘了写"。

### 改进 3：补"最优参数应跑赢简单定投" assert【验证】

**现状**：仅有打印 oracle

**改写**（加到结果呈现 第 0 条）：
```
0. 扫描结果合理性检查：
   assert len(scan_df) == 24, f"应扫描 24 个参数，实际 {len(scan_df)} 个"
   assert scan_df['ma_period'].min() == 5 and scan_df['ma_period'].max() == 120
   assert best['return'] >= result_dca['return'].iloc[-1], \
       "最优参数应至少不输给简单定投（否则扫描代码逻辑错误）"
   assert 5 <= int(best['ma_period']) <= 120, "最优参数应在扫描范围内"
```

最后一条 `best >= dca` 是关键——如果连最优参数都跑输简单定投，说明扫描代码或函数有 bug，**这条 assert 失败立刻定位**。

### 改进 4：让"扫描结果"以 DataFrame 而非 list 返回【精确】【同步】

**现状**（要求 第 3 条）：
```
3. 对每个参数调用已有的均线回测函数，记录最终收益率
```

**改写**：
```
3. 对每个参数调用已有的均线回测函数，记录最终收益率：
   - 用 list 收集 dict（每条含 ma_period 和 return）
   - 转成 DataFrame：scan_df = pd.DataFrame(scan_results) 包含列 ma_period, return
   - 后续 spec-07 会复用这个扫描代码思路（在训练集上重跑）
```

把"为什么用 DataFrame"显式化（便于 idxmax + 后续 spec-07 复用），同时为 spec-07 埋接续伏笔。

### 改进 5：补 idxmax 的隐性使用提示【可学习】【教学场景】

**现状**（要求 第 4 条）：
```
4. 找出收益率最高的参数和对应收益率
```

**改写**：
```
4. 找出收益率最高的参数和对应收益率：
   - 用 pandas 的 idxmax 拿到最高行的索引：best_idx = scan_df['return'].idxmax()
   - 取出整行：best = scan_df.loc[best_idx]
   - best['ma_period'] 是最优参数，best['return'] 是最优收益率
```

虽然 spec 应保持抽象，但 idxmax 是数据科学高频 idiom，第一次出现给个"参考实现 / 标准做法"对 0 基础学员的心智成本会显著降低。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 本节是本份 review 给方法/模板章节贡献的"原材料"。36 份 review 全部完成后，按标签横向聚合得到方法目录。

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式** — 与前 5 份一致，是【结构】方法点的第六份证据
- **【结构】接续型 spec 链的"复用调用"节点**：本 spec 不重写函数，只调用 `backtest_dca_with_ma`——spec 一句话点明"已有的"，AI 直接复用 — 可作为"接续 spec 中如何用一句话识别已有资产"方法点示范
- **【精确】视觉契约最完整**：柱状图所有元素（正绿负红、最优金色、灰虚线、零线、图例）都在 spec 内 — 是 q1 中【精确】方法点的最佳示范（高于 spec-04，因为 spec-04 还有部分 hex 码漂移）
- **【简洁】29 行** — 与 spec-03 并列 q1 最短，可作为"重叙事的 spec 应保持简洁"低位锚定
- **【教学场景：叙事张力】"刻意不写样本外"** — 本节最特殊的设计：spec 越精简、越"看起来美好"，spec-07 反转越有冲击力 — 这是 q1 独有的"反转 spec"模板

### B. 这份 spec 修补后可示范的原则

- **【动机】教学甜区参数（24 个）必须显式化**：当前完全缺位，是 q1 章过拟合教学的核心设计；修复后可作为"教学场景下 spec 的元说明"方法点示范
- **【动机】"刻意不做某事"也要在 spec 里说明** — 当前缺位；修复后是【动机】方法点的特殊形式（"为什么 spec 里没有 X"），可独立列出
- **【验证】"最优应至少不输给基线" assert** — 当前缺位；修复后是【验证】方法点的第六份证据，且这条特殊在"用业务逻辑而非数值范围作判据"
- **【可学习】高频 idiom（idxmax）的第一次出现给参考实现** — 修复后可作为"spec 抽象层次 vs 0 基础学员心智成本的权衡"方法点示范

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】"反转 spec"的设计模式**：q1 章 spec-06 的核心特殊性——它本身**必须看起来美好且无瑕疵**，才能让 spec-07 的"过拟合真相"产生最大冲击。这是教学型 spec 写作中独特的反转模式：spec-06 = 设伏 + spec-07 = 揭穿。本课程必须把这种"双 spec 反转结构"作为方法点列出——其他章节（如风控、Alpha 因子）也会用到。
- **【教学场景】参数扫描的"教学甜区"**：扫太少不诱惑、扫太多太露馅。24 是 q1 选择的甜区——本课程必须教学员"教学示例的参数数量是叙事服务的，不是科学最优"。这是普通量化教程不需要的元说明，但本课程作为教学产品必须显式承认。
- **【教学场景】视觉对比放大叙事张力**：金色 vs 绿/红的强对比、灰虚线 vs 实线柱的强对比，是为了让"找到了"的感觉最大化。这种"视觉服务于叙事"的写法在量化课程里反复出现（spec-02 双图、spec-04 三角标记、spec-06 金色高亮），应作为统一方法点列出。
