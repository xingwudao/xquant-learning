## 一、设计意图复盘

这是 q1 章引入「Beta」概念的关键 spec——学员第一次面对"我的策略到底有没有'打败'市场"这个量化交易最根本的问题。spec 短到 31 行，刻意设计为"算两个数 + 画一张图 + 给一句话评论"，把抽象概念压到最小动作。【结构】【教学场景】

**为什么用"一次性买入持有"作为基准**：原 spec 没说，背后的张力有三层：① Buy & Hold 是**最便宜的对照组**——零策略、零交易成本，定投策略要超越它才证明"分批买入"这个动作有价值；② 它和定投有同样的资金量假设（虚拟，便于纯收益率比较），不引入"实际可投金额"的争议；③ 它的收益率就是市场 Beta 的近似值，让"Alpha = 策略 - Beta"这个公式在第一次出现时有具体数字支撑。【动机】没写进 spec 主体是这份 spec 的最大薄弱处。

**为什么要画"逐月累计收益率曲线"而不是只比终点**：spec 第 4 条要求计算"基准的逐月累计收益率"（以月末收盘价对起始价的涨跌幅）。这是为了让对比图的两条线**频度对齐**——定投是月度数据，基准也按月对齐才能画在同一张图上。学员看图时直观看到"两条线一起波动 = Beta 主导 = 定投没有 Alpha"。【精确】【教学场景】

**条件式打印的克制**：spec 第 3 条用"如果定投跑赢/跑输"的二分支打印，第 4 条无论分支都补一句"两种策略都没有 Alpha"。这是教学叙事的精妙处——无论数据怎么变化，最终结论都收束到"Beta 主导"上，引出下一节均线择时。但**这种"按结果分支打印"是 q1 的反复出现的模式**（spec-07 同样有"过拟合 / 还行"两分支）——可作为方法点。【教学场景：叙事收束】

**没有锁定 benchmark_return 的精确数值**：spec 不写"benchmark_return 应在 0.10 到 0.50 之间"这类合理范围断言。这是合理的——具体数字依赖 spec-01 的数据日期，而后者本身也未锁死 end_date。但这条隐患**累积**了——spec-01 的时间炸弹通过这条接续链在 q1 后续每份 spec 上传染。【可复现】

## 二、与 notebook 实际产出的对照

| spec 要求 | notebook 实际实现 | 评价 |
|---|---|---|
| `benchmark_return = (final - initial) / initial` | 字符级一致 | 同步 |
| 打印起点价格、终点价格 | `:.3f` 三位小数 | spec 没规定精度，AI 自选 |
| 打印两个收益率 + 差异 | `:.2%` 两位百分号 | spec 没规定精度，AI 自选 |
| 二分支打印（跑赢/跑输） | ✓，加了"→"前缀符号 | notebook 自行美化 |
| 无论哪种分支补"没有 Alpha" | ✓ | 一致 |
| 对比图（定投 vs 基准） | figsize 12×6 + 零线 + 图例 | 一致 |
| 基准月度曲线计算逻辑 | `bnh_monthly_return = (monthly_last['Close'].values - initial_price) / initial_price` | spec 没显式给公式（只说"逐月累计收益率"），AI 推断为月末价对起始价 |
| 没要求零线 axhline | notebook 加了 `axhline(y=0)` | AI 自行补；这是 spec-02 已经建立的风格连贯性 |

**对照结论**：spec 大方向覆盖完整，但"基准月度曲线的具体计算公式"是 AI 推断（"对起始价的涨跌幅"措辞接近但不等价于公式）。打印精度、零线是 AI 沿用 spec-02 风格自行补全。

## 四、可改进点（带改写示例）

### 改进 1：把"为什么选 Buy & Hold 作基准"写进 spec【动机】

**现状**：
```
2. 基准策略（Buy & Hold）：在数据起始日用全部资金一次性买入，持有到最后一天
```

**改写**：
```
2. 基准策略（Buy & Hold）：在数据起始日用全部资金一次性买入，持有到最后一天
   · 选 Buy & Hold 而非沪深300指数本身：两者收益率几乎等价，但 ETF 可交易、有真实意义
   · 选 Buy & Hold 而非银行理财：基准必须和策略在同一资产类别，否则比较的是大类资产配置而非策略本身
   · 基准的收益率近似等于市场 Beta；定投策略要"超越"它才能证明"分批买入"这个动作有 Alpha
```

让学员**单独阅读 spec 时**就理解"基准是相对的"这层概念。

### 改进 2：显式给出基准月度曲线的计算公式【精确】【同步】

**现状**（要求 第 4 条）：
```
4. 计算基准的逐月累计收益率曲线（每月末收盘价相对起始价的涨跌幅），用于可视化对比
```

**改写**：
```
4. 计算基准的逐月累计收益率曲线（用于和定投月度曲线对比画图）：
   monthly_last = df.groupby(df.index.to_period('M')).last()
   bnh_monthly_return = (monthly_last['Close'].values - initial_price) / initial_price
   · 频度必须对齐定投（月末），否则两条线 X 轴长度不同
```

直接给公式片段，spec 与 notebook 字符级一致；同时显式提示"频度对齐"这个隐性约束。

### 改进 3：补 assert 验证基准计算【验证】

**现状**：仅有定性 oracle（打印 / 画图）

**改写**（加到结果呈现 第 0 条）：
```
0. 数据合法性检查：
   assert 'benchmark_return' in dir() or 'benchmark_return' in globals()
   assert isinstance(benchmark_return, float), "benchmark_return 应为 float"
   assert -0.5 < benchmark_return < 1.5, f"基准收益率异常: {benchmark_return:.2%}（合理范围 -50% 到 150%）"
   assert len(bnh_monthly_return) == len(result_dca), "基准月度曲线点数应与定投曲线一致"
```

最后一条是接续链的强约束 —— 失败立刻定位"频度对齐错了"。

### 改进 4：把"按结果分支打印"的逻辑显式化为模板【教学场景】【可学习】

**现状**：spec 第 3 条用 if-else 分支，但没解释"为什么要二分支"

**改写**：
```
3. 根据对比结果打印一句话说明（数据驱动的二分支评论）：
   · 用意：让学员看到"叙事跟着数据走"——同一份 spec，市场上涨/下跌跑出来的解读不同
   · 如果 dca_return > benchmark_return：「在下跌市中，定投通过分批买入避开了部分高位」
   · 否则：「在上涨市中，越早全部投入越好」
```

把"为什么二分支"写进 spec，学员仿写时就知道这是个**模板**而不是个例。

### 改进 5：补 Alpha-Beta 的口语化定义【可学习】

**现状**：第 4 条只说"两种策略都没有 Alpha"，但 spec 中"Alpha"是第一次出现

**改写**：
```
4. 无论哪种结果，都补充一句：「但两种策略都没有 Alpha——
   收益完全取决于市场涨跌（这部分叫 Beta）。Alpha 是'比市场多赚的部分'，
   需要策略做出某种判断，且判断比市场更聪明。」
```

第一次出现术语就给口语化定义。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 本节是本份 review 给方法/模板章节贡献的"原材料"。36 份 review 全部完成后，按标签横向聚合得到方法目录。

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式**：与 spec-01/02 完全一致 — 是【结构】方法点的第三份证据
- **【结构】接续型 spec 链的中段**：上下文段显式列出"已有 df / result_dca"两个变量，开门见山交代依赖 — 接续链方法点的中段示范
- **【教学场景：叙事收束】二分支打印 + 共同收束**：跑赢/跑输两个分支后都补一句"都没有 Alpha"——把数据走向的不确定性收束到固定教学结论 — 这是 q1 反复出现的模式（spec-07 同款），可作为"叙事 spec"方法点的原型
- **【精确】频度对齐意识**：第 4 条要求基准曲线"逐月"——意识到必须和定投频度一致才能画在同一张图上 — 可作为"接续链中数据频度必须对齐"方法点的示范
- **【正向】全部"做 X"** — 与 spec-01/02 一致

### B. 这份 spec 修补后可示范的原则

- **【动机】基准选择必须解释**：当前缺"为什么 Buy & Hold 而非别的"；修复后是【动机】方法点的第三份证据（spec-01/02/03 三连证据 = 这条原则在 q1 反复出现）
- **【精确】频度对齐计算公式显式给出**：当前用自然语言描述"逐月累计收益率"；修复后可作为"涉及具体计算时 spec 直接给公式片段 = 跨文件零漂移"的示范
- **【验证】assert 替代描述性"打印"**：当前只有打印；修复后是【验证】方法点的第三份证据
- **【教学场景】数据驱动的二分支评论是模板**：当前埋在 if-else 里；修复后可作为"教学型 spec 的叙事模板"独立列出

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】基准是相对的**：量化课程必须教"基准选什么决定了 Alpha 的定义"——选 Buy & Hold、选指数、选无风险利率、选同业对手，每种选择背后是不同的 Alpha 定义。这是数据科学 / 通用 prompt 工程不需要的领域知识，本课程必须显式教。
- **【教学场景】月度频度对齐的隐性约束**：spec-02 返回 PeriodIndex（按月），spec-03 的基准也必须按月，否则画图错位。这种**接续链中的频度约束**在量化课程里反复出现（后续 spec-05 也涉及），属于本课程特有的"领域必备配置"，应在方法章节单独列出。
- **【教学场景】"按结果分支打印"是叙事 spec 的核心模板**：当 spec 的输出依赖运行时数据，且课程要保留"数据驱动叙事"的教学价值时，二分支 / 多分支打印 + 共同收束是模板写法。本课程的 spec-03 和 spec-07 都用这个模板，应作为单独方法点列出。
