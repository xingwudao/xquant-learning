## 一、设计意图复盘

这是 q1 章引入「均线」与「信号」的可视化破冰 spec——学员第一次看到"价格 vs 均线"的几何关系，为下一节 spec-05 把"交叉"作为买入条件做铺垫。50 行的篇幅是 q1 前半部分最长的，原因是 spec 内嵌了三段代码片段（MA 计算、交叉判定、画图配色）——这是**有意识的"图形细节显式化"权衡**。【结构】【精确】

**为什么截取最近 1 年而不是全 5 年**：spec 第 2 条直接写"让图更清晰"。背后的张力是 5 年数据的均线交叉点会密到看不清三角标记，1 年既能看清交叉又能保持"还是同一份数据"的连续感。【动机】这条已写进 spec，是 q1 前 4 份 spec 中**第一次把"为什么"写进 spec 主体**的尝试，是好的进步。

**为什么三条均线 MA10 / MA20 / MA60**：spec 没说，但暗藏 3 个层次的对比意图——MA10 紧贴价格（"快"）、MA20 中速、MA60 平滑（"慢"）。教学叙事在 cell[12] 的 markdown 里通过 markdown 解读"三条均线的性格不同"展开，但 spec 本身没讲清这三条的选择是为了**对比快慢**而非随便挑三个。【动机】缺位。

**为什么只判定"Close vs MA20"的交叉而不是 MA10×MA20 双均线交叉**：原 spec 第 4 条明确"找出收盘价与 MA20 的交叉点"。这是教学甜区——"价格穿越均线"概念比"两条均线相互穿越"简单一倍，对 0 编程基础学员更友好。但**为什么选 MA20 而非 MA10/MA60**没解释——可能是 MA20（约 1 个月交易日）有传统金融意义。【动机】缺位是这份 spec 最薄弱处。

**spec 内嵌代码片段的张力**：spec 第 2/3/4 条直接给出 Python 代码片段（`df.loc[one_year_ago:].copy()`、`rolling(N).mean()`、上下穿判定逻辑）。这违反了"spec 抽象层次"——但这里是**有意识的权衡**：均线计算、`shift(1)` 比较、布尔索引这三个操作对 0 基础学员是"看了答案才能写"的难点。spec 用代码片段降低门槛，代价是和 notebook 同步成本变高（任一改动两边都要改）。【可学习】 vs 【抽象层次】的张力。

**结果呈现的克制**：只要"价格+均线对比图"和"交叉统计打印"。没有引入"按交叉信号回测"这一步——那是 spec-05 的工作。这种**"先看后做"的两步教学法**是 q1 章后半部分的核心节奏（spec-04 看图 → spec-05 按图回测）。【教学场景：渐进难度】

## 二、与 notebook 实际产出的对照

| spec 要求 | notebook 实际实现 | 评价 |
|---|---|---|
| `df_recent = df.loc[one_year_ago:].copy()` | 字符级一致 | spec 内嵌代码片段，零漂移 |
| `ma10/20/60 = rolling(N).mean()` | 字符级一致 | 同上 |
| 上下穿判定（shift(1) 比较） | 字符级一致 | 同上 |
| MA10 绿色虚线、MA20 蓝色实线、MA60 红色点线 | MA20 用 `'#1f77b4'` 而非"蓝色" | 同步偏差——AI 自行选了 matplotlib 默认蓝（C0），spec 写"蓝色"模糊 |
| 上穿绿色三角 marker='^' markersize=10 | `marker='^', s=100, zorder=5` | spec 用 `markersize` 但 notebook 用 `s`（scatter 用 s 而非 markersize），且加了 zorder | 隐性偏差 |
| 标题、图例、网格 | ✓，加了 X/Y 轴标签 | spec 没规定轴标签，AI 沿用 spec-01 风格补全 |
| 交叉统计打印格式 | spec 给了模板 `MA20 交叉统计（最近1年）：` | 字符级一致 |

**对照结论**：spec 内嵌代码片段的部分**字符级同步**（MA 计算、交叉判定）；但绘图细节（marker 大小用 markersize 还是 s、颜色名 vs hex 码、zorder）出现 AI 自行决定的偏差。**spec 内嵌代码降低了部分同步成本，但没消灭绘图细节的漂移**。

## 四、可改进点（带改写示例）

### 改进 1：把"为什么是 MA10 / MA20 / MA60"写进 spec【动机】

**现状**（要求 第 3 条）：
```
3. 基于 df_recent 计算三条均线：
   - ma10 = ...rolling(10).mean()
   - ma20 = ...rolling(20).mean()
   - ma60 = ...rolling(60).mean()
```

**改写**：
```
3. 基于 df_recent 计算三条均线（三个层次的快慢对比）：
   - ma10 = df_recent['Close'].rolling(10).mean()  # 快线，紧贴价格
   - ma20 = df_recent['Close'].rolling(20).mean()  # 中速，约 1 个月交易日
   - ma60 = df_recent['Close'].rolling(60).mean()  # 慢线，约 3 个月，趋势判定
   · 选这三个而非 MA5/15/30：让"性格"对比鲜明，便于后续 spec-05 用 MA20 作主要参数
```

让学员**单独阅读 spec 时**理解"参数选择是为了对比鲜明"。

### 改进 2：把"为什么和 MA20 判交叉"写进 spec【动机】

**现状**：
```
4. 找出收盘价与 MA20 的交叉点：...
```

**改写**：
```
4. 找出收盘价与 MA20 的交叉点（作为下一节回测的信号原型）：
   · 选 MA20 而非 MA10/MA60：MA20 约对应 1 个月交易日，是传统金融最常见的"短期趋势线"
   · 选"价格 × 均线"而非"两条均线交叉"：概念简化一半，对 0 编程基础学员更友好
   · 上穿：前一天 Close <= MA20，当天 Close > MA20
   · 下穿：前一天 Close >= MA20，当天 Close < MA20
```

### 改进 3：用变量绑定替代代码片段，降低同步成本【抽象层次】【同步】

**现状**：spec 内嵌三段代码（MA 计算、交叉判定、绘图配色）

**改写**（提取常量到要求开头）：
```
2. 计算参数：
   - 截取窗口：最近 1 年（one_year_ago = df.index[-1] - pd.DateOffset(years=1)）
   - 三条均线：MA10（绿色虚线，"快"）、MA20（蓝色实线 #1f77b4，"中"）、MA60（红色点线，"慢"）
   - 交叉判定：Close 与 MA20，shift(1) 比较前后两天
3. 计算三条均线（基于 df_recent['Close'].rolling(N).mean()）
4. 找出交叉点（用上述判定逻辑）
```

把"配色 + 颜色 hex 码"放在要求开头作为常量绑定，绘图段引用常量名 — 学员能看清"哪些是参数，哪些是计算"，同步时只改一处。

### 改进 4：补绘图 marker 参数细节【精确】【同步】

**现状**：
```
- 上穿点：绿色向上三角标记（marker='^', markersize=10）
```

**改写**：
```
- 上穿点：用 ax.scatter 而非 ax.plot 标记
  scatter 参数：marker='^', color='green', s=100, zorder=5
  · 用 scatter 而非 plot：散点不会被均线遮挡（zorder=5 置于最前）
  · s=100 而非 markersize=10：scatter 用 s（面积），plot 用 markersize（直径）——两者数值不通用
```

把 markersize / s 的隐性差异写明，避免 AI 自行混用。

### 改进 5：补交叉次数合理性 assert【验证】

**现状**：仅有打印 oracle

**改写**（加到结果呈现 第 0 条）：
```
0. 合理性检查：
   total_days = len(df_recent)
   assert cross_up.sum() + cross_down.sum() <= total_days // 5, \
       f"交叉次数 {cross_up.sum() + cross_down.sum()} 异常密集（应不超过 {total_days // 5}）"
   assert cross_up.sum() >= 1, "1 年内至少应有 1 次上穿"
   assert cross_down.sum() >= 1, "1 年内至少应有 1 次下穿"
```

机械断言失败立刻定位"shift 写错了 / 数据范围错了"。

## 五、可沉淀的方法点（用于书中方法/模板章节）

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式** — 与 spec-01/02/03 一致，是【结构】方法点的第四份证据
- **【动机】"截取 1 年让图更清晰"已写进 spec** — q1 前 4 份 spec 中第一次显式给出"为什么"，是【动机】原则的正面示范（前 3 份都缺位）
- **【精确】绘图细节高度具体**：figsize / 颜色 / linestyle / marker / 图例位置全指定 — 是 q1 中**最贴近 spec-01 改进 5（"图表细节显式"）的范例**
- **【正向】全部"做 X"** — 与前 3 份一致
- **【教学场景：渐进难度】"先看后做"两步法**：spec-04 只画图不回测，把"按交叉信号回测"留给 spec-05 — 可作为"复杂概念的可视化先行"方法点示范

### B. 这份 spec 修补后可示范的原则

- **【动机】参数选择必须解释**：当前"为什么 MA10/20/60""为什么和 MA20 交叉"缺位；修复后是【动机】方法点的第四份证据（spec-01/02/03/04 四连）
- **【精确】markersize vs s 的隐性差异写明**：当前只写 markersize 但 notebook 用 s；修复后可作为"matplotlib API 同名不同义陷阱必须显式化"的示范
- **【抽象层次】内嵌代码 vs 变量绑定的权衡**：当前内嵌三段代码，是有意识但有代价的选择；修复后可作为"哪些情况 spec 应内嵌代码、哪些应抽常量"的对比示范
- **【验证】交叉次数合理性 assert**：修复后是【验证】方法点的第四份证据

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】可视化先行（看后做）**：q1 章的 spec-04 → spec-05 是教学叙事的关键转折——先让学员"亲眼看到"均线和交叉点，再要求"按这些信号回测"。这种**"先建立直觉再写策略"的两步法**是量化课程特有的节奏，本课程必须显式教学员"什么时候 spec 应该单独出一份'只画图不计算'的可视化 spec"。
- **【教学场景】matplotlib 中的 scatter vs plot 选择**：在均线图上叠加交叉点必须用 scatter 才能保证 zorder 优先级。这是数据科学领域知识，0 基础学员第一次画"在折线图上加散点标记"会混淆这两个 API，spec 必须显式区分。
- **【教学场景】MA20 是教学甜区的传统选择**：约 1 个月交易日、概念上"短期趋势线"——本课程默认 MA20 作为均线择时主参数，但应在 spec 中显式承认"这是教学选择，不是金融真理"。这是量化教学特有的元元说明，避免学员把教学示例当作真实策略推荐。
