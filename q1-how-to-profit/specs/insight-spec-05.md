## 一、设计意图复盘

这是 q1 章引入「Alpha 尝试」的转折 spec——学员第一次让策略"做出判断"（按均线条件买入），跨过从"机械定投"到"有信号的策略"的门槛。35 行的篇幅承担的核心任务是**派生函数 `backtest_dca_with_ma` 并和简单定投对比**。【结构】【教学场景】

**为什么命名 `backtest_dca_with_ma` 而不是 `backtest_dca_v2`**：spec 第 3 条要求"返回格式与 result_dca 一致，额外包含 signal 列"。这是接续型 spec 链的关键设计——派生函数和原函数的**返回结构兼容**，spec-06 的参数扫描只调用接口、不关心实现细节。命名 `with_ma` 而非 `v2` 也强调"这是 v0.1 加上一个新维度"而非"换了一代"。【结构】【可学习】

**为什么 N 日均线初期跳过买入而不是继续买**：spec 第 4 条第一个子条目"前 N 天没有均线值，这些月份不买入"。背后的张力——如果初期没均线就"无条件买入"，等于该策略在前 N 天退化为简单定投，会污染 Alpha 的归因。但**没解释为什么**——学员可能困惑"为什么不用前 N 天的简单平均代替"。【动机】【可学习】缺位。

**为什么累计成本 0 时收益率设为 0 而不是 NaN**：spec 第 4 条第三个子条目专门强调"避免除零警告"。这是 q1 中**第一次显式处理边界条件**——0 编程基础学员遇到的第一个"代码可能报警告"的场景。spec 把这个陷阱前置说明，是好的反模式规避——但代价是 spec 长度增加。【精确】【教学场景】

**为什么测三个参数 MA10/MA20/MA60 而不是直接扫描所有参数**：spec 第 5 条要求"分别用 MA10、MA20、MA60 三个参数运行回测"——这是 spec-04 三条均线的**自然延伸**（看图 → 回测）。把"扫描所有参数"留给 spec-06，是为了让"过拟合"这个概念在 spec-06/07 集中爆发，而 spec-05 先建立"参数会影响结果"的直觉。【教学场景：渐进难度】

**对照图的克制**：只画"简单定投 + 三条 MA 曲线"4 条线在同一张图。没有画每个参数的"市值 vs 成本"双图（那是 spec-02 的事），也没引入夏普比率等专业指标。这种**"对比图聚焦单一指标"的写法**是 q1 章的统一风格。【教学场景：渐进难度】

## 四、可改进点（带改写示例）

### 改进 1：把"为什么前 N 天跳过"写进 spec【动机】

**现状**（要求 第 4 条第一子条目）：
```
- 均线计算初期（前 N 天）没有均线值，这些月份不买入
```

**改写**：
```
- 均线计算初期（前 N 天）没有均线值，这些月份不买入
  · 为什么不用"前 N 天简单平均替代"：那等于该策略在初期退化成简单定投，会污染 Alpha 归因——
    我们要的是"严格按均线条件"的策略效果，前 N 天的缺失正是这个策略的固有代价
  · 这个代价随 ma_period 增大而增大——下一节 spec-06 扫描参数时会看到 MA120 的"代价"
```

让学员理解"为什么这样设计"，并且为 spec-06 的过拟合讨论埋伏笔。

### 改进 2：把否定式"避免除零警告"改为正向指令【正向】

**现状**：
```
- 计算收益率时，累计成本为 0 的月份（尚未发生任何买入）收益率直接设为 0，不要用除法，避免除零警告
```

**改写**：
```
- 计算收益率时，先把整个 returns 数组初始化为 0；只对累计成本 > 0 的月份做除法填充
  公式：
    returns = np.zeros(len(cost_array))
    mask = cost_array > 0
    returns[mask] = (portfolio_value[mask] - cost_array[mask]) / cost_array[mask]
```

把"不要用除法"改成"先初始化 + 用 mask 选择性填充"——是 q1 唯一一处否定式，改正后全章保持纯正向风格。

### 改进 3：把"为什么测三个参数"写进 spec【动机】【教学场景】

**现状**：
```
5. 分别用 MA10、MA20、MA60 三个参数运行回测
```

**改写**：
```
5. 分别用 MA10、MA20、MA60 三个参数运行回测
   · 选 spec-04 看过的三个参数：让"看见 → 回测"的叙事闭合
   · 不在这一节扫描所有参数：先建立"不同参数会出不同结果"的直觉，把"暴力穷举所有参数"留给 spec-06，过拟合的概念会在那里集中爆发
```

让学员**单独阅读 spec 时**理解 q1 的教学叙事弧线。

### 改进 4：补函数签名兼容 assert【验证】【同步】

**现状**：要求第 3 条只描述"返回格式与 result_dca 一致 + signal"

**改写**（加到结果呈现 第 0 条）：
```
0. 函数兼容性检查：
   sample = backtest_dca_with_ma(df, ma_period=20)
   assert {'total_cost', 'total_shares', 'portfolio_value', 'return', 'signal'}.issubset(sample.columns)
   assert sample['signal'].dtype == bool, f"signal 应为 bool 类型，实际 {sample['signal'].dtype}"
   assert (sample['total_cost'] >= 0).all(), "累计成本应单调非负"
   # 接续兼容性：与 backtest_dca 的索引类型一致
   assert type(sample.index) == type(result_dca.index), "应与 backtest_dca 返回相同索引类型"
```

最后一条接续兼容断言是接续型 spec 链的强约束。

### 改进 5：把对比图配色绑定常量【精确】【同步】

**现状**：spec 第 3 条结果呈现"MA10、MA20、MA60 三条收益率曲线（不同颜色实线）"

**改写**：
```
3. 画一张对比图（figsize 12×6）：
   - 简单定投：灰色虚线（color='gray', linestyle='--'）
   - MA10/MA20/MA60：matplotlib 默认 C0/C1/C2 三色实线
     colors = ['#1f77b4', '#ff7f0e', '#2ca02c']  # 对应 C0/C1/C2
   - 零线、图例、网格 alpha=0.3
```

把配色 hex 码写进 spec，spec 与 notebook 字符级一致；同时显式说明"matplotlib 默认前三色"让学员理解颜色选择不是随机的。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 本节是本份 review 给方法/模法/模板章节贡献的"原材料"。36 份 review 全部完成后，按标签横向聚合得到方法目录。

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式** — 与前 4 份一致，是【结构】方法点的第五份证据
- **【结构】接续型 spec 链的派生节点**：`backtest_dca_with_ma` 派生自 `backtest_dca`，签名兼容、额外加 signal 列 — 可作为"派生函数命名 / 返回兼容 / 仅扩展不破坏"方法点的范例
- **【精确】边界条件显式处理**：前 N 天无均线、累计成本为 0 — 是 q1 中第一次正面处理边界条件，可作为"spec 必须预先识别边界场景"方法点的示范
- **【教学场景：渐进难度】先三参数对比再扫描**：本节用 [10, 20, 60] 建立"参数会影响结果"的直觉，把暴力扫描留给 spec-06 — 可作为"概念渐进式释放"方法点示范
- **【可学习】signal 列让"是否买入"可追溯**：返回结构里加 signal 列让学员能逐月查"哪个月买了"——支持"理解策略"而非只看结果数字

### B. 这份 spec 修补后可示范的原则

- **【正向】"避免除零"改为"先初始化 + mask 填充"**：当前是 q1 唯一一处否定式；修复后是【正向】方法点的反例对照（before / after 教学）
- **【动机】"为什么前 N 天跳过""为什么测三个"**：当前缺位；修复后是【动机】方法点的第五份证据
- **【验证】函数签名兼容 assert**：当前只描述列名；修复后可作为"接续链中函数签名兼容必须机械验证"方法点示范
- **【精确】配色 hex 码写进 spec**：当前用"不同颜色"模糊指代；修复后是【精确】方法点的第三份证据（与 spec-01 / spec-04 同源）

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】派生函数的命名约定**：q1 的 `backtest_dca` → `backtest_dca_with_ma` 命名，避免 `_v2` / `_advanced` 这种"代际感"过强的命名。**派生而非取代**是接续型 spec 链的命名核心，本课程必须显式教（spec-02 已示范封装，spec-05 示范派生）。
- **【教学场景】numpy/pandas 的"先初始化 + 选择性填充"模式**：避免 ZeroDivisionWarning 的标准 idiom。这是数据科学特有的写法，0 基础学员不知道"为什么不直接 if-else"。spec 应在第一次出现时显式给出公式片段（与 spec-04 内嵌代码片段同源）。
- **【教学场景】"先三参数对比再扫描"是过拟合教学的关键节奏**：spec-05 用 [10, 20, 60] 建立"参数影响结果"的直觉，spec-06 暴力扫描，spec-07 揭穿过拟合。这是 q1 章独有的三步教学法，本课程必须把这条节奏作为"批量参数实验的教学顺序"方法点列出，避免学员一上来就扫描所有参数（那会让"过拟合"概念失去对比基础）。
