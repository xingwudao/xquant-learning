## 一、设计意图复盘

这是 q1 章的**收束 spec + 反转 spec**——同时承担两个任务：① 给 spec-06 设的"参数扫描很美好"伏笔做反转；② 把"过拟合"这个量化交易最大敌人的概念在第一章就让学员**亲身踩一脚坑**。35 行的篇幅承担了课程**第一次教学反转**的全部分量。【教学场景：叙事反转】

**为什么 60/40 而非 70/30 或 80/20 分割**：原 spec 没说，背后的张力是"测试集要足够长才能看出'最优参数'失效"。70/30 在 5 年数据上测试集只有 1.5 年——可能恰好处在和训练集相似的市场结构，反转效果不够强；50/50 测试集太大，训练集只有 2.5 年，扫描结果稳定性不足。**60/40 是教学甜区**——训练集 3 年（足以让"最优参数"出现）+ 测试集 2 年（足以让"失效"暴露）。【动机】缺位是这份 spec 最大薄弱处。

**为什么命名是 train / test 而不是 in-sample / out-of-sample**：spec 第 2 条用"训练集 / 测试集"。这是**机器学习社区的标准术语**——比量化金融圈惯用的 "in-sample / out-of-sample" 对 0 基础学员更友好（"训练 / 测试"是日常考试比喻，"样本内 / 样本外"是统计学黑话）。spec 第 5 条还给出"考试不能用试卷复习"的类比（虽然类比写在 notebook markdown 里）。【可学习】

**条件式打印的反转设计**：spec 第 5 条用二分支打印——如果测试集表现差："这就是过拟合"，否则："结果还行但样本量有限"。**注意"否则"分支也是收束到"谨慎"**——无论数据怎么走向，教学结论都收束到"过拟合是真实风险"。这是 q1 第二次出现的"按结果分支打印 + 共同收束"模板（spec-03 第一次）。【教学场景：叙事收束】

**为什么 spec-07 复用 spec-06 的扫描逻辑而非用更复杂的 KFold 等方法**：spec 第 3 条只要求"扫描参数找最优"——明显是 spec-06 的简单复制。这是有意为之——0 基础学员在第一章不需要懂 cross-validation。**简单暴露过拟合 > 用复杂方法消除过拟合**：本节的目标是让学员"亲眼见到失效"，而不是"教会防过拟合"（那是后续章节的事）。【教学场景：渐进难度】

**spec 第 5 条结论的"凹槽叙事"**：先把"过拟合"分支放第一位（学员的注意力默认在第一个分支），把"还行"放第二位 + 仍然警示——这是写作技巧上的微妙安排，让无论数据走向如何，学员的脑子里都先入为主"过拟合"概念。【教学场景：叙事张力】

## 二、与 notebook 实际产出的对照

| spec 要求 | notebook 实际实现 | 评价 |
|---|---|---|
| 60/40 时间分割 | `split_idx = int(len(df) * 0.6)`, `train = df.loc[:split_date]`, `test = df.loc[split_date:]` | 一致 |
| 训练集上扫描 MA5-120 步长 5 | `for ma in range(5, 121, 5)` | 一致（与 spec-06 完全相同范围） |
| 在测试集上用最优参数运行 `backtest_dca_with_ma` | ✓ | 一致（接续链复用） |
| 同时在测试集上运行简单定投 `backtest_dca` | ✓ | 一致（接续链复用） |
| 打印训练集 / 测试集时间范围和数据条数 | `train.index[0].date() ... ({len(train)} 条)` | 一致 |
| 打印训练集最优、测试集表现、测试集简单定投 | 3 行打印 | 一致 |
| 二分支结论打印 | ✓ 前后用 `=` * 50 装饰 | spec 没说要装饰，AI 自行加视觉边框 |
| 分割点边界处理（train.loc[:split_date] / test.loc[split_date:]） | 注意：split_date 同时被两个集合包含 | spec 没明说"包含还是排除"，AI 自选了"两边都包含"——实际等价于训练集多一天而测试集多一天，影响微小但是个边界陷阱 |

**对照结论**：核心逻辑字符级一致；分割点边界（split_date 是否被两边都包含）和打印装饰是 AI 自行决定。**这个分割边界问题是 spec-07 的隐性 bug 风险**——如果数据更稀疏或者用 cross-validation 时这条会放大。

## 四、可改进点（带改写示例）

### 改进 1：把"为什么 60/40"写进 spec【动机】

**现状**（要求 第 2 条）：
```
2. 数据分割：按时间前 60% 为训练集，后 40% 为测试集
```

**改写**：
```
2. 数据分割：按时间前 60% 为训练集，后 40% 为测试集
   · 选 60/40：训练集 3 年（足以让"最优参数"在扫描中出现），测试集 2 年（足以让失效暴露）
   · 不选 70/30：测试集 1.5 年可能恰好和训练集市场结构相似，反转效果弱
   · 不选 50/50：训练集 2.5 年扫描结果稳定性不够
   · 注意：仅用时间前后切分，不打乱顺序——金融数据时间相关性强，shuffle 会让训练集"偷看到未来"
```

最后一条"不能 shuffle"是关键的金融数据特殊约束——一定要写进 spec，否则 AI 可能照搬通用 ML 习惯用 train_test_split。

### 改进 2：把分割点边界明确化【精确】【同步】

**现状**（要求 第 2 条）：
```
2. 数据分割：按时间前 60% 为训练集，后 40% 为测试集
```

**改写**：
```
2. 数据分割（注意：分割点不能同时属于两个集合）：
   split_idx = int(len(df) * 0.6)
   train = df.iloc[:split_idx]   # 前 60%（不含 split_idx 那一行）
   test = df.iloc[split_idx:]    # 后 40%（含 split_idx 那一行）
   · 用 iloc 整数切片而非 loc 标签切片：避免日期边界包含 / 不包含的歧义
```

把"分割点归属"显式化，消除 notebook 里 `train.loc[:split_date]` 和 `test.loc[split_date:]` 重叠一天的微妙问题。

### 改进 3：补关键 assert【验证】

**现状**：仅有打印 oracle

**改写**（加到结果呈现 第 0 条）：
```
0. 数据分割合法性检查：
   assert len(train) + len(test) == len(df), \
       f"训练 + 测试应等于全集，实际 {len(train) + len(test)} vs {len(df)}"
   assert train.index[-1] < test.index[0], \
       "训练集最后一天必须早于测试集第一天，否则数据泄漏"
   assert 0.55 < len(train)/len(df) < 0.65, \
       f"训练集应约 60%，实际 {len(train)/len(df):.1%}"
```

第二条 `train.index[-1] < test.index[0]` 是对**数据泄漏**的机械防线 — 是金融时间序列建模最致命的 bug。

### 改进 4：把"按结果二分支收束"显式化为模板【教学场景】【可学习】

**现状**（结果呈现 第 5 条）：
```
5. 对比结论：
   - 如果最优参数在测试集上比简单定投还差：打印「这就是过拟合——你在历史数据上找到的是巧合，不是规律」
   - 否则：打印「结果看起来还行，但仍需谨慎，样本量有限」
```

**改写**（在二分支前加一句教学注）：
```
5. 对比结论（数据驱动的二分支评论 — 与 spec-03 同款叙事模板）：
   · 用意：无论测试集结果好坏，都收束到"谨慎"的教学结论；
     运气好（测试集刚好不差）的学员看到"还行但谨慎"，运气差的看到"过拟合"
   · 如果 test_ma_return < test_simple_return：「这就是过拟合——你在历史数据上找到的是巧合，不是规律」
   · 否则：「结果看起来还行，但仍需谨慎，样本量有限」
```

把"为什么二分支"写明，让模板可学习而非个例。

### 改进 5：补"为什么不用 KFold / cross-validation"的元说明【教学场景】【动机】

**现状**：spec 没解释为什么用最简单的时间分割

**改写**（加到任务段后）：
```
## 任务

将数据分为训练集和测试集，验证「最优参数」在未见过的新数据上是否仍然有效。

> **教学注**：本节用最简单的"时间前后切分"暴露过拟合，不引入 KFold / Walk-forward / Purged-CV 等专业方法。
> 学员第一次踩坑要简单干净——把"用更复杂方法消除过拟合"留给后续章节。
> 简单暴露过拟合 > 用复杂方法消除过拟合。
```

把教学叙事节奏写进 spec 让 AI 知道"这是有意识的简化"。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 本节是本份 review 给方法/模板章节贡献的"原材料"。36 份 review 全部完成后，按标签横向聚合得到方法目录。

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式** — 与前 6 份一致，是【结构】方法点的第七份证据（q1 章 7 份 spec 全部遵守 = 这是 q1 最强的方法证据）
- **【结构】接续型 spec 链的收束节点**：本 spec 同时调用 `backtest_dca_with_ma` 和 `backtest_dca`，是 q1 接续链的终点收束 — 可作为"接续 spec 的最末节如何同时复用前面多个资产"方法点示范
- **【教学场景：叙事反转】"先伏笔后揭穿"的双 spec 设计**：spec-06 设"参数扫描很美好"的伏笔，spec-07 揭穿过拟合 — 是 q1 章独有的反转模板，可作为"教学型 spec 何时使用'双 spec 反转结构'"方法点的核心范例
- **【教学场景：叙事收束】二分支打印 + 共同收束**：与 spec-03 同款（跑赢 / 跑输都收束到"没有 Alpha"，过拟合 / 还行都收束到"谨慎"）— 是【教学场景】方法点的第二份证据
- **【可学习】用日常类比降低术语门槛**：notebook markdown 里"考试不能用试卷复习"的类比——可作为"金融术语第一次出现配日常类比"方法点示范

### B. 这份 spec 修补后可示范的原则

- **【动机】关键比例（60/40）必须解释**：当前缺位；修复后可作为"教学甜区比例必须显式化"方法点示范，与 spec-06 改进 1（24 个参数甜区）形成方法点的双证据
- **【精确】分割点边界归属显式化**：当前 loc 切片有"两边都包含"的微妙问题；修复后可作为"切片操作必须明确边界归属（用 iloc 整数切片）"方法点示范
- **【验证】数据泄漏机械防线**：当前缺位；修复后是【验证】方法点的最强示范——`train.index[-1] < test.index[0]` 这条 assert 失败会立刻定位最致命的 bug
- **【教学场景】"为什么不用更复杂方法"也要在 spec 里说明** — 当前缺位；修复后是 q1 中【动机】的"反向声明"形式（与 spec-06 改进 2"为什么不做样本外"同形），可独立列出为"教学型 spec 的元说明"方法点

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】"过拟合教学的双 spec 反转结构"**：spec-06（设伏：扫描看起来很美好）+ spec-07（揭穿：测试集失效）。这是 q1 章独有的双 spec 反转设计——本课程必须显式把这种结构作为方法点列出，因为后续章节（q3 因子有效性、q5 风控参数）会反复使用相同的反转模式。
- **【教学场景】金融时间序列不能 shuffle**：通用 ML 教程一上来就 train_test_split + shuffle=True，但金融数据时间相关性强，shuffle 会让训练集"偷看到未来"。本课程必须在 spec-07 这种第一次出现训练 / 测试集分割的场景显式提示——这是数据科学领域知识 0 基础学员必踩的坑。
- **【教学场景】"先用简单方法暴露问题，再用复杂方法解决"的教学节奏**：spec-07 故意用最简单的时间分割（不用 KFold）暴露过拟合，把"如何防过拟合"留给后续章节。本课程必须把这条节奏作为方法点列出——避免学员要求"一次教完所有方法"。
- **【教学场景】数据泄漏（look-ahead bias）的机械防线**：金融 ML 中数据泄漏比传统 ML 更致命（金融数据有强趋势性，泄漏一天可能让回测虚增几倍收益）。`train.index[-1] < test.index[0]` 这种 assert 在量化课程里必须作为标配方法点列出。
