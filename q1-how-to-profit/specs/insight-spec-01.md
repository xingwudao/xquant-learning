## 一、设计意图复盘

这是全书第一份 spec，承担**破冰**作用——学员第一次经历"复制 spec → AI 跑出代码 → 看到自己的图"完整闭环。33 行的篇幅、四段式结构、零金融术语都是这个目标的具象化。【结构】【教学场景】

**为什么选 510300.SS**：原 spec 没说，但合理推断三层：① ETF 而非指数（510300 vs 000300.SH）— ETF 可在二级市场交易，后续回测的"买入"动作有现实意义；② 选 510300 而非同样跟踪沪深 300 的 510310 / 510330 — 流动性最大、成立最久（2012）、5 年数据完整；③ 用 yfinance 而非 akshare — 一行 `download` 取数，q1 阶段不引入额外依赖学习成本。【动机】缺位是这份 spec 的主要薄弱处——三个"为什么"都没写进 spec 主体。

**为什么 start=2021-01-01 不写 end_date**：覆盖一个上下行周期（2021 春节高点 → 2024 谷底 → 2024 末反弹），是后续讲 Beta、定投、过拟合的最低数据需求。但**不写 end_date 是个隐患**——同一份 spec 在 2026、2028、2030 跑出来的数据范围不同，违反【可复现】，是这份 spec 最大单点风险。

**字体三选一**：`Arial Unicode MS / SimHei / STHeiti` 是 mac+Windows 上能显示中文的最小可用集。spec 里按 mac / Windows 分支写，notebook 里实际用单份 fallback 链。两种写法等价，但字面不同违反【同步】。

**结果呈现的克制**：只要"前 5 行 + 时间范围 + 折线图"。没有让学员第一份 spec 就接触 K 线图、技术指标这些"花哨但概念重"的东西，给后续章节留出引入概念的台阶——这是优秀的渐进式教学设计。【教学场景：渐进难度】

## 二、与 notebook 实际产出的对照

| spec 要求 | notebook 实际实现 | 评价 |
|---|---|---|
| macOS Arial Unicode MS 优先 / Windows SimHei | `rcParams['font.sans-serif'] = ['Arial Unicode MS', 'SimHei', 'STHeiti']` 单份配置 | notebook 用 fallback 链统一，实现更优；但 spec 与 notebook 字面不同，违反【同步】 |
| 不指定 figsize | `(12, 5)` | spec 缺失，AI 自行选；下次重跑可能选不同尺寸 |
| 不指定 X/Y 轴标签 | "日期" / "价格（元）" + grid alpha=0.3 + tight_layout | spec 没要求，AI 主动补全 |
| 不要求 progress 关闭 | `progress=False` | AI 自行加，避免输出冗余 |
| `auto_adjust=True, multi_level_index=False` | ✓ | 字符级一致 |
| 起始日期 2021-01-01，不指定 end | ✓ 字面一致 | spec 与 notebook 一致——但 spec 本身是问题（见改进 1） |

**对照结论**：spec 大方向覆盖完整，但"图的细节"和"字体配置"是 AI 自行补全 — 同一份 spec 给不同模型 / 不同时间跑出来的图会有视觉差异。

## 四、可改进点（带改写示例）

### 改进 1：锁死 end_date，消除"时间炸弹"【可复现】

**现状**（spec 第 23 行）：
```
- 起始日期：2021-01-01（获取最近约5年数据），不指定结束日期（默认到最新）
```

**改写**：
```
- 起始日期：2021-01-01
- 结束日期：2026-01-01（本课程数据快照截止日；后续所有图表与回测数字以此为基准）
```

修复成本极低，价值极高 — 学员 2030 年跑出来的图也和书上一致。

### 改进 2：把"为什么"写进 spec【动机】

**现状**：
```
- 标的代码：510300.SS（上交所沪深300ETF）
```

**改写**：
```
- 标的代码：510300.SS — 上交所沪深300ETF
  · 选 ETF 而非指数（000300.SH）：ETF 可在二级市场交易，后续回测的"买入"动作有现实意义
  · 选 510300 而非 510310 / 510330：流动性最大、成立最久（2012）、5 年数据完整
```

让学员**单独阅读 spec 时**就能学到这种判断，而不是依赖 book 侧边栏。

### 改进 3：字体配置直接对齐 notebook 实际写法【同步】

**现状**：
```
- macOS 字体优先使用 Arial Unicode MS，备选 STHeiti
- Windows 字体使用 SimHei
```

**改写**：
```
- 字体配置（fallback 链跨平台自动选第一个可用字体）：
  plt.rcParams['font.sans-serif'] = ['Arial Unicode MS', 'SimHei', 'STHeiti']
  plt.rcParams['axes.unicode_minus'] = False
```

直接给代码片段，spec 与 notebook 字符级一致。

### 改进 4：补一个机械可验证的 assert【验证】

**现状**（要求 第 5 条）：
```
5. 确保索引为 DatetimeIndex，列名包含 Close
```
（只是描述，没有真正验证）

**改写**（移到结果呈现，作为新的第 0 条）：
```
0. 数据合法性检查：
   assert isinstance(df.index, pd.DatetimeIndex), "索引应为 DatetimeIndex"
   assert 'Close' in df.columns, "列名应包含 Close"
   assert len(df) >= 1000, f"数据量异常，仅获取到 {len(df)} 条"
```

失败时 AI 立刻定位问题。

### 改进 5：补 figsize / 轴标签到 spec【精确】【同步】

**现状**：spec 没指定 figsize、X/Y 轴标签、grid alpha
**改写**（加到结果呈现 第 3 条）：
```
3. 画一条收盘价折线图：
   - figsize=(12, 5)
   - 标题：「沪深300ETF (510300) 收盘价走势」
   - X 轴标签：日期；Y 轴标签：价格（元）
   - 添加浅色网格（alpha=0.3）
```

把已经在 notebook 里的细节显式化、可学习。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 本节是本份 review 给方法/模板章节贡献的"原材料"。36 份 review 全部完成后，按标签横向聚合得到方法目录。
> 标签体系（10 个）：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式**：上下文 / 任务 / 要求 / 结果呈现 — 第一份 spec 就严格遵循，可作为方法章节"四段骨架"的原型示例
- **【精确】关键参数全显式**：`auto_adjust=True, multi_level_index=False` — 每个非默认参数点名；反例是"用 yfinance 默认参数"
- **【正向】全部"做 X"**：无 "不要" / "避免" 句式 — 可作为"正向指令"原则的简洁例证
- **【简洁】33 行**：每条要求都"删了就坏" — 可作为"spec 不是越长越好"的低位锚定（与后期 100 行的 spec 形成对照）
- **【教学场景：渐进难度】结果呈现克制**：不引入 K 线 / 指标 / 复杂图，给后续章节留台阶 — 可作为"第一份 spec 应做减法"原则的示范

### B. 这份 spec 修补后可示范的原则

- **【可复现】锁死 end_date**：当前的"不指定结束日期"是反模式；修复后可作为"任何'最近 N 天/年'都是反模式，必须锁死起止日期"的原型 before/after 教学
- **【动机】把"为什么"写进 spec**：当前缺位；修复后可作为"参数选择必须解释，不只是声明值"的示范
- **【同步】spec 直接给代码片段**：当前 spec 用自然语言分支描述字体（mac / Windows），notebook 实际用 fallback 链；修复后可作为"涉及具体配置时 spec 直接给代码 = 跨文件零漂移"的示范
- **【验证】机械 assert 替代描述性"确保..."**：当前只有"确保索引为 DatetimeIndex"的描述；修复后可作为"validation oracle = 失败立刻定位"的示范
- **【精确】图表细节显式（figsize / 轴标签 / 网格）**：当前留给 AI 补；修复后可作为"输出契约必须包含视觉细节"的示范

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】量化场景的"实现细节就是知识"**：选 510300 而非 000300、选 yfinance 而非 akshare、选 `auto_adjust=True` 而非默认 — 这些是数据科学 / 量化的领域知识，**不能像通用 prompt 工程那样"只写做什么不写怎么做"**。本课程必须显式承认这种张力，并教学员"哪些选择是 spec 里要写明的领域信息"。
- **【教学场景】0 基础学员的字体痛点**：matplotlib 中文显示是劝退点之一。本份 spec 把"字体三选一 + axes.unicode_minus=False"写成了固定模板 — 这个具体技巧值得作为"领域必备配置块"在方法章节单独列出（类似"jupyter 中文环境初始化"模板）。
- **【教学场景】跨平台 spec**：mac + Windows 双支持是本课程的硬约束（学员设备分布）。spec 必须默认覆盖两个平台，且差异最小化。这是普通技术教程不需要的额外维度，但本课程必须显式教 — 否则学员仿写时会忘掉非自己平台的兼容。
