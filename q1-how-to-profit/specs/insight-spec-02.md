## 一、设计意图复盘

这是 q1 章的第二份 spec，承担**第一次回测体验**的破冰作用——学员在 spec-01 看到价格曲线之后，第一次看到"投入了多少钱、变成多少钱"的真实数字。33 行的篇幅、把回测逻辑显式拆成 6 个公式步骤是这个目标的具象化。【结构】【教学场景】

**为什么把"每月第一个交易日"作为买入规则**：原 spec 没明说，背后的张力是"日级别买入太密集会让公式繁琐、年级别又太稀疏看不出走势"。月度是教学甜区——对 5 年数据正好出 60 个数据点，足够画出一条平滑收益曲线，又不会让回测公式（份额累加、成本累加）变得难以心算。【动机】缺位是这份 spec 的薄弱处之一——这个"为什么是月而不是周或日"没写进 spec。

**为什么显式定义函数 `backtest_dca(df, monthly_amount=1000)`**：这是接续链的关键设计——spec-05 的 `backtest_dca_with_ma` 会复刻这个签名，spec-07 会在训练 / 测试集上复用 `backtest_dca`。spec 第 3 条明确写"后续步骤会复用此函数"，把"接续意图"前置告知 AI——这是接续型 spec 链的范式写法。【结构】【动机】

**回测公式拆 6 行小步**：spec 第 4 条把回测逻辑拆成"按月分组 → 月份额 → 累计份额 → 累计成本 → 月末市值 → 累计收益率"6 行子条目。这种拆法本质是把"回测"从一个抽象概念降格成 6 个可独立验证的算术步骤，让 0 编程基础学员也能逐行追代码。【可学习】【精确】

**结果呈现的"双图"克制**：只要"持仓市值 vs 累计成本"和"累计收益率"两张图，没有引入 Sharpe、最大回撤等更专业的回测指标。这给 q1 后半部分留出概念引入的台阶——q5/q6 引入风险指标时，学员不会觉得"我刚学完一个新东西又来一个"。【教学场景：渐进难度】

## 四、可改进点（带改写示例）

### 改进 1：明确返回 DataFrame 的索引类型【精确】【同步】

**现状**（要求 第 5 条）：
```
5. 调用函数得到结果：result_dca = backtest_dca(df)，包含列：total_cost, total_shares, portfolio_value, return
```

**改写**：
```
5. 调用函数得到结果：result_dca = backtest_dca(df)
   - 索引：PeriodIndex（按月，例如 2021-01）
   - 列：total_cost, total_shares, portfolio_value, return
   - 后续画图时调用 result_dca.index.to_timestamp() 转换为时间戳
```

把"PeriodIndex"显式化，spec-03/05 的画图代码就不会因索引类型不一致出 bug。

### 改进 2：把"为什么是月度"写进 spec【动机】

**现状**：
```
2. 策略规则：每月第一个交易日，用 1000 元按收盘价买入
```

**改写**：
```
2. 策略规则：每月第一个交易日，用 1000 元按收盘价买入
   · 选月度而非周/日：月度对 5 年数据约 60 个点，画图能看出走势；公式拆 6 步对 0 编程基础学员心算友好
   · 选 1000 元：整数好心算（5 年总投入 60×1000=6 万元）；具体数额对收益率结果不影响（线性缩放）
```

### 改进 3：补机械可验证的 assert【验证】

**现状**（要求 第 5 条）：只描述返回列名

**改写**（加到结果呈现 第 0 条）：
```
0. 数据合法性检查：
   assert isinstance(result_dca, pd.DataFrame), "应返回 DataFrame"
   assert {'total_cost', 'total_shares', 'portfolio_value', 'return'}.issubset(result_dca.columns)
   assert len(result_dca) >= 50, f"5 年应至少 50 个月，仅得到 {len(result_dca)} 个"
   assert result_dca['total_cost'].iloc[-1] == 1000 * len(result_dca), "累计成本应等于月数 × 1000"
```

最后一条断言是回测正确性的强约束 —— 失败立刻定位"成本累计错了"。

### 改进 4：补打印格式与图例细节【精确】【同步】

**现状**：结果呈现 第 1 条"打印 5 个数字"
**改写**：
```
1. 打印（每行一个）：
   - 投资期间: {起始月} 到 {结束月}
   - 投资月数: {N} 个月
   - 总投入金额: {金额:,.0f} 元
   - 最终市值: {金额:,.0f} 元
   - 最终收益率: {百分比:.2%}
2. 画两张图（上下排列，figsize 12×8）：
   图1 标题"持仓市值 vs 累计成本"，Y 轴"金额（元）"，图例 + grid alpha=0.3
   图2 标题"累计收益率"，Y 轴"收益率 (%)"，零线 axhline(y=0)
```

### 改进 5：在 spec 内提示函数会被后续复用【结构】【可学习】

**现状**：
```
3. 将回测逻辑封装为函数 backtest_dca(df, monthly_amount=1000)，返回 DataFrame（后续步骤会复用此函数）
```

这条已经做到了，但可以更显式：
```
3. 将回测逻辑封装为函数 backtest_dca(df, monthly_amount=1000)
   · 后续 spec-05 会基于此函数派生 backtest_dca_with_ma（保持签名兼容）
   · 后续 spec-07 会在训练/测试集上分别调用此函数
```

让学员**单独阅读 spec 时**就理解"为什么要封装函数"——是为了接续，不是为了"看起来专业"。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 本节是本份 review 给方法/模板章节贡献的"原材料"。36 份 review 全部完成后，按标签横向聚合得到方法目录。

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式**：上下文 / 任务 / 要求 / 结果呈现 — 与 spec-01 完全一致，可作为四段式"在第二份 spec 上复用"的范本
- **【结构】接续型 spec 链的开端**：要求第 3 条"后续步骤会复用此函数"是接续链的显式声明，让 AI 在写代码时就预留接口 — 这是接续型 spec 链方法的核心范式
- **【精确】回测公式拆 6 行小步**：spec 第 4 条把回测拆成"分组 / 月份额 / 累计份额 / 累计成本 / 月末市值 / 收益率"6 行 — 可作为"复杂逻辑要拆成可逐行验证的子步骤"原则的示范
- **【正向】全部"做 X"**：要求段没有"不要" / "避免" — 与 spec-01 一脉相承
- **【教学场景：渐进难度】只要双图**：不引入 Sharpe / 最大回撤等专业指标 — 给后续章节留台阶

### B. 这份 spec 修补后可示范的原则

- **【精确】返回值的索引类型也要写明**：当前 PeriodIndex 是隐性约定；修复后可作为"DataFrame 契约不仅要写列名，还要写索引类型 / 频度"的示范
- **【动机】参数选择必须解释**：当前缺"为什么 1000 元 / 为什么月度"；修复后与 spec-01 改进 2 同源，是【动机】方法点的第二份证据
- **【验证】assert 替代描述性"包含"**：当前只描述列名；修复后可作为"validation oracle = 失败立刻定位"的第二份示范（与 spec-01 改进 4 形成方法点的第二份证据）
- **【同步】打印格式 / 轴标签显式化**：当前留给 AI 补；修复后可作为"输出契约必须包含视觉细节"的第二份证据

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】函数封装作为接续设计**：q1 章的 7 份 spec 是一条接续链——spec-02 的 `backtest_dca` 流到 spec-05 派生 `backtest_dca_with_ma`，再流到 spec-07 在训练/测试集分别调用。**这种"前 spec 输出 = 后 spec 输入"的接续链是量化课程特有的结构**，不是通用 prompt 工程的内容。本课程必须显式教"什么时候在 spec 里封装函数 = 为后续 spec 准备接口"。
- **【教学场景】月度回测频度的教学甜区**：日级别回测要处理停牌、分红等边界条件；年级别又看不出策略动态。月度是 0 基础学员的甜区——本课程必须显式承认这种张力，教学员"教学场景下选频度的判断标准"。
- **【教学场景】PeriodIndex vs DatetimeIndex 的隐性约定**：pandas 这两种索引在画图时不互通（PeriodIndex 要 `.to_timestamp()`）。这是数据科学领域知识，0 基础学员第一次遇到会很困惑。spec 里把"返回索引类型"写明是规避这个陷阱的关键。
