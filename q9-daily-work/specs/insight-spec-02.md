## 一、设计意图复盘

这是 q9 的核心 spec——也是全书数学密度最高的一份。任务包含**双因子 × 三窗口 = 6 组 IC + 按波动率分组的 IC 对比**，是 spec-01 的"GDP 单因子单窗口"的 6 倍复杂度。spec 用四步阶梯化要求（25 标的池→因子矩阵→6 组 IC→分组 IC）让任务可拆——这是收官章对"复杂任务必须分层"的成熟示范。【结构】【可学习】

**为什么扩到 25 只标的**：spec 没明说，合理推断三层：① 12 国 ETF 矩阵（截面 ≥ 12）才能稳定算 Spearman——`compute_grouped_ic` 第 444 行的 `if len(c) < 12` 就是这个约束的反向证据；② 加 3 个大宗商品（黄金/原油/白银）扩大波动率谱系，让"高波动 vs 低波动"分组有显著差异；③ 25 是教学场景的合理上限——再多就管理负担过重。这三层"为什么"在 spec 主体中**全部缺位**——是这份高分 spec 仍然失分的主要点。【动机】缺位

**`compute_ic_series` 函数签名的精确度**：spec 要求 4 把函数签名写到位（`compute_ic_series(factor_df, fwd_df, min_assets=5)`），并显式说明"对每个交易日，取因子值和前向收益的交集标的（至少 min_assets 个）"——这是把"截面 IC"的逻辑用一行代码契约固化。`min_assets=5` 这个具体值不是凭空选——和 spec-01 GDP 因子的 `len(common) >= 4` 形成同一族阈值（小到刚够算 Spearman、大到能稳定）。【精确】

**`min_assets=12` 在分组 IC 中的提升**：spec 要求 5 把分组 IC 的"交集标的至少 12 个"和"每组至少 6 个"分开。这是金融常识——分组后样本减半，单组 6 个仍能算 Spearman 但稳定性下降，因此交集要先够 12。spec 把这个金融工程经验显式化了，但**没解释为什么从 5 跳到 12**——这是【动机】的一个具体失分点。

**前向收益窗口 [10, 20, 40] 的几何级数选择**：spec 要求 3 写 `[10, 20, 40]`——这是一个**有意识的几何级数**（不是 [10, 15, 20]），覆盖"短线交易→中线持有→月度调仓"的差异化时间尺度。20 日是后续两份 spec 共用的"主窗口"（spec 要求 4 末尾保留 `fwd_df = fwd_dfs[20]` 给 spec-03 用）。这种"窗口选择即接续约定"的设计，spec 没有显式标注——是【结构】【可学习】的一个微小遗憾。

**2×3 子图布局的克制选择**：spec 要求 4 末尾"画 2×3 子图（figsize 18×8）：每格画 IC 柱状图（width=2, alpha=0.2, steelblue）+ 60 日滚动均值曲线（darkblue）+ IC 均值红色虚线"——把 6 组 IC 一次性可视化。`alpha=0.2` 让柱状图退到背景、`darkblue` 滚动均值线突出，是教过 6+ 子图组合后才会有的 "信息层级"考虑。【精确】

## 四、可改进点（带改写示例）

### 改进 1：锁数据快照日，明示"为什么固定到 2025-12-30"【可复现】【动机】

**现状**（要求 2）：
```
- 时间范围：2015-01-01 至 2025-12-30
- 下载 end：2025-12-31（yfinance end 为半开区间）
```

**改写**：
```
- 固定分析窗口：2015-01-01 至 2025-12-30
- 下载 end：2025-12-31（yfinance end 为半开区间）
- 跨度选 ~10 年，理由：① 覆盖 2015 牛/2016 商品熊/2020 疫情/2022 加息/2024 反弹四个周期，
  样本量足以让 60 日滚动均值线（要求 4）平滑；② 25 标的 × ~2700 交易日 ≈ 68k IC 计算，
  够稳定但不慢
```

时间锁 + 跨度动机一并解决。

### 改进 2：把 `min_assets` 阈值的"为什么"写进 spec【动机】

**现状**（要求 4 / 要求 5）：
```
- min_assets=5
- 要求交集标的至少 12 个，每组至少 min_assets=6 个
```

**改写**：
```
- min_assets=5（整体 IC：Spearman 在 ≥5 时方差可控；<5 单点噪音过大）
- 分组 IC：交集 ≥12（分组后每组才能 ≥6），每组 min_assets=6
  阈值依据：金融工程惯例——分组样本 ≥6 才能稳定算 Spearman
```

让学员**仿写时换标的池就知道阈值要不要调**。

### 改进 3：补 IC 数值合法性 assert【验证】

**现状**（验证段）：
```
- IC 对比表有 6 行（2 因子 × 3 窗口），IC 均值绝对值在合理范围（接近 0）
```

**改写**：
```
- 表格行数：assert len(ic_results) == 6
- IC 数值合法：assert all(-1 <= ic_ts.dropna().min() and ic_ts.dropna().max() <= 1 
  for ic_ts in ic_results.values())
- 25 标的全加载：assert len(all_data) == 25
- 分组 spread 方向：assert high_ic.mean() < low_ic.mean()  # 高波动组 IC 应低于低波动组
```

最后一条把 spec 验证段的"显示高波动组 IC 低于低波动组"从描述升级为机械 assert。

### 改进 4：标注 `fwd_df = fwd_dfs[20]` 是"接续约定"【结构】【可学习】

**现状**（要求 4 末尾）：
```
- 保留 ic_ts = ic_results["风险调整动量 → 20日"] 和 fwd_df = fwd_dfs[20] 供后续使用
```

**改写**：
```
- 保留供 spec-03 使用的接续变量：
  · ic_ts = ic_results["风险调整动量 → 20日"]（spec-03 的 RAM IC 基线）
  · fwd_df = fwd_dfs[20]（spec-03 计算波动率/反转因子 IC 时复用 20 日窗口）
  · vol_df（spec-03 的"波动率因子"直接复用，无需重算）
  · all_data（spec-03 反转因子 -df["close"].pct_change(5) 直接复用 25 标的数据）
  说明：20 日是全章主窗口；选择"中等持有期"避开短线噪音和长线漂移
```

显式化"哪些变量是 spec-02→spec-03 的接续约定"——让学员看清这条接续链。

### 改进 5：要求 1 标题 markdown 加副标题，提示读者"这是核心实验段"【教学场景】

**现状**（要求 1）：


**改写**：

```
创建 Step 2 标题 markdown，主题"动量因子——涨得好的会继续涨吗？"，副标题
```

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【精确】函数签名级精度**：要求 4 把 `compute_ic_series(factor_df, fwd_df, min_assets=5)` 写到位、要求 5 把 `compute_grouped_ic(factor_df, fwd_df, vol_df, min_assets=6)` 写到位——AI 几乎没有自由发挥空间。可作为"spec 给函数签名 = AI 实现一致"的示范
- **【精确】可视化视觉权重显式**：`width=2, alpha=0.2, color="steelblue"`（背景柱）+ `darkblue`（前景滚动均值）+ `red, linestyle="--"`（基准线）——三层视觉层级全点名。可作为"复杂图表必须细节到 alpha"的示范
- **【结构】复杂任务的阶梯化拆分**：6 个 cell 的 Step 2 拆成 4 步阶梯（构池→因子矩阵→6 组 IC→分组 IC），每步前一步的输出是下一步的输入。可作为"复杂任务 spec 要分层、不要一锅炖"的示范
- **【验证】方向性 assert**：验证段"分组 IC 表显示高波动组 IC 低于低波动组（spread > 0）"——不是数值范围，而是方向的机械检验。可作为"方向性指标的验证写法"的示范
- **【同步】13/13 字符级一致**：连 `width=2`、`alpha=0.2`、颜色码、函数签名都和 notebook 完全一致——是收官章应有的同步水平。可作为"高复杂度 spec 也能做到字符级同步"的标杆
- **【可学习】接续变量显式保留**：要求 4 末尾"保留 ic_ts 和 fwd_df 供后续使用"——spec-02→spec-03 的接续点显式声明。可作为"跨 spec 接续链 = 末尾显式 export"的示范

### B. 修补后可示范的原则

- **【可复现】"到次年末"是反模式**：分析窗口必须固定到实际最后纳入的交易日；下载 end 可以使用下一自然日表达 yfinance 半开区间。修复后可作为"任何'到当年/次年末'都必须改成快照日"的强证据（与 q1/insight-spec-01、q9/insight-spec-01 形成连续证据链——同标签三票，应该升级为方法章节的核心原则）
- **【动机】阈值数字必须解释**：`min_assets=5/6/12` 三个值在 spec 中都是天降数字。修复后可作为"任何参数阈值必须配一句出处"的示范
- **【动机】"为什么是这个标的池/这个窗口"必须写进 spec**：当前 spec 没解释 25 标的、`[10,20,40]` 窗口的来历。修复后可作为"标的池/窗口选择 = 教学知识点，必须显式"的示范
- **【可学习】接续变量除了"保留"还要"标注用途"**：当前只说"保留 ic_ts 和 fwd_df 供后续使用"，没说后续怎么用。修复后可作为"接续变量必须配下游用途说明"的示范

### C. 领域 / 教学场景特殊考量

- **【教学场景】IC 计算的"截面工程化"**：`compute_ic_series` 这个函数把"对每天求一次 Spearman、跳过样本不足的天、收集成时间序列"的工程化模式封装成一个函数——这是量化研究员的 daily routine。可作为"领域必备模式 = 让 AI 写出能复用的函数而非散装代码"在方法章节单列
- **【教学场景】分组 IC 的诊断价值**：要求 5 用波动率分组——把整体 IC 接近 0 的"看起来无效"因子拆解成"高波动组 IC 负 vs 低波动组 IC 正"的"分场景有效"。这是量化研究员"找问题、定位失效原因"的核心工具。spec 把这种诊断模式显式教给学员，是收官章独有的高级内容
- **【教学场景】2×3 子图布局的"教学剂量"**：6 组 IC 一次性 2×3 排版，让学员一眼看到"窗口越长 IC 越稳但越小、双因子在 20 日窗口最优"等多个结论。这种"信息密度极高的单图"是量化研究报告的典型——教学中要训练学员"读这种图"的能力
- **【教学场景】函数 + 数据矩阵的"研究模板"**：spec 把"build factor_df → build fwd_df → 调用 compute_ic_series 生成 ic_ts → 可视化"这条链固化为模板——任何新因子来了都按这个模板跑一遍。可作为"领域研究模板 = spec 训练学员形成习惯动作"的示范
