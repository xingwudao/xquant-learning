## 一、设计意图复盘

这是 q9 的收官 spec——也是**全书的收官 spec**。任务是"把 spec-02 发现的因子失效模式（高波动时动量 IC 为负）转化为可回测的工程化策略改进"，跨越**因子研究 → 策略工程 → 回测对比**三层。spec 综合应用了 q3（仓位）、q4（调仓）、q5（验证）的所有积累——是全书"综合特性"的最强证据。【结构】综合特性

**为什么从 25 标的切回 Q3 三只标的**：spec 没明说，合理推断：① Q3 三只标的（沪深300/纳指100/黄金）是"两层选择框架"的下层——上层 GDP 因子已经选出中美两大经济体作为候选池；② Q3 是学员熟悉的环境，避免"换标的+换策略"双变量混淆；③ 三标的回测计算量小，让学员能多次迭代调阈值。三层动机在 spec 主体中**全部缺位**——这是综合应用 spec 的常见失分点。【动机】缺位

**`VolFilteredOptimizer` 包装类的设计选择**：spec 要求 7 不让学员从头写 optimizer，而是写一个**包装类**——保留 `base_optimizer` 引用、复用 `required_indicators`、只在 `optimize()` 末尾"打补丁"。这是软件工程的"装饰器模式"在 spec 中的具象化。spec 用"先调用 base、再判断市场波动率、超阈值则减半权重转 CASH"的步骤化文字，避免直接给代码——是【抽象层次恰当】的好示范。但 spec **没解释为什么用包装而不是修改 TopNRankingOptimizer**——这是【动机】的具体失分。【精确】

**"市场平均波动率中位数作为阈值"的克制选择**：spec 要求 7 写"波动率阈值取市场波动率的中位数"——这是一个**故意保守**的选择：50% 的天数判定为"高波动"——意味着策略一半时间在减仓。这种"过滤太狠"的设定不是 bug 而是教学设计：让学员看到"过滤会大幅减少夏普 vs 显著降低回撤"的张力。但 spec 没解释这种张力——学员仿写时把阈值改到 75 分位会得到完全不同的结论却不知道为什么。【动机】缺位

**"先讲 ML 后做工程"的节奏**：spec 要求 3 把"机器学习组合（决策树/随机森林、XGBoost/LightGBM、神经网络）"作为"未来方向"显式列出——不让学员现在学，但点明方向。这是收官章对"教学边界"的处理：承认课程边界、但给学员未来路径。【教学场景：教学边界】

**双策略对比的克制可视化**：spec 要求 8 只画一张净值曲线对比图（基准蓝 + 过滤橙），不引入更复杂的"逐月超额收益"或"滚动夏普"——避免在"全书最后一张图"叠加新概念。【教学场景：渐进难度】

**oxq 模块清单的扩张**：spec 要求 5 列出 9 个 oxq 模块（`Engine`、`Strategy`、`Momentum`、`RollingVolatility`、`Ratio`、`TopNRankingOptimizer`、`Threshold`、`SimBroker`、`StaticUniverse`、`RebalanceFrequencyRule`），是评审三份中模块数最多的——回测策略需要的全套组件。这条要求显式遵循"先读后写"原则。【同步】【教学场景：oxq】

## 四、可改进点（带改写示例）

### 改进 1：消除 `pd.Timestamp.now()`【可复现】

**原问题**（要求 6）：
```
- 起始日期 2021-01-01，结束日期用 pd.Timestamp.now().strftime("%Y-%m-%d")
```

**改写**：
```
- 起始日期 2021-01-01
- 结束日期 2026-03-26（复现书中结果的实际数据截止日）
- 下载 end 2026-03-27（yfinance end 为半开区间）
- 注：不要使用 pd.Timestamp.now() ——每天跑结果不同，违反可复现原则
```

`pd.Timestamp.now()` 让 vol_threshold（中位数）依赖回测当天的数据范围——同一份 spec 在不同日期跑出来的"高波动天数占比"会不同，对比图也会变。这是评审三份中最严重的可复现违反。

### 改进 2：把"为什么阈值取中位数 / 为什么减半"写进 spec【动机】

**现状**（要求 7）：
```
- 波动率阈值取市场波动率的中位数
- 若超过阈值则将所有持仓权重减半、多余部分分配给 CASH
```

**改写**：
```
- 波动率阈值：市场波动率中位数
  · 选中位数（不是 75 分位）：故意"过滤狠"——50% 时间减仓，让学员看到
    "过滤会显著降低夏普 vs 显著降低回撤"的张力（这正是因子评估的实战教学价值）
  · 学员可以自行调到 60/75 分位看效果变化
- 减半权重（不是 0.7×/0.3×）：
  · 是教学场景的简化选择——既不全清仓（过激）也不微调（无效），便于学员看见对比效果
  · 实战中应做参数扫描（参考 q5 验证方法）找最优系数
```

让学员**仿写时知道哪些是教学简化、哪些是工程必要**。

### 改进 3：要求 7 末尾的 `optimize` 描述更明确"取哪个 sym"【精确】

**现状**：
```
- optimize 方法：先调用 base optimizer 获取权重，再从 indicators 中取最新日期查 market_vol
```

**改写**：
```
- optimize 方法：
  1. 先调用 self.base.optimize(signals, indicators) 获取权重
  2. 取 indicators 中第一个 sym 的最新日期（所有 sym 同步推进，取任一即可）作为 latest_date
  3. 查 self.market_vol.loc[latest_date]——若 NaN 跳过、若 ≤ vol_threshold 直接返回 base 权重
  4. 若 > vol_threshold：所有非 CASH 权重 ×= 0.5、减少部分追加到 CASH
```

把"取一个 sym 的最新日期"显式化——避免 AI 在 "for sym ... break" 和 "max(dates)" 间随机选。

### 改进 4：拆分要求 3/4 的文档型条目【简洁】

**现状**：要求 3（因子评估发现 markdown）+ 要求 4（策略改进方向 markdown）共 12 行
**改写**：把要求 3 和 4 合并为单条要求 3："创建 markdown 单元格群（因子评估发现 + 策略改进方向）"，子条目用列表
- 三因子各有侧重 + 关键发现（高波动 IC 负）
- ML 组合是未来方向（决策树/XGBoost/神经网络）
- 当前用最简单方式：基于发现加波动率过滤
- 两层选择框架（GDP→候选池，动量→择时）

要求 9 条 → 8 条，主线更聚焦。

### 改进 5：补"过滤策略回撤改善"的方向性 assert【验证】

**现状**（验证段）：
```
- 基准策略和波动率过滤策略均有有效回测结果（累计收益、夏普等非 NaN）
- 净值曲线图显示 2 条线
```

**改写**：
```
- 数值合法：assert all 4 metrics（cum_ret/vol/dd/sharpe）非 NaN，每个策略
- 方向性验证（核心命题）：assert abs(result_vf.max_drawdown()) < abs(result_base.max_drawdown())
  说明：spec 的核心命题是"波动率过滤改善回撤"——这条 assert 失败意味着教学结论站不住
- 视觉验证：净值曲线图有 2 条线（蓝 #3498DB / 橙 #E67E22），蓝线在橙线"高波动期"上方
```

把 spec 的核心命题（"过滤改善回撤"）固化为机械检验。

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【结构】综合应用 spec 的"问题→工程→对比"三段式**：要求 2-3（补因子 + 发现）→ 要求 5-7（工程实现）→ 要求 8（对比验证）。这种"研究发现 → 工程改进 → 回测对比"的三段式是综合应用 spec 的模板。可作为"收官章 / 综合应用 spec 的结构原型"
- **【精确】包装类的步骤化描述（不给代码）**：`VolFilteredOptimizer` 用"先调 base、再打补丁"的文字描述包装模式——既给约束又不越权写代码。可作为"复杂类设计 = 描述步骤而非给代码"的示范
- **【同步】oxq 模块清单显式列出**：要求 5 列出 9 个 oxq 模块——是评审三份中最完整的"先读后写"清单。遵循 q2+ 规范
- **【教学场景】教学边界显式声明**：要求 3 把"机器学习组合"显式列为"未来方向"——承认课程边界、给学员路径。可作为"收官章 spec 应该有的教学边界声明"
- **【教学场景】渐进难度的最后一图**：要求 8 只画一张净值曲线对比图，不叠加滚动夏普/超额收益图。这种"最后一图保持简洁"是渐进难度的尾段示范
- **【可学习】综合应用 spec 整合多章接续**：spec 整合了 spec-02 的 vol_df/all_data + q3 的标的池 + q4 的调仓周期 + q5 的回测验证——展示"如何把多份 spec 的产出串起来"。可作为"综合应用 spec = 多接续链汇聚"的示范

### B. 修补后可示范的原则

- **【可复现】禁用 `pd.Timestamp.now()` / `today` 等运行时日期**：当前 spec 已改为固定 `Q3_ANALYSIS_END = "2026-03-26"`。修复后可作为"任何运行时日期 = 反模式"的强证据。这与 q1/insight-spec-01、q9/insight-spec-01、q9/insight-spec-02 形成**同标签四票连续证据**——【可复现】应升级为方法章节的核心原则
- **【动机】教学性参数（阈值、系数）必须标注"为什么这个值"**：当前 `0.5×` 减半、阈值取中位数都是天降数字。修复后可作为"教学场景的参数选择 = 必须显式说明是简化、是过激或是惯例"的示范
- **【验证】综合应用 spec 必须有"核心命题方向 assert"**：当前验证段只有"非 NaN"和"图有 2 条线"。修复后可作为"综合应用 spec 的核心命题（过滤改善回撤）必须固化为机械检验"的示范
- **【精确】实现细节模糊处必须二选一明示**：`optimize` 方法的"取最新日期"在 spec 中模糊，AI 选了"取第一个 sym 后 break"——功能对但 spec 应明示。可作为"实现细节有歧义时 spec 必须二选一"的示范

### C. 领域 / 教学场景特殊考量

- **【教学场景】"因子→策略"工程化模式**：spec 要求 7 的 `VolFilteredOptimizer` 包装模式——把"因子评估发现的失效条件（高波动）"翻译成"工程化的过滤规则"。这是量化研究员从研究到生产的核心动作。可作为"领域工程化模式 = spec 训练学员的核心 routine"在方法章节单列
- **【教学场景】综合应用 spec 的"两层选择框架"**：要求 4 的"上层 GDP 选候选池 / 下层动量做择时"——这是 q1-q9 全书的总主线。spec 把全书结构在最后显式化。可作为"收官 spec 应该有的总主线呼应"的示范
- **【教学场景】回测对比的"故意保守"教学设计**：阈值取中位数（过滤太狠）、减半权重（不极端）是教学场景的简化选择。spec 应该（修补后）显式声明这一点——避免学员把教学简化误认为最优工程。可作为"教学简化 vs 工程最优 = 必须区分"的示范
- **【教学场景】"研究发现 → 策略改进"的因果链显式化**：spec 把"spec-02 发现高波动时动量 IC 负 → spec-03 加波动率过滤 → 回测看回撤改善"这条因果链写在要求 4 和小结里——不让学员独立悟。这种"研究→工程"因果链的显式化是收官章独有的教学动作
- **【教学场景】oxq 综合用法清单**：9 个 oxq 模块（Engine/Strategy/Momentum/RollingVolatility/Ratio/TopNRankingOptimizer/Threshold/SimBroker/StaticUniverse/RebalanceFrequencyRule）一次性出现——是 oxq 用法的最完整展示。可作为"oxq 综合应用 spec 的模块清单模板"
