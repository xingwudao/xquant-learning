## 一、设计意图复盘

这是 q2 三资产筛选的**第三步也是收官 spec**——把"中美都是股市可能同跌"的开放问题用相关性数据闭环掉，并用 `StaticUniverse` 把"标的池"概念具象化。spec-03 是 q2 的**理论合龙点**：① 验证黄金作为对冲资产的低相关性；② 用累计收益率/波动率/最大回撤的对照表证明组合优于单押；③ 引出"标的池"这个 q3+ 会反复用到的核心概念。三件事压在一份 spec 里。【结构】【教学场景】

**为什么用 510300/513100/518880 三资产组合**：完成 q2 的"一中一美一对冲"叙事——沪深300（中国股市）+ 纳指100（美国股市）+ 黄金（避险资产）。三是教学最少数：少于三无法体现"分散"，多于三引入解释成本。spec 没解释这层选择，但比 spec-01/spec-02 好的是——上下文段已经把"中美都是股市可能同跌、加什么能保护组合"的问题立起来，黄金的引入有了 narrative。

**为什么用 numpy 做年化波动率**：要求第 2 条特别说"导入 numpy（年化波动率计算需要）" — 这个"为什么需要"的小注释是 spec 的亮点，体现 spec-writing-guide 原则 1「自包含」。但**没说为什么是 √252**（年交易日数）这个常识，对 0 基础学员仍是黑箱。

**与 spec-02 的接续清晰**：spec-03 上下文段明确写"已有 510300 + 513100 数据" + "学员已通过 GDP 锁定中美" + "当前问题：中美都是股市可能同跌"——这是**整套 q2 三 spec 中接续最清晰的一份**。比 spec-02 的"默认 spec-01 已创建实例但不明说"进步明显。但仍漏写一个细节：是否复用 spec-02 已经创建的 `downloader/provider`。

**最大问题——spec 把"组合 vs 单押"的全套指标计算硬写**：要求第 8 条说"计算两种方案的日收益率序列 → 计算并对比累计收益率、年化波动率、最大回撤"——三个金融指标的计算逻辑没在 spec 里说，要求第 6 条只说"计算日收益率 (pct_change)"也没说什么是 pct_change。学员 0 金融基础时，这三个术语的"机械计算"是知识黑箱。spec 要么应该写明计算公式（"最大回撤 = (累计净值 / 累计净值的历史最大值 - 1) 的最小值"），要么应该指示 AI 用 oxq 已有的指标计算函数（如 `result.max_drawdown()`） — 这正是 spec-01 没让 AI 先读 oxq 源码的延续问题。【教学场景】【可学习】

**spec 内嵌大段 markdown 表格的反模式**：结果呈现第 3 条直接给了一份 markdown 表格模板（"|指标|单押沪深300|三资产等权组合|"）—— 这是 spec 反模式 #12「spec 内嵌大段代码」的边缘情况。表格是输出契约的一部分有合理性，但具体的小数位、对齐方式、是否带百分号都是 AI 自补——其实 spec 应该给 `pd.DataFrame.to_string()` 的具体调用，不是 markdown 字符串。

## 四、可改进点（带改写示例）

### 改进 1：锁死 end_date 与 as_of_date【可复现】【同步】

**现状**：
```
3. ... 起始日期 `2021-01-01`，结束日期为当天
10. ... snapshot = universe.get_universe(as_of_date="2026-02-28")
```

第 3 条用 today、第 10 条写死 2026-02-28——同一份 spec 内时间锚不一致。notebook 实际**两处都用 today**，spec/notebook 反向漂移。

**改写**：
```
3. ... 起始日期 `2021-01-01`，结束日期 `2026-01-01`（与 spec-01/spec-02 数据快照截止日一致）
10. ... snapshot = universe.get_universe(as_of_date="2026-01-01")
```

并把 notebook 的 `as_of_date=today` 改回字面锁定。

### 改进 2：金融指标的计算路径明示【教学场景】【可学习】

**现状**（要求 第 8 条）：
```
- 计算并对比：累计收益率、年化波动率、最大回撤
```

**改写**：
```
8. 对比实验——组合 vs 单押：
   - 单押方案：100% 沪深300（单押日收益率即 returns["沪深300"]）
   - 等权组合：三资产日收益率取算术平均（returns.mean(axis=1)）
   - 计算三个指标（先读 oxq/metrics.py 看是否有现成函数；若没有，按下列公式计算）：
     · 累计收益率：(1 + 日收益率).prod() - 1
     · 年化波动率：日收益率.std() * sqrt(252)（252 是年交易日数）
     · 最大回撤：累计净值 / 累计净值的历史最大值 - 1，取最小值
```

公式写明 + 让 AI 先读 oxq 看有没有现成函数，避免学员仿写时被金融术语绊倒。

### 改进 3：表格输出契约用代码片段而非 markdown【同步】【精确】

**现状**（结果呈现 第 3 条）：
```
| 指标 | 单押沪深300 | 三资产等权组合 |
|------|------------|---------------|
| 累计收益率 | X.XX% | X.XX% |
...
```

**改写**：
```
3. 用 pd.DataFrame 构造对比表并打印：
   metrics = pd.DataFrame([
       {"方案": "单押沪深300", "累计收益率": f"{cum_return:.2%}", ...},
       {"方案": "三资产等权组合", ...}
   ])
   print(metrics.to_string(index=False))
```

字面与 notebook 一致，且可执行——避免 spec/notebook 字符级漂移。

### 改进 4：要求开头加「先读 oxq 源码」指令【教学场景】

**现状**：spec-03 用到 `LocalMarketDataProvider.get_bars` 和 `StaticUniverse.get_universe` 两个 oxq 接口，但都没让 AI 先读源码。

**改写**（在「## 要求」开头插入第 0 条）：
```
0. 本 spec 用到 open-xquant 的两个新接口。先阅读：
   - `oxq/data/local.py`：`LocalMarketDataProvider.get_bars(symbol, start, end)` 返回值结构
   - `oxq/universe/static.py`：`StaticUniverse` 构造与 `get_universe(as_of_date)` 返回的 snapshot 结构
   参考 https://github.com/xingwudao/open-xquant/tree/main/examples/tutorials
```

### 改进 5：补机械断言【验证】

**现状**：只有 `snapshot.symbols` 一处可机械验证。

**改写**（结果呈现 加新的第 0 条）：
```
0. 数据合法性检查：
   assert corr.shape == (3, 3), "相关性矩阵应为 3x3"
   assert (corr.values.diagonal() == 1.0).all(), "对角线应为 1.0"
   assert -1 <= corr.values.min() and corr.values.max() <= 1, "相关系数应在 [-1, 1]"
   assert set(snapshot.symbols) == {"510300.SS", "513100.SS", "518880.SS"}, "标的池应包含 3 只 ETF"
```

最后一条特别重要：明确标的池的"成员断言"，是 q3-q9 反复使用 `universe` 对象时学员能"机械确认无遗漏"的基线。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 标签体系：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式** + **三件事的顺序编排清晰**：相关性分析 → 对比实验 → 标的池构建——三件事虽然多，但按"分析 → 对照 → 沉淀概念"的认知顺序排列，叙事弧线不乱。可作为"多任务 spec 的顺序编排"示范
- **【精确】配色 + figsize + 标题全显式**：与 spec-01/spec-02 一致，证明该原则在三份 spec 上稳定执行
- **【动机】上下文段把开放问题立清楚**：上下文段写"中美都是股市可能同涨同跌，加什么能保护组合"——这是上一份 spec（spec-02）的结尾问题，spec-03 接续时把问题写在上下文段里。可作为"接续型 spec 链中上文问题在下文上下文段重述"的示范
- **【教学场景：先猜后验】**：与 spec-02 一致——结果呈现的打印里有"黄金常被称为避险资产 → 数据说了算：分散风险靠不同步" 的"假设 → 验证"闭环
- **【教学场景】"概念落地"型 spec 收官**：要求第 10 条用 `StaticUniverse` 把 q2 整章探讨的"标的池"概念具象化为可调用对象——这是从"教学概念 → 框架对象"的过渡。可作为"理论合龙 spec"的示范
- **【正向】全部"做 X"**：无否定堆叠

### B. 这份 spec 修补后可示范的原则

- **【可复现】同份 spec 内时间锚一致**：当前价格端用 today、universe 端写死 2026-02-28、且 notebook 反向漂移——三处不一致。修复后可作为"任何时间相关锚点必须全 spec 统一锁定"的示范，证据强度极高（一份 spec 内三处反例）
- **【教学场景】金融指标的计算路径明示**：当前"计算累计收益率/年化波动率/最大回撤"是黑箱，0 基础学员仿写易卡。修复后可作为"领域术语必须给可执行公式或指向框架现成函数"的示范
- **【同步】markdown 表格 → DataFrame.to_string 的转换**：当前 spec 内嵌 markdown 表格、notebook 用 `to_string` — 两种格式字面级不可能同步。修复后可作为"输出契约必须用可执行代码片段而非 markdown"的示范
- **【教学场景】oxq 引入期"先读后写"**：与 spec-01、spec-02 同——三份 spec 均未让 AI 先读 oxq 源码。修复后可作为该原则的示范，且**三份 spec 的同一缺失证据强度最高**

### C. 领域 / 教学场景特殊考量

- **【教学场景】"理论合龙"型 spec 模板**：spec-03 不是"做事"也不是"立论"，而是"把整章探讨的概念落地到框架对象"。其特征：① 先把开放问题（"加什么能保护组合"）数据化解决；② 用对照实验验证选择（组合 vs 单押）；③ 用框架对象（`StaticUniverse`）把概念封装成下游可用的产物。可作为"章节末 spec 收官"的独立模板
- **【教学场景】q2 三 spec 构成"三独立维度评估资产"的完整闭环**：spec-01 评信号质量（个股 vs ETF 噪音）、spec-02 评宏观锚点（GDP 锁地区）、spec-03 评分散效果（相关性锁组合）——三个独立维度共同覆盖"该选哪些标的"问题。这是 q2 章节级的【结构】方法点：**章节内 spec 的命题应当是"维度互补"而非"步骤递进"**——q1 是"步骤递进型"（数据 → 回测 → 基准 → 时机 → 调参 → 过拟合），q2 是"维度互补型"。两种 spec 链结构都是教学有效的，建议在方法章节单独列出对照
- **【教学场景】"标的池"概念的具象化**：StaticUniverse 在本 spec 是首次出现（虽然 spec-01 import 过但实际只用 `StaticUniverse((symbol,))` 包单一资产，没用其"标的池"语义）。spec-03 用三资产的 `name="global-macro-etf"` 让标的池作为可命名、可快照的研究对象——这是 q3-q9 反复使用的核心概念第一次落地。spec 应该用更显眼的 narrative 强调这是"概念落地点"，而非夹在第 10 条
- **【接续型 spec 链】上下游变量在上下文段重述**：spec-03 上下文段明确列出了 spec-02 已有的产物（510300 + 513100 数据）和已有的认知（学员已通过 GDP 锁定中美）——这是 q2 三份 spec 中接续最规范的写法，可作为"接续型 spec 链的上下文段写作模板"
