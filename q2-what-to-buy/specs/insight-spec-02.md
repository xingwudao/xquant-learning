## 一、设计意图复盘

这是 q2 三资产筛选的**第二步**——从"个股 vs ETF"过渡到"全球 ETF 几百只怎么选"。spec 用 GDP 数据作为锚点，把"经济最强的国家资本市场最有潜力"的直觉**反过来用数据验证**。这是 q2 章节的一个核心方法论："先猜后验、数据说了算"，spec-02 是这个方法论的第一个真正示范（spec-01 是论证组合、不是论证选择）。【教学场景：先猜后验】【动机】

**为什么是前 10 大经济体而不是 5 大或 20 大**：10 国是教学最优数 — ① 包括中美 + 8 个对照（足以让"中美遥遥领先"的视觉效果突出）；② 一张柱状图能装下 10 行不至于拥挤；③ 国家代码列表 `["USA", "CHN", "DEU", ...]` 长度可控。spec 没解释这层选择——【动机】缺位。

**为什么 GDP 取 2020-2024 而不是更长**：5 年窗口对世行 GDP 数据是合理的——只取最新一年用，但向前留 4 年缓冲是"预防最新年数据延迟发布"。这是数据领域的实战经验，但 spec 没写理由。

**为什么纳指 100 而不是标普 500**：spec 选 `513100.SS` 而非 `513500.SS`（标普 500ETF）— 选纳指 100 是因为后续 q3-q9 的策略示范需要"高波动 vs 低波动"的对照（纳指 100 显著比标普 500 波动大），且纳指 100 在 A 股市场流动性最大。但**这是埋伏笔的关键设计**，spec 完全没解释。【动机】缺位是这份 spec 的最大短板。

**spec 的克制点**：没有把 `WorldBankDownloader` 和 `read_factor` 的接口签名硬写进 spec，而是**用代码片段示范**——让学员看到具体调用方式，AI 只需要"按片段调"。这是比 spec-01 进步的地方（spec-01 把 `Strategy/Engine.run` 的范式硬写在自然语言里，反而更难理解）。但同样**没指示 AI 先读源码**——spec-writing-guide 原则 4 仍然没遵守。

**与上一份 spec 的接续**：spec-02 假设学员"已用 ETF 替代个股"，直接用 `YFinanceDownloader` + `LocalMarketDataProvider` 下载 510300/513100 — 但**没说要复用 spec-01 创建的实例**。学员复制到 TRAE 时会发现 `downloader` / `provider` 已存在（来自 spec-01 cell），还是要再次创建？spec 没明示。这是【接续型 spec 链】的常见漏洞。

## 四、可改进点（带改写示例）

### 改进 1：锁死 end_date，与 GDP 数据时间一致【可复现】

**现状**（要求 第 6 条）：
```
- 起始日期 `2021-01-01`，结束日期为当天
```

**改写**：
```
- 起始日期 `2021-01-01`，结束日期 `2026-01-01`（与 q1/spec-01 数据快照截止日一致）
- GDP 数据时间窗：start_year=2020, end_year=2024（与价格数据时间范围对齐）
```

让 GDP 与价格在时间维度上互相印证，避免学员 2030 年跑出来"GDP 是 2024 的、价格到 2030"的不对称数据。

### 改进 2：把"为什么这么选"全部写进 spec【动机】

**现状**：spec 只说"下载前 10 大经济体 GDP" / "下载 510300 和 513100"。

**改写**（要求 第 2 条加注 + 第 6 条加注）：
```
2. 使用 WorldBankDownloader 下载全球前 10 大经济体 GDP（2020-2024 共 5 年）：
   countries = ["USA", "CHN", "DEU", "JPN", "IND", "GBR", "FRA", "ITA", "BRA", "CAN"]
   · 选 10 个：足以让中美遥遥领先的视觉对比突出，单图能装下不拥挤
   · 选 5 年：取最新一年使用，向前留缓冲（防止当年数据延迟发布）

6. 下载两只对应的 ETF：
   - 510300.SS（沪深300ETF，代表中国核心资产 — A 股最大盘）
   - 513100.SS（纳指100ETF，A 股上市 — 美股科技龙头集中代表）
   · 选纳指 100 而非标普 500：① 后续 q3-q9 策略示范需要"高波动"对照标的；② A 股市场流动性更好
```

### 改进 3：补机械可验证的 assert【验证】

**现状**：只有定性 oracle。

**改写**（结果呈现 加新的第 0 条）：
```
0. 数据合法性检查：
   assert len(gdp_latest) == 10, "应有 10 国 GDP 数据"
   assert gdp_latest.loc["USA"] > gdp_latest.loc["DEU"], "美国 GDP 应大于德国"
   assert gdp_latest.loc["CHN"] > gdp_latest.loc["JPN"], "中国 GDP 应大于日本"
   assert csi300_norm.iloc[0] == 100, "归一化起点应为 100"
```

利用领域常识（中美 GDP 排序）做"语义级验证"，比单纯的 `len()` 检查更能防呆。

### 改进 4：与 spec-01 实例复用关系明示【接续型 spec 链】

**现状**：spec-02 直接用 `YFinanceDownloader().download(...)` + `LocalMarketDataProvider().get_bars(...)`，没说与 spec-01 创建的实例什么关系。

**改写**（在「## 上下文」中加一行）：
```
- 复用 spec-01 中已创建的 `downloader = YFinanceDownloader()` 和 `provider = LocalMarketDataProvider()` 实例（不要重复创建）
```

避免学员 / AI 在 notebook cell 间重复创建 oxq 实例（虽然不报错但是反模式）。

### 改进 5：要求开头加「先读 oxq 源码」指令【教学场景】

**现状**：直接给 `WorldBankDownloader.download` / `read_factor` 的调用片段。

**改写**（在「## 要求」开头插入第 0 条）：
```
0. 本 spec 用到 open-xquant 的世行数据模块。先阅读：
   - `oxq/data/worldbank.py`：`WorldBankDownloader.download` 接口签名
   - `oxq/data/factor.py`：`read_factor` 的返回值格式
   参考 https://github.com/xingwudao/open-xquant/tree/main/examples/tutorials
```

让 AI 看完源码再写代码，spec 的代码片段当作示范、不当作硬约束 —— 即使 oxq 升级，spec 不需要改。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 标签体系：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【结构】严格四段式**：与 q1/spec-01、q2/spec-01 一致——证明四段骨架在跨章节、跨复杂度场景都稳定
- **【精确】用代码片段而非自然语言描述接口**：`wb.download("gdp", countries=countries, ...)` 直接给可执行片段——比把"调用 download 函数，传入 'gdp' 字符串、countries 参数..."写成自然语言精确十倍。可作为"接口调用必须给代码片段"的方法示范
- **【教学场景：先猜后验】**：spec 在结果呈现的打印里**预先写好"最直觉的思路：经济最强的国家..."**和"数据说了算：锁定中美" — 把"假设 → 验证 → 结论"的方法论在 spec 内部就示范一遍，这是 q2 章节的核心教学方法的可复用模板
- **【教学场景】跨数据源混合 spec**：本 spec 同时用 GDP 数据（世行宏观）和股价数据（yfinance 行情）—— 两类数据有不同的更新频率、不同的时间粒度。spec 用统一的"先 download → 再 read"两步式访问把它们包在同一接口风格下。这是数据科学领域 spec 的常见模式，可作为"多源数据 spec 统一接口"示范
- **【正向】全部"做 X"**：无否定堆叠

### B. 这份 spec 修补后可示范的原则

- **【可复现】GDP / 价格时间窗对齐**：当前 GDP 写死 2020-2024、价格用动态 today——这种"半锁半动"的不一致是隐蔽的反模式。修复后可作为"同一份 spec 内所有数据时间维度必须对齐"的示范
- **【动机】"为什么这么选"必须写进 spec**：当前"前 10 大 / 纳指100 而非标普500 / GDP 取 5 年"三处选择全无解释。修复后可作为"参数选择必须解释、不只是声明值"的示范，与 q1/spec-01 同类
- **【接续型 spec 链】实例复用关系明示**：当前 spec-02 默认 spec-01 已创建 `downloader/provider` 但没明说。修复后可作为"接续型 spec 链上下游变量在上下文段明示"的示范
- **【教学场景】oxq 引入期"先读后写"**：与 spec-01 同——当前没指示 AI 先读 oxq 源码。修复后可作为该原则的示范

### C. 领域 / 教学场景特殊考量

- **【教学场景】"先猜后验"型 spec 模板**：q2 全章核心方法论是"先猜后验"。本 spec 在结果呈现里**预先把猜想（"经济最强 → 资本市场最有潜力"）和结论（"数据说了算：锁定中美"）都写进打印语句**。这种"在 spec 内部完成假设 → 验证 → 结论闭环"的写法，是教学型 spec 区别于工程型 spec 的关键招式。建议在方法章节单独列出"假设 - 验证 - 结论"三段打印的固定结构
- **【教学场景】数据驱动选材的"锚点"型 spec**：spec-02 不是为了"画个 GDP 图"，而是为了"用一个外部锚点（GDP）证明后续筛选的合理性"。这种"锚点型 spec"在量化教学中频繁出现（按 PE 锚定、按市值锚定、按因子分位锚定）。其特征：① 用一个外部数据源做依据；② 依据 → 选择的逻辑链显式；③ 选择结果直接喂给下游 spec。值得作为方法章节的独立模板
- **【教学场景】纳指100 / 标普500 选择的领域知识必须显式**：选 513100 而非 513500 是基于教学需要（高波动对照），但这是只有"看过 q3-q9"的研发者才能做的伏笔。spec 必须把这层"未来章节铺垫"显式写出来——否则学员仿写时无法理解为什么不能换成更广基的标普 500
