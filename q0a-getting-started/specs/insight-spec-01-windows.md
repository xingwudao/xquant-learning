## 一、设计意图复盘

这是 macOS 版的孪生 spec——同一任务（装好量化课程编程环境）的另一个平台版本。Windows 在 0 编程基础学员的设备分布里占比往往超过 mac，这份 spec 的成败直接决定课程的覆盖率。任务结构与 mac 版严格对称：第零步看目录 → 第一步装 uv → 第二步建 venv → 第三步装依赖 → 第四步跑 check_env.py。【教学场景】【结构】

**为什么用 PowerShell 而非 cmd**：`irm | iex` 是 PowerShell 原生管道，比 cmd 走 curl + bat 干净。spec 选 PowerShell 是技术正解，但**没有显式告诉学员"打开 PowerShell 而非默认 cmd"**——这是 Windows 学员的第一道隐形门槛（点开"终端"在 Win10 默认是 cmd，Win11 默认是 PowerShell）。【动机】缺失会直接卡死部分学员。

**为什么 uv 装完后让学员"关闭并重新打开终端"**：Windows PATH 在当前会话不生效，必须重开窗口。这是 Windows 与 mac 的硬差异——mac 用 `source $HOME/.local/bin/env` 立即激活，Windows 没有等价命令。spec 用了"重开终端"作为最朴素的解。可行，但学员可能因为"不知道为什么"而怀疑安装失败。【动机】

**为什么 check_env.py 与 mac 版字符级一致（除 venv 激活提示）**：环境配置 oracle 应当**跨平台同一份脚本**——只有"未激活时给出的激活命令"必须按平台变化（`source .venv/bin/activate` vs `.venv\Scripts\activate`）。这种"单点差异，其余对称"的写法是跨平台脚本的最佳实践。【同步】

**克制的边界**：没有处理 Windows Defender 拦截、长路径限制、Git Bash vs PowerShell 选择等冷门问题。spec 把自己锁在"PowerShell 可用 + 目录已开"之后，边界干净——但 Windows 的"边缘 case"密度比 mac 高，这份 spec 在错误恢复路径上预留得偏少。【简洁】vs【教学场景】的张力。

## 四、可改进点（带改写示例）

### 改进 1：套上 q1 四段式骨架【结构】【可学习】【同步】

**现状**：与 mac 版同——"第 N 步"自定结构。

**改写**：与 mac 版**完全相同的外层四段框架**，让两份 spec 在结构层字符级对称：

```
**上下文**
学员刚在 TRAE 中打开了一个空目录作为课程学习根目录。本机尚未安装 Python / uv / 任何依赖。操作系统为 Windows 10/11。这是课程的第一份 spec。

**任务**
在当前目录下完成 Windows 量化课程编程环境配置：装 uv、建 Python 3.12 虚拟环境、装 7 个依赖、跑通 check_env.py 自检。

**要求**
（原"第零~四步"内容）

**结果呈现**
1. PowerShell 运行 `python check_env.py` 输出 `结果: 9/9 全部通过!`
2. 退出码为 0
3. 当前目录下存在 `.venv\` 与 `check_env.py`
```

外层结构与 mac 版完全对称，差异只在【上下文】"操作系统为 Windows"+ 路径分隔符 + venv 激活路径。

### 改进 2：补"打开 PowerShell"的零步说明【动机】【教学场景】

**现状**：直接说"在 PowerShell 中运行"，假设学员已在 PowerShell。

**改写**（在第一步之前补一段）：
```
本 spec 所有命令在 PowerShell 中运行（不是 cmd）。
打开方法：开始菜单搜索 "PowerShell"，点击"Windows PowerShell"或"PowerShell 7"。
如果你不确定打开的是哪个：在终端运行 `$PSVersionTable.PSVersion`，能输出版本号说明在 PowerShell 中。
```

补这一步的成本极低，但能拦掉一大波 Windows 学员第一道隐形门槛。

### 改进 3：补中文用户名路径风险提示【教学场景】【验证】

**现状**：完全未提。

**改写**（注意事项加一条）：
```
- 如果你的用户名是中文（如 `C:\Users\张三\`），uv 或 pip 可能在路径处理上报编码错误。
  解决方法：在非家目录的英文路径下创建课程目录（如 `D:\quant-learning\`），把 TRAE 重新指向那里。
```

这是 Windows 学员的高频炸点。0 基础学员遇到此类报错完全无法 debug，会直接放弃课程。

### 改进 4：第零步命令数与 mac 版对称【同步】

**现状**：mac 用 `pwd`（1 条命令），Windows 用 `cd 或 echo %cd%`（2 条命令）。

**改写**：Windows 版统一为 `cd`（PowerShell 中 `cd` 不带参数与 `pwd` 行为一致）：
```
请先在 PowerShell 运行 cd（PowerShell 中不带参数等价于 pwd）获取当前目录路径
```

两份 spec 第零步均为 1 条命令，对称破缺消除。

### 改进 5：注意事项第 1 条措辞与 mac 版对称【同步】

**现状**：
- mac："如果终端提示 uv: command not found，请先运行 source $HOME/.local/bin/env"
- Windows："如果提示 uv 不是可识别的命令，请关闭终端重新打开"

两条想说同一件事："uv 装完了但 PATH 没生效"——但描述差异让两份 spec 看起来在解决不同问题。

**改写**（统一句式）：
```
- 如果终端提示 `uv 不是可识别的命令` (Windows) 或 `uv: command not found` (mac)：
  说明 uv 装完了但 PATH 还没生效。
  Windows: 关闭并重新打开 PowerShell
  mac: 运行 source $HOME/.local/bin/env
```

让两份 spec 的注意事项段在结构上字符级一致。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 标签体系（10 个）：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【验证】check_env.py 跨平台共享一份脚本**：mac/Windows 两份 spec 的 check 脚本字符级一致（仅 venv 激活提示路径分隔符不同）。可作为"跨平台 spec 的 oracle 应单点差异化"的原型示范——把可对称的部分对称到底，让差异显式收敛
- **【正向】全部"运行 X / 创建 X"**：无 "不要" 句式，与 mac 版对称
- **【教学场景】重开终端的运维提醒**：每次新终端先激活虚拟环境是 0 基础学员高频遗忘点。spec 把它写进注意事项 = 减少课程后续章节的重复支持负担。可作为"持续性运维提醒"在环境类 spec 的标配模板
- **【教学场景】沙箱外执行双层提示**：首行 markdown 引用 + 第二段自然语言提醒，与 mac 版对称。可作为"AI IDE 沙箱处理"模板的双平台一致示范

### B. 修补后可示范的原则

- **【结构】环境类 spec 也要套四段式**：当前用"第 N 步"自定结构与 mac 版同病；修复后可作为"跨平台 spec 外层结构必须字符级对称"的原型 before/after 教学
- **【动机】平台特有差异必须解释**：当前"PowerShell 而非 cmd"、"Windows 重开终端"零解释；修复后可作为"凡是平台差异点都要在 spec 里答'为什么'，否则学员怀疑装错了"的示范
- **【同步】对称性微差异是反例**：当前第零步命令数、注意事项措辞两处不对称；修复后可作为"跨平台 spec 的对称性精确到句式级"的示范
- **【教学场景】Windows 中文用户名路径风险**：当前未提；补上后可作为"平台特有炸点必须主动列入 spec 错误恢复段"的示范

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】跨平台 spec 的对称性硬约束**：mac/Windows 双 spec 是本课程硬约束。两份 spec 应满足：① 外层结构字符级对称（4 段同名）；② 步骤命名 / 数量对称；③ 检查项 / 退出条件对称；④ 仅在"具体命令字面"层差异化。本份 spec 在 ①②④ 上达标，③ 上局部破缺（第零步、注意事项第 1 条）——可作为"对称性 checklist"模板。方法章节必须有这一节
- **【教学场景】Windows 平台天然的复杂度溢出**：与 mac 比，Windows 多出"PowerShell 选择 / 中文路径 / 重开终端 / 长路径限制"等炸点。spec 必须为高频炸点预留错误恢复路径——这是 Windows 版相对 mac 版要"多写一些"的部分，对称性不该是字数完全相同，而是"覆盖率"对称
- **【教学场景】"门槛 spec"位置敏感性**：与 mac 版同——本份 spec 静默失败 = 后续 35 份课程崩塌。Windows 版因平台炸点更多，门槛风险高于 mac 版。方法章节应承认"位置 + 平台"双因素决定 spec 重要性
- **【教学场景】uv / Python 3.12 是项目硬约定**：与 mac 版同。版本号、工具名、命令字面级精确——这是【精确】原则在环境类 spec 的最强落点。Windows 版不能因为"PowerShell 命令历史习惯"偏离硬约定，比如不能"偶尔用 pip 直接装"留口子
