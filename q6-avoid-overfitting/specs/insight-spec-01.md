## 一、设计意图复盘

这是 q6 章首份 spec，承担**从 q1 的 60/40 split 升级到系统方法论**的衔接职责。q1/spec-07 用一行"前 60% / 后 40%"完成了过拟合现象的演示，这里把同一现象**显式化为"在好市场上挑参数 → 拿到差市场验证"**——把时段语义（2017-2020 好市场 vs 2021-2023 差市场）写进 spec，比抽象的"前/后切分"教学张力强一个量级。【动机】【教学场景】

**为什么是 490 个组合**：5×5 周期 × 4 频率 × 6 止损阈值，再加约束（频率 ≤ 信号窗口）。约束写法本身是教学点——`add_constraint("RebalanceFrequencyRule.interval_days <= mom.period")` 用可读字符串表达式而非 lambda，符合本章"spec 要让 0 基础学员能看懂"的语境。【精确】**为什么分无止损/含止损两组**：止损引入第四个维度，把 70 组合扩展到 420，这恰好是 Step 4「规则负担」的伏笔——本份 spec 在 Top 20 表里就让止损过拟合现象浮出水面，给后续 spec-04 蓄势。【结构：跨 spec 蓄势】

**克制的部分**：spec 没让 AI 自己决定"如何切分样本内/样本外"——直接锁死 `IS_START=2017-01-01 / IS_END=2020-12-31 / OOS_START=2021-01-01 / OOS_END=2023-01-31`。这是【可复现】维度的关键决定：如果留给 AI 选，今年跑、明年跑可能切出不同的窗口，整章的过拟合证据就站不住。【可复现】

**张力的部分**：要求 8 提到"`GridSearch.run()` 不支持直接传入 rules，需要编写 `grid_search_with_rules()` 包装函数"，并指定从 `oxq.optimize.search` 导入私有函数 `_apply_params` / `_apply_rule_params`。这里 spec 暴露了 oxq 框架的能力边界——本应是 oxq 修补的设计缺陷被推到了 spec 层面，违反了 spec 写作指南原则 4「先读后写」中"框架升级 spec 不需要修改"的承诺。【教学场景：框架张力】

## 四、可改进点（带改写示例）

### 改进 1：把"参数判断段"写进 spec【同步】【精确】

**现状**：spec 没提"参数判断"，但 notebook 有 60+ 行专门做这个。

**改写**（在要求 11 后新增第 12 项）：
```
12. 参数判断（按 Peterson 稳健性原则）：
    - 按动量窗口、波动率窗口、调仓频率分组，打印每个值的样本外夏普均值和标准差
    - 标注「✔ 可取」（均值 > 0 且标准差 < 全列标准差最大值 × 1.5）
    - 对止损分组同样处理，与无止损组的样本外夏普均值对比
    - 打印综合判断：可取的 mom/vol/freq 区域、止损是否有效
```

补这一段后，notebook 里那 60 行就是"显式契约"，下次重跑可保字符级一致。

### 改进 2：figsize 与画图结构对齐 notebook【同步】

**现状**（要求 10）：散点图 10×8 + 柱状图 12×6，两张图。

**改写**：
```
10. 画 1 张图（figsize=(16, 6)，1 行 2 列子图）：
    - 左：样本内夏普 vs 样本外夏普散点图，画 y=x 对角线（虚线）+ Top 20 红色大点
    - 右：Top 20 的样本内/样本外夏普对比柱状图（每组双柱）
```

### 改进 3：把 `_apply_params` 等私有函数推回 oxq【教学场景：框架张力】【正向】

**现状**（要求 8）：spec 让 AI 从 `oxq.optimize.search` 导入 `_apply_params` / `_apply_rule_params` 私有函数写包装函数。

**改写**（先在 oxq 升级 `GridSearch` 接受 `rules` 参数，本 spec 改为）：
```
8. 用 GridSearch 跑两组参数空间：
   - GridSearch(paramset).run(strategy, rules=[...], market=..., broker_factory=..., 
     start=IS_START, end=IS_END, metric="sharpe_ratio")
   - 分别在样本内、样本外跑无止损组和含止损组
```

把"框架内部细节"留给 oxq 而非 spec——符合"先读后写"原则中"框架版本会迭代，硬写的接口会过时"的规避。

### 改进 4：拆分要求段，引入子标题【简洁】【结构】

**现状**：11 项要求线性堆叠，第 7 项内部就 7 个子项。

**改写**（要求段加三层小节）：
```
## 要求

### 步骤 A：基础准备（导入、数据、策略）
1-5. ...

### 步骤 B：参数空间与网格搜索
6-8. ...

### 步骤 C：分析、画图、判断
9-12. ...
```

让学员用 spec 时一眼看到"三步走"骨架，而不是 11 项扁平列表。

### 改进 5：约束语法的"为什么"写进 spec【动机】【教学场景】

**现状**（要求 7）：`add_constraint("RebalanceFrequencyRule.interval_days <= mom.period")` 只列约束，未解释。

**改写**：
```
- 约束「调仓频率 ≤ 信号窗口」的动机：
  · 信号 20 天更新一次，调仓 60 天才看一次 = 在用过期信号做决策
  · 这是领域知识，不是技术约束——spec 要写明
- add_constraint("RebalanceFrequencyRule.interval_days <= mom.period")
- add_constraint("RebalanceFrequencyRule.interval_days <= vol.period")
```

让学员单读 spec 时就能学到"领域约束怎么编码"。

## 五、可沉淀的方法点（用于书中方法/模板章节）

### A. 这份 spec 已经示范的原则

- **【可复现】时段切分要锁死起止四个日期**：`IS_START / IS_END / OOS_START / OOS_END` 四个常量全部写明，是【可复现】维度的标杆。可作为"任何'最近 N 年/最早 N 年'切分都是反模式，必须锁死起止"的高分例证（q1/insight-spec-01 改进 1 的同款原则，本 spec 是已经做对的）
- **【动机】时段语义写进 spec**：把"2017-2020 好市场（用来优化）/ 2021-2023 差市场（用来验证）"写到要求里——不是"前 60% / 后 40%"那种结构性切分，而是有金融含义的切分。可作为"切分 spec 必须解释每段的市场含义"原则的示范
- **【精确】约束表达式的可读字符串语法**：`"component.param op component.param"` 比 lambda 更适合 0 基础学员。可作为"DSL 字符串 > 函数对象"原则的示范
- **【结构：跨 spec 蓄势】止损组埋伏笔**：在 spec-01 就把止损纳入参数空间，让"止损组样本外比无止损组差"在本 spec 浮现，给 spec-04「规则负担」铺路。可作为"早期 spec 应给后续 spec 埋叙事钩子"的示范
- **【验证】机械可验的组合数**：验证段写"约束后总组合数约 490"，是数字断言而非"看起来对"，AI 跑出来不对立刻知道

### B. 这份 spec 修补后可示范的原则

- **【同步】参数判断段必须显式化**：当前 spec 漏写"按参数分组打印样本外均值"这一段，notebook 里 60+ 行 AI 自行补全。修复后可作为"任何 notebook 里超过 30 行的代码段都必须在 spec 里有显式契约"的 before/after
- **【教学场景：框架张力】私有 API 不应进 spec**：当前 spec 让 AI 导入 `_apply_params` 等下划线开头函数——属于把框架的设计缺陷推给 spec。修复路径是先升级 oxq，spec 只用公开 API。可作为"spec 露出私有 API = oxq 该升级"原则的反面示范
- **【简洁】要求段超过 8 项就该加子标题**：当前 11 项扁平列表 + 三层嵌套，超出 q1/spec-01 的简洁基线。修复后可作为"三层嵌套阈值"的范例
- **【同步】figsize 必须与 notebook 字符级一致**：当前 spec 写 10×8 / 12×6 两张图，notebook 实际 16×6 双面板。修复后是"图表结构契约"原则的示范

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景】"好市场/差市场"切分是金融场景独有的【动机】写法**：通用 ML 领域的"训练/验证/测试"切分通常是随机或时序前后；量化场景必须写明每段对应的市场行情语义（牛/熊/震荡），否则学员看不出过拟合的"对照实验"是怎么搭起来的。本 spec 是这种写法的原型示范
- **【教学场景】GridSearch + 自定义包装函数**：q6 是本课程第一次让学员看到"oxq 公开 API 不够用，需要写包装函数"。这种张力**应当被显式承认**——可在书中方法章节加一节"什么时候 spec 该绕过框架"，本 spec 的 `grid_search_with_rules` 是反例素材
- **【教学场景】Peterson 稳健参数原则的引入位置**：本 spec 是 Peterson 第一次出现的位置——后续 spec-02/03/04 都引用他。可在书中方法章节专门写一节"领域权威的引用契约"——什么时候在 spec 里点名引用、引用谁、引用哪条原则
