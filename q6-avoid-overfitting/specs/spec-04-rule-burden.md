# Spec: 规则越多回测越好——是好事吗？——规则负担

> 所有命令在沙箱外运行。

## 上下文

在 `q6-avoid-overfitting.ipynb` 中已有：
- 参数优化 + 样本内/样本外
- Walk-forward（Rolling + Anchored）
- 交叉验证（5 折 Time-Series Split）

当前问题：前三步都在检验"参数选得对不对"。但还有一种过拟合来自"规则太多"——每加一条规则，就多了一个可以调节的旋钮。旋钮越多，越容易在样本内"调出"好看的结果。

## 任务

在 notebook 中新建代码单元格，通过两个堆叠实验展示规则负担。

## 要求

1. 阅读以下 oxq 模块的源码，了解可用的交易规则及其参数：
   - `oxq.rules.StopLossRule` — 止损规则（`threshold` 参数）
   - `oxq.rules.TakeProfitRule` — 止盈规则（`threshold` 参数）
   - `oxq.rules.TrailingStopRule` — 移动止损规则（`trail_pct` 参数）
   - `oxq.rules.MaxDrawdownRule` — 最大回撤风控（`max_drawdown` 参数）
   - `oxq.core.Engine` — 回测引擎（`Engine().run(strategy, market=..., broker=..., start=..., end=...)` 注意：构造函数无参数）
   - 确认 `ParameterSet.add_constraint()` 的约束表达式格式

2. **4A 规则堆叠实验**：
   - 基础策略：TopNRanking，固定参数 mom=20, vol=20, freq=10
   - 逐层添加规则，每层在样本内优化该层的参数：
     - Layer 0：基础策略（固定参数，无额外规则）
     - Layer 1：+ 优化调仓频率（freq = FREQS，约束 freq <= mom.period）
     - Layer 2：+ 优化止损（sl = SL_THRESHOLDS）
     - Layer 3：+ 优化止盈（tp = TP_THRESHOLDS，约束 sl < tp）
     - Layer 4：+ 优化移动止损（trail = TRAIL_PCTS，约束 trail >= sl）
     - Layer 5：+ 最大回撤风控（max_dd = MAX_DD_VALS，约束 max_dd >= sl）
   - 每层在样本内选最优 → 在样本外验证
   - 约束过滤不合理组合（止损 > 止盈、移动止损 < 固定止损等）
   - Layer 0 没有参数可优化，直接用 `Engine().run()` 跑回测
   - 打印每层的样本内/样本外夏普、收益、衰减

3. **4B 指标堆叠实验**：
   - 从 1 个参数开始，逐步增加可优化的参数维度：
     - A：只优化动量窗口（1 个自由度，PERIODS）
     - B：+ 波动率窗口（2 个自由度）
     - C：+ 调仓频率（3 个自由度，约束 freq <= mom, freq <= vol）
     - D：+ 止损阈值（4 个自由度，约束同 C）
     - E：+ 止盈 + 移动止损（6 个自由度，粗网格 [10,20,30]，约束 freq<=mom, freq<=vol, sl<tp, trail>=sl）
   - 每层在样本内优化所有参数 → 样本外验证
   - 约束过滤不合理组合，与 4A 一致
   - 打印每层的自由度、组合数、样本内/样本外夏普、衰减

4. 打印两个汇总表：
   - 4A：层名、样本内夏普、样本外夏普、衰减、样本内收益、样本外收益、自由度
   - 4B：配置名、自由度、样本内夏普、样本外夏普、衰减、样本内收益、样本外收益

5. 画两张图：
   - 图 1（figsize 12×6）：4A 规则堆叠——样本内夏普和样本外夏普随层数的变化（双折线）
     - x 轴：Layer 0-5
     - 两条线：样本内（蓝色上升）vs 样本外（红色先升后平/降）
     - 标题「规则堆叠：样本内 vs 样本外」
   - 图 2（figsize 12×6）：4B 指标堆叠——同样画样本内/样本外双折线
     - x 轴：自由度 1-6
     - 标题「自由度越多 ≠ 越好」

6. 打印分析（根据实际数据动态描述）：
   - 4A 结果：样本内夏普随规则增加的变化趋势，以及样本外的对应变化
   - 规则负担的含义：每条新规则增加一个可调旋钮，优化器多了一个维度可以"调出"好看的结果
   - 4B 结果：自由度增加后，样本内提升多少，样本外提升多少——多出来的"提升"是自欺欺人
   - 引用 Brian Peterson 的观点：Rule burden is a form of overfitting
   - 总结：能用 2 个参数解决的问题，不要用 6 个。简单就是美。
