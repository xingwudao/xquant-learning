## 一、迁移结论

本 spec 已从早期动态结束日写法迁移为固定窗口：

- `DATA_START = "2017-01-01"`
- `ANALYSIS_END = "2026-05-06"`
- `n_splits = 5`
- `expanding = True`

迁移后，5 折切分点固定，任意时间重跑都不会因为数据结束日变化而改变折边界。

当前 SDK 的交叉验证高级执行接口不接收本章实验所需的 `rules` 参数。Notebook 因此使用 `TimeSeriesCV.split()` 生成切分，再用本章 helper 在每折训练集上做网格搜索，并组装为 `CVResult`。

这保持了时间序列交叉验证的实验含义：训练集始终在验证集之前，训练集逐折扩大，绝不让未来数据进入过去。

## 三、最佳实践

本 spec 现在示范了承接型实验的三个关键要求。

第一，切分型实验必须锁死数据端点。交叉验证看起来只配置折数，但折边界完全依赖数据范围。

第二，承接型 spec 应做减法。参数空间、策略对象和基础 helper 已在前两步定义，本 spec 只负责生成 5 折切分、运行每折验证、分析一致性。

第三，金融交叉验证必须保持时间顺序。不能使用随机 K-Fold；每折训练集必须早于验证集。

---

## 四、可沉淀的方法点

- 交叉验证的输出不能只看均值，必须同时给标准差、范围和负折数。
- 参数一致性要进入 spec 主体：读取每折 `best_params`，统计最常出现值和唯一组合数。
- `expanding=True` 应显式承接 Anchored 思路：既然上一实验显示累积历史数据更稳定，本实验继续采用扩展训练集。
- 当 SDK 高级执行接口不满足实验参数时，可以使用 split API + 本章 helper，同时保留标准结果对象。
