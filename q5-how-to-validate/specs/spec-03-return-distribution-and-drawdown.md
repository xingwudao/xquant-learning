# Spec: 收益结构什么样？——分布与回撤

> 所有命令在沙箱外运行。

## 上下文

在 `q5-how-to-validate.ipynb` 中已有：
- 四个策略 vs 基准的整体指标对比
- 逐年收益拆解，发现某些策略有负收益年份
- TopNRanking 在市场反复波动时的结构性弱点

当前问题：同样赚钱的两个策略，赚法可能完全不同。需要看收益分布和回撤深度。

## 任务

在 notebook 中新建代码单元格，分析月度收益分布和历史回撤事件。

## 要求

1. 阅读以下 oxq 接口的源码：
   - `result.monthly_returns()` — 返回月收益率 Series
   - `result.drawdown_series()` — 返回回撤序列（每天距离前高跌了多少，负值）

2. 月度收益分布分析（四个策略）：
   - 统计：总月数、盈利月数、亏损月数、胜率（盈利月 / 总月）
   - 平均盈利月收益、平均亏损月收益、盈亏比（平均盈利 / 平均亏损绝对值）
   - Profit Factor（总盈利 / 总亏损的绝对值）
   - 最好月、最差月、中位数月收益
   - 最长连续盈利月数、最长连续亏损月数

3. 打印月度统计对比表：
   - 行：四个策略
   - 列：胜率、平均盈利月、平均亏损月、盈亏比、盈亏因子、中位数月收益、最好月、最差月、最长连赢、最长连亏

4. 画月度收益分布直方图（figsize 14x8，2x2 子图）：
   - 每个策略一个子图
   - 用 20 个 bins
   - x=0 处画一条红色垂直虚线
   - 子图标题含策略名和盈亏因子

5. 回撤深度分析（四个策略）：
   - 用 `drawdown_series()` 获取回撤序列
   - 识别每次回撤事件（回撤从 0 跌下去，到恢复到 0）：
     - 开始日期、最低点日期、恢复日期（如果尚未恢复则标"未恢复"）
     - 深度（最低点的值）
     - 下跌天数（开始到最低点）
     - 恢复天数（最低点到恢复，未恢复则标"—"）
     - 总持续天数
   - 按深度排序，打印前 5 大回撤事件

6. 打印回撤汇总：最大回撤、平均回撤深度、平均恢复时间、最长恢复时间

7. 画水下曲线（figsize 14x10，4 个子图垂直排列，共享 x 轴）：
   - 每个策略一条水下曲线
   - 用 `fill_between` 从 0 往下填充为浅蓝色
   - 子图标题含策略名和最大回撤值
   - x 轴只在最底部子图显示

8. 拆解 TopNRanking 2021 年月度收益（用数据说明胜率陷阱）：
   - 提取 TopNRanking 2021 年的月度收益序列
   - 计算 2021 年的：胜率、平均盈利月收益、平均亏损月收益、盈亏比、累计收益
   - 打印 2021 年逐月明细
   - 打印四个策略 2021 年的月度对比表（胜率、平均盈利月、平均亏损月、盈亏比、年收益）
   - 对比说明：TopNRanking 2021 年的胜率未必低，但盈亏比远低于其他策略——赚的月份赚得少、亏的月份亏得狠，导致全年亏钱

9. 打印分析（根据实际数据动态描述方向）：
   - 胜率的陷阱：零基础学员天然追求高胜率，但胜率本身意义不大——真正决定收益的是盈亏比
   - 用 2021 年四个策略的月度对比数据说明：同样的胜率，完全不同的结局——盈亏比才决定最终收益
   - 盈亏因子（Profit Factor）的含义：大于 1 说明赚的比亏的多，大于 2 算优秀
   - 最大回撤 vs 恢复时间对比
   - 「很多优秀的策略胜率并不高——它们的秘诀是'小亏大赚'：用止损控制每次亏损的幅度（小亏），赶上趋势时不封顶地赚（大赚）。追求高胜率，往往是捡了芝麻丢了西瓜。」
   - 「水下曲线告诉你一个残酷的事实：你有多少时间是在'还没回本'的状态中度过的。回撤不只是一个数字——它是一段煎熬。」
   - 「评估够全面了吗？其实还有一个隐藏问题：你用的参数（动量窗口 20 天、波动率窗口 20 天）是不是恰好选对了？」

## 结果呈现

1. 月度统计对比表（4 行 × 8 列）
2. 月度收益分布直方图（2×2）
3. TopNRanking 2021 年逐月明细 + 四策略 2021 年月度对比表
4. 前 5 大回撤事件表（每个策略）
5. 回撤汇总表
6. 水下曲线图（4 个子图）
7. 分析文字
