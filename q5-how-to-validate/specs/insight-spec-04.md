## 一、设计意图复盘

spec-04 是 q5 章的收尾——回答最后一个体检维度:**好策略不怕参数变**。这是 q5 整章最具方法论意义的认知升级:不只看单点数字,还看"参数附近的形状"——高原(plateau)还是山峰(peak)。【动机】

**与 q1/spec-06 参数扫描的关键区别**:q1 是"找最优参数"——学员希望挑出最佳值得回测投入。q5/spec-04 完全相反——**已经选定参数 20/10/5%,现在测试这些选择稳不稳**。同样是参数扫描的技术,但教学目的截然不同。spec-04 在上下文段写了「所有分析都基于特定参数」,但**没有显式说明"这次扫描不是为了找最优"**——这是【动机】维度的隐藏弱点。【动机:与 q1 重复方法的差异化未显式】

**为什么扫描这三个维度(vol_period / mom_period / stop_loss)**:这三个是 spec-01-03 用过的参数。spec-04 不引入新参数,只是把已经"用过"的参数全部抓回来体检。这种"不做加法"的克制,与 spec-01 的"四策略集合体检"相呼应——q5 章一以贯之地不引入新对象,只检验老对象。【教学场景:渐进难度】

**高原型 vs 山峰型的隐喻设计**:要求 6 写「想象你在山上找营地」——这是 q5 整章最妙的隐喻。零基础学员一秒理解"plateau >> peak":高原稳,山峰脆。spec 没把这句藏在 book 注释里,而是写进【要求】让 AI 输出——【动机】维度满分。【动机】

**统一 y 轴的视觉决策**:要求 5 三个子图共享 y 轴范围——这是参数敏感性可视化的关键细节。如果 y 轴各自缩放,RP 和 TNR 看起来都"波动很大"(因为各自填满图);统一后,RP 是平的、TNR 是尖的,视觉冲击力立现。这是【精确】维度的高分体现。【精确】

**克制的张力**:spec 没要求"在最优参数处用红圈标注"或"加置信区间"——保持图表简洁,把"高原 vs 山峰"作为唯一视觉主题。这种"一图一论点"的克制是好 spec 的隐藏品质。【简洁】

## 四、可改进点

### 改进 1:显式区分 q1 与 q5 的扫描教学目的【动机】

**现状**(上下文段):
```
当前问题:所有分析都基于特定参数(period=20, frequency=10, threshold=0.05)。
如果参数稍微变一下,结论还成立吗?
```

**改写**:
```
当前问题:所有分析都基于特定参数(period=20, frequency=10, threshold=0.05)。
如果参数稍微变一下,结论还成立吗?

注意:这里的参数扫描和 q1/spec-06 不同——
  · q1 是"用扫描找最优参数"(目的:挑值)
  · q5 是"用扫描判稳健性"(目的:判形状)
同样的技术,完全不同的目的。q5 不是要找出比 20 更好的窗口,
而是要判断"在 20 附近的所有值,结果是不是都差不多"。
```

**为什么**:学员第二次见到参数扫描会困惑"上次用过这个,现在为什么又用?"——必须显式区分。这是 handbook 第六节"与 q1/insight-spec-01 重复的方法点"的对偶情况(同方法不同教学目的)。

### 改进 2:锁 end_date 与库版本【可复现】

**现状**:沿用 spec-01 的隐患

**改写**(在要求 1 之前加):
```
0. 数据快照与版本(沿用 spec-01 的 START/END/库版本):
   - 时间窗口:2021-01-01 ~ 2026-03-18(本章数据快照)
   - open-xquant >= 0.4.0
```

**为什么**:与 spec-01 的 改进 1 配套——q5 章四份 spec 必须共享同一个时间锁。

### 改进 3:解释扫描点的选择【动机】

**现状**(要求 1):
```
扫描 vol_period = [10, 15, 20, 25, 30, 40]
```

**改写**:
```
扫描 vol_period = [10, 15, 20, 25, 30, 40]
  · 中心 20 是 spec-01 选定值
  · 左右对称展开 ±10 / ±15(覆盖 10/15/25/30)
  · 加 40 看远端外推是否崩坏(40 是 20 的 2 倍)
  · 不试 5 因 5 天太短无法稳定估计波动率
```

**为什么**:学员看到 [10,15,20,25,30,40] 会想"为什么不是 [10,20,30]?为什么有 40 没有 5?"——参数选择的【动机】必须写进 spec,否则学员仿写时无法自己挑值。

### 改进 4:统一表格行数(8 vs 7+1)【同步】

**现状**(要求 3):
```
- 打印对比表(8 行 × 5 列):止损阈值、累计收益、夏普比、最大回撤、交易次数
```

**实际 notebook**:7 阈值表 + 1 行"无止损"单独 print

**改写**:
```
3. 止损阈值敏感性:
   - 标题注明「基于 RiskParity 策略,调仓频率 10 天」
   - 扫描 thresholds = [0.02, 0.03, 0.05, 0.07, 0.10, 0.15, 0.20]
   - 表格 8 行 × 5 列(7 个阈值 + 1 行"无止损"作为最后一行,
     "无止损"行的"止损阈值"列填字符串「—」或「无」):
     止损阈值、累计收益、夏普比、最大回撤、交易次数
```

**为什么**:如果 spec 说 8 行表,notebook 应该是统一的 8 行表(其中无止损是最后一行),而不是分成"7 行表 + 1 个 print"。这是【同步】维度最容易漂移的地方。

### 改进 5:补 RP 极差 < TNR 极差的具体差距 assert【验证】

**现状**(验证段):
```
- RiskParity 极差明显小于 TopNRanking 极差
```

**改写**:
```
执行成功的标志:
- 三组表分别 6/6/8 行
- 折线图 3 个子图,y 轴范围一致(共用 y_min/y_max)
- assert rp_range < tnr_range, "RP 极差应小于 TNR 极差"
- assert tnr_range >= rp_range * 1.5, \
    f"TNR 极差应至少为 RP 极差的 1.5 倍才有教学说服力(实际 RP={rp_range:.2f}, TNR={tnr_range:.2f})"
```

**为什么**:如果某些数据期下 TNR 极差只比 RP 多 5%,学员看不出"高原 vs 山峰"的差异——教学失败。这条断言保证 q5 章的核心结论始终视觉可辨;若断言失败,说明数据期或参数列表需要调整。

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【动机】山顶营地隐喻**:把"高原型 vs 山峰型"翻译成"找营地"——可作为"金融抽象概念必须用日常隐喻锚定"的最佳示范
- **【精确】y 轴统一范围的视觉一致性**:三子图共享 ylim 让"RP 平、TNR 尖"立刻可见——可作为"对比图必统一坐标"的标准句
- **【动机】"无止损"作为基线参与对比**:要求 3 把"无止损"加入扫描参与同框——可作为"敏感性分析必含极端基线"的示范
- **【正向】"根据实际数据动态描述方向"**:第四次出现——q5 章四份 spec 完全一致的"防 LLM 幻觉"标准句,可作为强证据进入方法章节
- **【验证】结论维度的可机械检判据**:验证段"RP 极差 < TNR 极差"是行为型断言而非形态型——可作为"验证不止查形,还查论点是否成立"的示范

### B. 这份 spec 修补后可示范的原则

- **【动机】重复方法的差异化教学**:当前 spec-04 的扫描和 q1/spec-06 的扫描共用技术但不同目的——修复后可作为"同方法不同动机时,spec 必须显式区分"的示范
- **【动机】扫描点选择的解释**:当前 [10,15,20,25,30,40] 没说为什么——修复后可作为"参数列表的设计必须有可解释的对称性"的示范
- **【可复现】跨 spec 时间锁继承**:q5 章四份 spec 应共享同一 START/END——修复后可作为"接续型 spec 链的时间锁继承"原则
- **【验证】结论强度的下界 assert**:当前只查"小于"——修复后(加 1.5x 倍数)可作为"教学结论必须有显著性下界"的示范

### C. 领域/教学场景特殊考量

- **【教学场景】"高原型 vs 山峰型" 是策略稳健性的核心概念**:零基础课程很少教,但 q5 把它做出来了。这是 q5 整章最具方法论价值的"原创式教学",值得在书中方法章节作为"扫描技术的两层教学:挑值/判形"双面应用单列
- **【教学场景】参数扫描的两面性是 q1↔q5 跨章呼应的关键链路**:q1 教"扫描找最优"是过拟合诱因(book 中后续 q6 会揭示),q5 教"扫描判稳健"是过拟合的解药——同一个技术在书的前后扮演不同角色,这种"工具语义随章节变化"的设计本身值得在方法章节单列
- **【教学场景】夏普比极差作为"形状量化指标"**:零基础学员看不出图表"高原 vs 山峰",但 max-min 极差是一个数字——把视觉特征量化成一个数,是金融数据可视化的高阶教学动作;可作为"图表+数字双通道教学"的示范
