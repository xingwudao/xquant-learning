## 一、设计意图复盘

spec-03 是 q5 章里**最庞大的一份**——93 行、9 条 requirements、7 类输出物(月度统计表、月度直方图、TopNRanking 2021 拆解、四策略 2021 对比、前 5 大回撤事件表、回撤汇总表、水下曲线图)。这份 spec 的核心任务是教**两个隐藏的金融认知**:**胜率陷阱**和**回撤恢复时间**——都是零基础学员最容易踩的坑。【动机】【教学场景】

**为什么用月度而不是日度**:日度太碎(每天涨跌都算"赢/输",数据噪声大),年度太粗(去年赚钱掩盖了 8 个亏损月)——月度是教学甜蜜点。spec 没解释这一点,但选月度本身的合理性能看出设计者的克制。【动机:克制理由未写出】

**胜率陷阱的最佳教学路径——用数据反驳直觉**:要求 8 单独拆 TopNRanking 2021 年。这是 q5 整章最精彩的"用数据说话"环节:整体看 TNR 的胜率不低,逐年看 2021 亏钱;月度拆开发现 2021 的胜率仍不低,但盈亏比烂——一步步揭穿"高胜率=好策略"的迷思。【动机:三层揭谜的教学设计】

**回撤的双维度——深度+恢复时间**:大部分书只讲最大回撤数字,这份 spec 要求 5/6/7 把"事件"拆出来:开始日期、最低点日期、恢复日期、下跌天数、恢复天数——把"-15%"这个数字还原成"一段煎熬"。水下曲线图(要求 7)从视觉层面强化"你有多少时间在水下"。【教学场景:从数字到体验】

**最大风险——spec 自己变成 kitchen sink**:9 条 requirements 涵盖月度+回撤+特定年份拆解+水下曲线,涉及 7 类输出物——已经接近 handbook 反模式 #3「一份 spec 塞 5 个任务」。设计者意识到这是"q5 实验密度最高的一节",但拆成两份 spec(月度分布 / 回撤分析)可能更清晰。【简洁:9 分扣最多】

## 四、可改进点

### 改进 1:把 spec 拆成两份(月度分布 / 回撤分析)【简洁】【结构】

**现状**:9 条 requirements,7 类输出,93 行——单份 spec 容量过载

**改写**(拆分):
```
spec-03a-monthly-distribution.md:
  - 要求 1-4(月度统计、月度直方图)
  - 要求 8(TopNRanking 2021 深挖)
  - 输出 3 类:统计表、直方图、2021 拆解

spec-03b-drawdown-analysis.md:
  - 要求 1(drawdown_series)、5、6、7(回撤事件、汇总、水下曲线)
  - 输出 3 类:前 5 大回撤事件表、回撤汇总、水下曲线
```

**为什么**:接近 kitchen sink 是反模式 #3。学员一次复制 93 行给 AI,LLM 注意力分散,中间某条要求容易丢;拆成两份后,每份 < 50 行,执行成功率显著上升。

### 改进 2:把回撤事件阈值显式化【精确】【可复现】

**现状**:spec 没说;notebook 用 `if val < -0.001` 判断回撤开始,`val >= -0.001` 判断恢复

**改写**(要求 5):
```
5. 回撤深度分析(四个策略):
   - 用 drawdown_series() 获取回撤序列
   - 识别每次回撤事件(threshold = -0.001,即跌幅 ≥ 0.1% 才算事件,
     避免数值精度噪声生成大量假事件):
     - 起点:dd 从 0 跌穿 threshold 的那天
     - 恢复:从最低点反弹到 dd ≥ threshold 的那天
     - 未恢复:序列结束仍未回到 threshold 之上
```

**为什么**:这个阈值会显著改变事件数。-0.001 → 假设有 50 个事件;-0.01 → 可能只剩 10 个。学员看到 spec 复制给 AI,AI 没有 threshold 默认值,不同 AI 选不同值得出完全不同的"前 5 大回撤"。

### 改进 3:连胜连亏算法显式化【精确】

**现状**(要求 2):
```
- 最长连续盈利月数、最长连续亏损月数
```

**改写**:
```
2. 月度收益分布分析(四个策略):
   ...
   - 最长连续盈利月数:用月收益符号变化划分 streak,取最长正 streak 长度
   - 最长连续亏损月数:同上,取最长负 streak 长度;月收益 = 0 算亏损
```

**为什么**:边界情况(月收益恰好为 0 算输还是赢)是 streak 算法的隐藏分歧点。notebook 把 0 归入"亏"(`if ret > 0` 走赢分支,`else` 走亏分支),spec 必须明示——否则不同 AI 实现会差 1-2 个月长度。

### 改进 4:补机械 assert【验证】

**现状**:验证段说"胜率 0-100%、回撤 ≤ 0、4 个子图"

**改写**:
```
执行成功的标志:
- assert all(0 <= s["win_rate"] <= 1 for s in monthly_stats.values())
- assert all((dd_data[name]["series"] <= 0).all() for name in strat_names), \
    "回撤序列应始终 ≤ 0"
- assert all(len(dd_data[name]["events"]) >= 1 for name in strat_names), \
    "每个策略应至少有 1 次回撤事件"
- 月度统计表 4 行;直方图 4 子图各有红色 x=0 虚线
- TopNRanking 2021 年逐月明细行数与 2021 实际月数(应为 12 或更少)一致
- 水下曲线 4 子图,曲线始终 ≤ 0
```

**为什么**:`(dd <= 0).all()` 是回撤定义的核心断言——如果 oxq 的 drawdown 实现有 bug,会立刻发现。

### 改进 5:与 spec-01 颜色对齐【同步】

**现状**:水下曲线全部用 #4472C4(蓝色)——所有 4 个策略一个颜色

**改写**(要求 7):
```
7. 画水下曲线(figsize 14x10,4 个子图垂直排列,共享 x 轴):
   - 每个策略一条水下曲线,用与 spec-01 一致的颜色:
     EqualWeight=#4472C4, RiskParity=#ED7D31, TopNRanking=#70AD47, RP+止损5%=#FFC000
   - 用 fill_between 从 0 往下填充为该策略的浅色版(alpha=0.4)
   - ...
```

**为什么**:学员从 spec-01 净值曲线认识了"绿色 = TNR",到 spec-03 水下曲线全是蓝色,无法快速联结"哪条曲线是 TNR";颜色继承是跨 spec 视觉一致性的关键。

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【动机】"胜率陷阱"的三层揭谜**:整体胜率→年度亏损→月度盈亏比——一步步揭穿直觉,可作为"用数据反驳直觉"的高级模板
- **【教学场景】回撤的双维度教学**:不只讲"最深的坑",还讲"在水下多久"——可作为"金融指标必须从数字还原为持有体验"的示范
- **【动机】特定年份拆解**:用 TopNRanking 2021 这个具体案例做"胜率陷阱"的实证——可作为"案例驱动比抽象解释强 10 倍"的示范
- **【正向】「根据实际数据动态描述方向」标准句**:第三次出现,与 spec-01/02 一致——可作为"q5 章一致的防 LLM 幻觉规约"的高频证据

### B. 这份 spec 修补后可示范的原则

- **【简洁】kitchen sink 拆分**:当前 9 条 requirements 过载——修复后(拆 03a/03b)可作为"spec > 60 行就该拆"的 before/after
- **【精确】数值阈值必须写进 spec**:当前 -0.001 阈值是 AI 自由心证——修复后可作为"任何'看起来微小'的阈值都是 spec 必写项"
- **【精确】算法分歧点(连胜连亏中 0 算输还是赢)必须明示**:当前隐式——修复后可作为"边界情况必须写进 spec"
- **【验证】回撤序列的不变量断言(始终 ≤ 0)**:修复后可作为"金融指标的不变量断言"标准模板

### C. 领域/教学场景特殊考量

- **【教学场景】胜率陷阱是零基础学员的最大认知偏误**:零基础学员看到"60% 胜率"立刻以为是好策略,但实际胜率与盈利无关。这条原则在 spec-01 的 Alpha 概念之后,是 q5 章的第二大认知升级。值得在书中方法章节单列「金融领域反直觉概念清单」
- **【教学场景】回撤事件的"事件化"思维**:把"-15%"还原成"开始-最低-恢复-未恢复"的事件结构,是金融数据可视化的高阶动作。零基础课程很少教,但 q5 把它做出来了——可作为"零基础课程也能教高阶可视化"的证据
- **【教学场景】单年份(2021)案例的力量**:学员对"2021 年特别凄惨"有切身记忆;用具体年份做案例比"某些年份"抽象表述强得多。这条值得在书中方法章节作为"教学案例选择原则"列出
