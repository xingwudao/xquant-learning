## 一、设计意图复盘

spec-02 接续 spec-01 完成 q5 体检的第二步——**逐年拆解**。spec-01 算出"整体看起来还行",spec-02 立即把"还行"拆成 5 年成绩单。这种节奏设计有意识地模拟"逐步揭穿"的叙事:学员看完整体表会安心,看完逐年表会警觉。【动机】

**为什么 TopNRanking 单独深挖**:spec 要求 5 让 AI 用 6 个不同动量窗口(5/10/15/20/25/30)重跑 TopNRanking,目的是**证伪"参数选错了"这个最容易的归因**。如果换一遍窗口都亏,就剩"动量策略本身的结构性弱点"这一种解释——这是用数据排除法支撑结论,是 q5 整章"先猜后验"理念的精彩落点。【动机】【验证】

**为什么不深挖 RiskParity 或 EqualWeight 的负年份**:克制——一份 spec 一个深挖动作。如果都挖,学员注意力会分散。这种"挑最有教学价值的那个深挖"的选择,值得作为"spec 不是清单,是叙事"的示范。【教学场景:渐进难度】【简洁】

**接续型 spec 的复用**:spec-02 假定 `results`、`run_strategy`、`TNR_PORTFOLIO`、`eq_bh_portfolio` 都来自 spec-01。这些变量在 spec-02 没声明,但**学员复制 spec-02 给 AI 时,AI 会在同一个 notebook 上下文中看到这些变量**——这是 q5 内部的接续型 spec 链(spec-01 → 02 → 03 → 04 共享同一个 notebook 状态)。【结构:接续型】

**两个动态描述指令**:要求 6 的「根据实际数据动态描述方向」与「TopNRanking 负收益年份深挖结论」——前者防 LLM 硬写,后者要求 LLM 基于数据下结论。这是 q5 整章一致的"防幻觉"约定。【正向】

**克制的张力**:spec 没要求"画 TopNRanking 各窗口的逐年柱状图",只要表格——因为后面 spec-04 才系统讲参数敏感性。这种"避免话题串联"的边界感,是好 spec 的隐藏品质。【教学场景:渐进难度】

## 四、可改进点

### 改进 1:深挖任务避免重复回测【精确】【同步】

**现状**(要求 5):
```
- 用不同动量窗口 mom_period = [5, 10, 15, 20, 25, 30] 分别构造 ... 并调用 run_strategy ...
- 提取负收益年份的年收益率
- 打印对比表:动量窗口 vs 该年收益
- 如果所有窗口都亏钱,说明不是参数问题
```

**改写**:
```
5. 深挖 TopNRanking 在负收益年份(每个动量窗口只跑一次,结果存 dict 复用):
   - 扫描 mom_periods = [5, 10, 15, 20, 25, 30]
   - 对每个窗口:运行一次回测,提取 daily_returns,存入 deep_dive[period] = result
   - 从 deep_dive 中直接读取数据,绘制对比表(行=窗口,列=负收益年份)
   - 用同一个 deep_dive 字典判断「是否所有窗口在所有负年份都亏」,
     不要重新调用 run_strategy
```

**为什么**:notebook 现在跑了 12 次回测,其中 6 次是重复的——既慢又有风险(两次跑结果不一致就判断错)。

### 改进 2:对比表用 DataFrame 输出【精确】【同步】

**现状**(要求 3):
```
- 行:年份
- 列:四个策略各一列(年收益),最后一列是基准年收益
- 负收益用 * 标注
```

**改写**:
```
3. 打印逐年对比表(用 pandas DataFrame,index=年份,columns=策略名+基准):
   annual_df = pd.DataFrame(...)
   - 用 pandas Styler 或者格式化函数把负收益加 * 标注
   - DataFrame 存入变量 `annual_df` 供 spec-03 复用
```

**为什么**:DataFrame 是后续分析的标准入口;现在的 print 字符串无法复用,而且 .applymap 比手动循环格式化更稳。

### 改进 3:把"最小样本过滤"写进 spec【精确】

**现状**:spec 没说;notebook 用 `if len(yr_ret) < 10: continue`(隐式)

**改写**(加到要求 2):
```
2. 按年度拆解四个策略 + 基准:
   - 用 daily_returns() 按年分组
   - 跳过样本不足的年份:if len(year_data) < 20: continue
     (覆盖学员当年只有几个交易日的边界情况,如 2026 年初跑回测时 2026 仅有 1 周数据)
   - 计算每年的:年收益率(累乘)、年波动率、年夏普比、年内最大回撤
```

**为什么**:这是隐式约定,如果不写,不同 AI 可能选 < 10 / < 20 / < 30 / 不过滤,导致表的行数不同。

### 改进 4:补值域 assert【验证】

**现状**(验证段):"完整年份覆盖、5 列、6 行"

**改写**:
```
执行成功的标志:
- assert annual_df.shape == (年数, 5), "应有 5 列(4 策略+基准)"
- assert annual_df.index.is_monotonic_increasing, "年份应递增"
- # 累乘验证:每个策略各年的(1+ret).prod() 应约等于整体累计收益
  for name in strat_names:
      annual_compound = np.prod([1 + annual_data[name][y]["return"] for y in years])
      assert abs(annual_compound - (1 + results[name].total_return())) < 0.01
- TopNRanking 深挖表有 6 行(若有负收益年份);
  无负收益年份时跳过深挖并明确打印「TopNRanking 每年都是正收益,无需深挖」
```

**为什么**:累乘断言是这份 spec 最关键的"算对了"判据——年度数字加起来应该等于整体数字。少了它,很多隐蔽 bug 看不出来。

### 改进 5:把柱状图颜色与 spec-01 对齐【同步】

**现状**:spec 没指定具体颜色;notebook 用 #4472C4/#ED7D31/#70AD47/#FFC000/#A5A5A5

**改写**(在要求 4):
```
4. 画逐年收益柱状图(figsize 14x6):
   - 颜色与 spec-01 净值曲线保持一致:
     EqualWeight=#4472C4, RiskParity=#ED7D31, TopNRanking=#70AD47,
     RP+止损5%=#FFC000, 等权买入持有=#A5A5A5
   - 每年一组 5 根柱子(4 策略 + 基准)
   - y=0 处画一条水平线
   - 标题「逐年收益对比」
```

**为什么**:同一个策略在 spec-01 和 spec-02 用不同颜色,学员视觉上无法联结——这是跨 spec 同步性的常见漏洞,需在 spec-01 引入颜色后,后续每份 spec 显式继承。

## 五、可沉淀的方法点

### A. 这份 spec 已经示范的原则

- **【动机】证伪式深挖**:要求 5 用「换遍所有窗口都亏」证伪"参数选错了"——可作为"spec 教学不止下结论,还教推理路径"的示范
- **【教学场景:渐进难度】克制单点深挖**:只挖 TopNRanking,不挖 RiskParity——可作为"spec 不是清单,是叙事:每章只讲一个高潮"的范例
- **【正向】动态描述指令防幻觉**:「不要硬写'更高'或'更低'」——可作为"q5 整章一致防 LLM 幻觉的标准句"
- **【结构】接续型 spec 链**:spec-02 复用 spec-01 的 results / run_strategy / 基准变量——可作为"q5 章内 4 个 spec 共享 notebook 状态"的接续型 spec 范例

### B. 这份 spec 修补后可示范的原则

- **【精确】「重复回测」是隐藏反模式**:当前 notebook 跑了 12 次但应只 6 次——修复后可作为"扫描类 spec 必须强制结果缓存"的 before/after
- **【精确】表格输出形态(DataFrame vs print)必须显式**:当前 spec 留给 AI 自由选——修复后可作为"凡是后续 spec 要复用的输出,必须指定数据结构"的示范
- **【验证】累乘断言**:当前没有"年度累乘 = 整体累计"的机械验证——修复后可作为"金融数据的内部一致性断言"的示范
- **【同步】跨 spec 颜色继承**:当前各 spec 颜色靠 AI 巧合一致——修复后可作为"q5 章共享视觉编码"的范例

### C. 领域/教学场景特殊考量

- **【教学场景】"分段检验" 是金融领域的最低基本动作**:零基础学员看到"年化 10%"会觉得每年都赚 10%,但实际上可能 3 年大赚、2 年大亏。spec-02 把这层"统计学常识"显式教,值得在方法章节单列
- **【教学场景】最小样本过滤是数据质量的常识**:年内 < 10 个交易日不进表,是金融统计的默契。spec 必须写明,否则 AI 可能用 1 月份的 5 天数据计算"年化"得出离谱数字
- **【教学场景】"换遍所有参数都亏" 是排除法的强证据**:这种"穷举式证伪"是 q5 引入的高阶推理模式,值得后续 q6/q7 沿用并在方法章节锚定为"先猜后验"的高阶版本
