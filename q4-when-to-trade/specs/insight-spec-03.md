## 一、设计意图复盘

这是 q4 章的**收束之作**——不仅完成"止盈"对比实验，更承担把全章认知抬升到方法论层级的任务："不是所有听起来合理的规则都有效"。整章三份 spec 在编排上是**渐进反直觉**结构：spec-01 频率有效但有成本权衡 → spec-02 止损逻辑清晰但没压住回撤 → spec-03 止盈在不同阈值上给出反向结论。固定窗口下 10% / 20% 止盈降低收益，30% 止盈反而提高收益和简化夏普，这种安排让学员**用真实数据撞上"直觉不稳定"的反例**，而不是听老师讲"止盈是个有争议的话题"。【教学场景：实验设计】

**为什么 4 组对比是 [None, 0.10, 0.20, 0.30]**：阈值梯度比 spec-02 的止损更宽（10/20/30 vs 5/10/15）——止盈触发要求"涨到一定程度"，太紧会让太多信号被切掉，对比失真；跨度更大也让"同一类规则换个阈值，结论可能反过来"更有说服力。**问题同样是没把这层解释写进 spec**。【动机】

**为什么"基准 = 最优频率 + 最优止损"**：要求第 3 条用 `BEST_SL = max(...)` 动态选 spec-02 的最优止损阈值，与 spec-01 的 `BEST_FREQ` 叠加作为 spec-03 的基础。这是接续型 spec 链的**累积式接续**——不是简单复用前一个 spec，而是把前两个 spec 的"最优结果"叠在一起。每加一层规则，都是在前一层最优配置之上加。【结构】【教学场景：累积接续】

**为什么必须打印交易记录前 10 笔**：要求第 8 条让学员看到 `order_type` 字段。这是 q4 整章**唯一的微观证据**——前面所有对比都是宏观指标，前 10 笔把"规则机制"从宏观可视化下沉到单笔交易级别。前 10 笔可能全是 `market`，这本身也说明条件单是否触发取决于具体时点。【验证】【教学场景】

**为什么收尾箴言**：「让利润奔跑，截断亏损」（Let profits run, cut losses short）。spec-03 用它收束全章，但**先用数据验证再引箴言**——这个顺序确保箴言不是空喊。第二句"加规则之前，先问自己：这条规则的逻辑站得住脚吗？数据支持吗？"，把全章方法论提炼成两句问句，给后续 q5/q7 留下钩子。【动机】【教学场景】

**金融"止盈"如何零基础友好**：标题"赚了要不要走？"翻译；分析段反复用"落袋为安""锁定利润""万一涨上去又跌回来"等口语表述。要求第 9 条逐条对比"止损截断亏损（逻辑清晰）vs 止盈截断利润（越紧伤害越大）"——把两个相邻概念放一起对比，是零基础学员理解新概念的最佳路径。【可学习】

## 四、可改进点（带改写示例）

### 改进 1：解释 TP_THRESHOLDS 三个数字的来源【动机】

**现状**（要求 第 4 条）：
```
4. 定义 4 组对比：不止盈 / threshold=0.10 / 0.20 / 0.30
```

**改写**：
```
4. 定义 4 组对比：不止盈（None 对照组）/ threshold=0.10 / 0.20 / 0.30
   · 0.10（10%）= "很紧"——任何小涨幅就触发，频繁切断利润
   · 0.20（20%）= "中等"——常见的零售投资者参考阈值
   · 0.30（30%）= "很松"——只在显著上涨时触发
   止盈阈值梯度比止损更宽（spec-02 是 5%/10%/15%），是因为：
   · 止盈触发要求"涨到一定程度"，太紧会切掉太多信号导致对比失真
   · 跨度更大让"同一类规则不同阈值可能结论相反"的结论更有说服力
   学员仿写可换成 [0.05, 0.15, 0.25] 等，保持递增的松紧梯度即可。
```

把"为什么是这三个数 + 为什么比止损梯度更宽"两层意图都写进 spec——和 spec-01/spec-02 的同类改进互为印证。

### 改进 2：补一个机械可验证的 assert【验证】

**现状**：验证段只描述"对比表有 4 行 + 交易记录有 `order_type` 列"。

**改写**（加到结果呈现后，"自检"小节）：
```
0. 自检：
   assert len(results_tp) == 4 and None in results_tp, "应包含 None 对照组 + 3 个止盈阈值"
   # 完整交易记录中应至少留下一种条件单触发痕迹
   smallest_tp = min(tp for tp in TP_THRESHOLDS if tp is not None)
   order_types = {t.order.order_type for t in results_tp[smallest_tp].trades}
   assert 'stop' in order_types or 'limit' in order_types, \
       f"完整交易记录应包含 stop 或 limit 类型订单（实际：{order_types}）"
```

让"两种规则共存"从描述变成机械判据。

### 改进 3：把"float 转换"的常见坑写进 spec【精确】【同步】

**现状**：spec 没提 fee 字段是 Decimal 类型。

**改写**（加到结果呈现"对比表"前）：
```
注意：result.trades[i].fee 是 Decimal 类型，print 格式化时需转 float：
  total_fee = sum(float(t.fee) for t in r.trades)
否则 print(f"{total_fee:>10.0f}") 会报 TypeError。
```

预防性写明常见坑——spec-01/spec-03 都漏了这条，统一加上后是【同步】方法点的好示范。

### 改进 4：在 spec 里点明"累积接续"的设计【结构】

**现状**：spec 第 3 条只说"用代码动态选出夏普比率最高的止损阈值"，没显式说明这是和 spec-01 的 BEST_FREQ 累积叠加。

**改写**：
```
3. 累积接续：本 spec 在 spec-01 + spec-02 的最优配置之上加止盈：
   - BEST_FREQ：来自 spec-01 含成本最优频率（已在 notebook 中存在该变量）
   - BEST_SL：本 spec 现场动态选 = 在 spec-02 results_sl 字典里夏普最高的非 None 阈值
   - 本 spec 在此基础上继续测 4 组止盈阈值
   
   动态选 BEST_SL 的代码：
   BEST_SL = max([sl for sl in STOP_THRESHOLDS if sl is not None],
                 key=lambda sl: results_sl[sl].sharpe_ratio())
   
   不要硬编码任何最优值——保证数据日期变化时 spec-03 自动跟随重选基准。
```

让"累积接续"从隐式架构变成显式契约——这是接续型 spec 链最有价值的设计模式之一。

### 改进 5：分析段补"越紧伤害越大"的具体输出【动机】【同步】

**现状**（要求 第 9 条）：
```
- 与 Step 2 止损的对比：止损截断亏损（逻辑清晰），止盈截断利润（越紧伤害越大）
```

**改写**：
```
- 止盈对总收益的影响——逐阈值打印 ret_change_pct，按阈值升序排列：
  for tp in [0.10, 0.20, 0.30]:
      ret_change = (results_tp[tp].total_return() - base_ret) * 100
      print(f"止盈 {tp:.0%}：收益变化 {ret_change:+.2f}pp")
  让"越紧伤害越大"通过数据明确呈现（10% 比 30% 损失更大）
- 与 Step 2 止损的对比：
  · 止损截断亏损（下跌中越拿越亏，跑了是对的，逻辑清晰）
  · 止盈截断利润（上涨中越拿越赚，跑了是错的，越紧伤害越大）
```

让"越紧伤害越大"从书中 markdown 单方面承担，前移到 notebook 代码本身的输出——【同步】维度立刻+1。

## 五、可沉淀的方法点

> 标签体系：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【结构】累积式接续——前 N 个 spec 最优结果叠加**：要求第 3 条用 `BEST_FREQ`（来自 spec-01）+ `BEST_SL`（本 spec 现场选自 spec-02 的 results_sl）+ 4 组止盈阈值——三层最优配置层叠。可作为方法章节"接续型 spec 链有两种模式：简单复用 vs 累积叠加"的累积模式原型。
- **【验证】微观证据：交易记录前 10 笔**：要求第 8 条把宏观指标对比下沉到单笔交易级别，让学员**亲眼看到** `order_type` 字段。前 10 笔可能全是 `market`，完整交易记录里再检查 `stop` / `limit` 触发痕迹。可作为"宏观指标 spec 应配套微观证据"的示范——和"实验先发现、文字后兜底"互为表里。
- **【动机】"先用数据撞上反例 → 再用箴言收束"**：spec-03 整体设计是"实验得出反直觉结论（止盈降低收益）→ 引出 'Let profits run, cut losses short' 箴言 → 提炼方法论问句"。这个顺序确保箴言不是空喊。可作为"金融教学 spec 引用专业箴言时必须先有数据证据"的示范。
- **【动机】对照已学概念解释新概念**：要求第 9 条 "止损截断亏损（逻辑清晰）vs 止盈截断利润（越紧伤害越大）"——用 spec-02 已学的止损解释 spec-03 新引入的止盈。可作为"邻近概念用对照法解释"的示范（与 spec-02 的"Signal vs Rule"互证）。
- **【可学习】零基础口语翻译完整**：标题"赚了要不要走？"、"落袋为安"、"万一涨上去又跌回来"——把"止盈"完整翻译成生活语言。可作为"金融术语必须双语化"的示范。

### B. 修补后可示范的原则

- **【动机】参数列表必须解释来源**：当前 TP_THRESHOLDS 三个数没说为什么是这三个；修补后可作为"魔法数字必须给出反推逻辑"的示范——q4 三份 spec 全部存在这个问题，是章级共性短板，修补后是【动机】方法点的最强证据（同一标签 N 次出现 = 重点教）。
- **【验证】机械 assert 替代描述性验证**：当前只有验证段描述；修补后"order_types 应包含 stop 或 limit"的 assert 可作为"复合机制 spec 的验证应检查机制痕迹（不只检查数值）"的示范。
- **【结构】累积接续的显式契约**：当前隐式靠 `BEST_FREQ` + `BEST_SL` 串起来；修补后显式声明可作为"接续型 spec 链应在每份 spec 开头列出 incoming 接口"的示范。
- **【同步】预防性写明常见类型坑（Decimal vs float）**：q4 整章 spec 都漏了 float 转换；修补后可作为"领域库的特殊类型必须在 spec 里点名"的示范——和 q1/insight-spec-01 关于"实现细节就是知识"互证。

### C. 领域 / 教学场景特殊考量

- **【教学场景】渐进反直觉——用参数反例完成认知抬升**：q4 三份 spec 编排是"频率有效 → 止损逻辑清晰但未压住回撤 → 止盈阈值结论反向"。spec-03 的核心价值不是"教止盈怎么用"，而是"用数据告诉学员同一类规则也可能因参数不同而得出相反结论"。可作为方法章节"对比实验 spec 的高级形态——让一组对比给出反直觉结果，反例本身就是知识"的示范。
- **【教学场景】箴言引用的"先证据后箴言"原则**：本课程引用任何金融圈传世箴言（"让利润奔跑，截断亏损"、"市场永远是对的"等）都应先用学员自己跑出的数据验证再引——否则就退化成"传统教材的权威说教"。spec-03 的收尾是这个原则的最佳示范，可作为方法章节"引用领域箴言的两步法"的核心案例。
- **【教学场景】方法论提问作为接续锚**：spec-03 结尾两个问题"这条规则的逻辑站得住脚吗？数据支持吗？"——第二个问题（"换个时段还成立吗？"）正是 q5"样本外测试"要回答的核心问题。这种"前一章 spec 的收尾 = 后一章 spec 的开题"是接续型 spec 链跨章串联的接口设计，可作为"章节收尾 spec 应埋下一章的钩子"示范。
- **【教学场景】oxq order_type 的领域知识**：交易记录里 stop（止损触发）/ limit（止盈触发）/ market（调仓）三种类型对应不同执行机制——这是 oxq 框架的设计也是真实交易的常识。spec-03 通过"打印前 10 笔"让学员第一次看到 `order_type` 字段，再通过完整交易记录观察条件单触发痕迹，是把框架知识和领域知识合并教学的好例子。
