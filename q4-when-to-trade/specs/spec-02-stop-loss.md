## 任务

在 notebook 中新建代码单元格，给策略加上止损规则，对比不同止损阈值的效果。

## 要求

1. 阅读以下 oxq 模块的源码，了解接口的输入、输出和参数含义：
   - `oxq.rules.StopLossRule`（注意：StopLossRule 作为 rule 传入 `Engine.run(rules=[...])`，不再是 Strategy 字段）

2. 解释 StopLossRule 的工作原理：
   - 当持有仓位时，StopLossRule 会以 `avg_cost x (1 - threshold)` 为止损价，提交一个 stop SELL 挂单给 SimBroker
   - SimBroker 在后续每个 bar 检查：如果收盘价跌到止损价以下，自动触发卖出
   - 这是一个**挂单机制**——不是立即卖，而是"预设一个底线，跌破了自动执行"
   - **为什么 Signal 做不到这件事**：Signal 只看市场数据（动量、波动率），不知道你的买入价是多少。止损需要知道"我买了多少钱、现在亏了多少"——这只有 Rule 能做

3. 从 spec-01 的回测结果中，**用代码动态选出**含成本后夏普比率最高的频率作为基准（不要硬编码数字）：
   ```python
   BEST_FREQ = max(FREQUENCIES, key=lambda f: results_with_fee[f].sharpe_ratio())
   ```

4. 定义 4 组对比：无止损 / threshold=0.05 / 0.10 / 0.15

5. 全部含成本回测（使用与 spec-01 相同的 fee_model）。调用 spec-01 定义的 `run_backtest` 辅助函数，将止损规则作为 order_rules 传入：
   ```python
   run_backtest(
       frequency=BEST_FREQ,
       fee_model=PercentageFee(rate=Decimal("0.001"), min_fee=Decimal("5")),
       order_rules=[StopLossRule(threshold=T)],
   )
   ```
   在 `run_backtest` 内部，止损规则与 `RebalanceFrequencyRule` 一起传入 `Engine.run(rules=[...])`。

6. 打印对比表：止损阈值、累计收益率、年化波动率、最大回撤、夏普比率、交易次数、总手续费

7. 画净值曲线对比图（figsize 12x6）：
   - 无止损（灰色虚线）vs 3 种止损阈值（不同颜色实线）
   - 归一化到起点 = 100
   - 标题「止损规则对比（含交易成本）」

8. 在无止损的净值曲线上，标注最大回撤发生的时段（用浅红色阴影区域），让学员直观看到这一段下跌的位置——便于和有止损的几条曲线在同一时段对比走势。

9. 打印分析（根据实际数据动态描述方向，不要预设结果方向）：
   - 止损对最大回撤的影响——是收窄了还是加深了？
   - 止损对年化波动率和夏普比率的影响——分别是抬高还是压低？
   - 不同阈值（5% / 10% / 15%）之间的差异——哪个阈值在哪个指标上占优
   - 止损触发了多少额外交易？带来了多少额外成本？
   - 「止损不是免费的——每次止损触发都是一次额外的卖出，之后再买回又是一次手续费。」
   - 「'保护本金'听起来稳，但'保护'在你这段数据上到底意味着压低回撤、压低波动，还是别的什么——交给数据回答。」

10. 固定窗口参考结果（`START = "2021-01-01"`，`END = "2026-03-18"`，基准频率为 spec-01 含成本简化夏普最高的 10 天），用于判断是否复现书稿：
    - 无止损：`101.68% / 11.19% / -12.81% / 1.31 / 372 / 3021`
    - 5% 止损：`100.78% / 10.71% / -13.25% / 1.36 / 389 / 3903`
    - 10% 止损：`100.32% / 11.01% / -13.08% / 1.32 / 378 / 3270`
    - 15% 止损：`95.45% / 11.14% / -14.99% / 1.26 / 374 / 3070`
    - 这组数据里止损没有收窄最大回撤，主要压低了波动；分析不要写成"止损保护了本金"的固定结论
