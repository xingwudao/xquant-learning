## 一、设计意图复盘

这是 q4 三份 spec 中**概念密度最高**的一份。不仅要让学员跑出"加止损 vs 不加"的对比表，更要建立"Signal 看市场、Rule 看持仓"这个根本性的角色区分——这是后续 q5/q7 调度多种规则时的认知基础。spec 把这层概念解释从书中前移到 spec 本身（要求第 2 条），是因为学员复制 spec 给 AI 时会读到这段，AI 输出的解释印到 notebook 里就能给学员第二次曝光。【结构】【教学场景】

**为什么"挂单机制"要在 spec 里说清楚**：StopLossRule 不是"每 bar 检查跌幅 → 立即卖"，而是"提交 stop SELL 挂单 → SimBroker 后续每 bar 检查"。这层间接性是 oxq 框架的设计选择——更接近真实券商的工作方式（GTC 挂单）。spec 要求第 2 条把这个机制掰开讲，并对比 Signal 和 Rule，是对【动机】的优秀示范——"为什么必须用 Rule 实现止损"通过对比 Signal 的局限说清楚了。【动机】【教学场景】

**为什么 BEST_FREQ 用代码动态选**：要求第 3 条明确"用代码动态选出含成本后夏普比率最高的频率，不要硬编码数字"。这是接续型 spec 链特有的【可复现】考量——spec-01 在不同数据日期下"最优频率"可能不同，硬编码会让 spec-02 在数据更新后跑出错误的基准。这条指令是整章三份 spec 中防漂移最重要的一招。【可复现】【教学场景：接续型 spec 链】

**为什么 4 组阈值是 [None, 0.05, 0.10, 0.15]**：spec 没显式解释这三个数，但意图明显——5%/10%/15% 覆盖"很紧/中等/很松"三档，None 作为对照组。spec-02 比 spec-01 在阈值选择上更克制（spec-01 是 4 个数 + 对应名词，spec-02 是 3 个数 + None 对照）——因为止损只需要看"加不加"和"松紧梯度"。**和 spec-01 一样的问题**：spec 没解释这三个数字的"为什么"。【动机】

**为什么要在无止损曲线上标注最大回撤**：要求第 8 条用浅红色阴影标出最大回撤时段——让学员"直观看到止损本来可以避开的坑"。这是优秀的视觉教学设计：把抽象的"最大回撤数字"转化为图上的具体时段，再对比加止损后的曲线在那段时间发生了什么。是【精确】（图表元素具体到 axvspan）和【动机】（教学目的明确）的合体示范。【精确】【教学场景】

**金融术语"止损"如何零基础友好**：spec 标题用"亏了就跑"做大白话翻译，分析段反复用"亏到一定程度就跑""保护本金""活着才能赚钱"等口语表述。但要求第 2 条直接用了"avg_cost x (1 - threshold)"等技术表述——给 AI 看的 spec 主体里有少量术语是合理的（AI 能正确处理），但"挂单机制"对零基础学员的解释还是偏技术。【可学习】

## 四、可改进点（带改写示例）

### 改进 1：解释 STOP_THRESHOLDS 三个数字的来源【动机】

**现状**（要求 第 4 条）：
```
4. 定义 4 组对比：无止损 / threshold=0.05 / 0.10 / 0.15
```

**改写**：
```
4. 定义 4 组对比：无止损（None 对照组）/ threshold=0.05 / 0.10 / 0.15
   · 0.05（5%）= "很紧"——任何小回撤就触发，对急跌敏感、对慢熊会反复割肉
   · 0.10（10%）= "中等"——常见的零售投资者参考阈值
   · 0.15（15%）= "很松"——只在显著下跌时触发
   选这三档是为了让对比覆盖"松紧光谱"。学员仿写可换成 [0.03, 0.08, 0.20] 等，
   只要保持递增的松紧梯度即可。
```

让学员理解"为什么是这三个数"，仿写时不会误以为是固定答案——和 spec-01 的 FREQUENCIES 改进同源。

### 改进 2：补一个机械可验证的 assert【验证】

**现状**：验证段只描述"对比表有 4 行 + 灰虚线 vs 彩实线 + 阴影时段"。

**改写**（加到结果呈现后，"自检"小节）：
```
0. 自检：
   assert len(results_sl) == 4 and None in results_sl, "应包含 None 对照组 + 3 个阈值"
   # 止损规则触发会增加交易次数
   base_trades = len(results_sl[None].trades)
   for sl in [0.05, 0.10, 0.15]:
       assert len(results_sl[sl].trades) >= base_trades, \
           f"止损 {sl} 的交易次数应不少于无止损（实际 {len(results_sl[sl].trades)} vs {base_trades}）"
```

让"加止损必然增加交易次数"从描述变成机械判据。

### 改进 3：把 spec-02 自身的接续接口契约显式声明【结构】

**现状**：spec-02 隐式约定后续 spec 可以从 `results_sl` 字典读取自己的产出，但 spec 没显式声明。

**改写**（加到结果呈现后，"对外接口"小节）：
```
本 spec 留给 spec-03 复用的接口：
- 变量 results_sl：dict[Optional[float], BacktestResult]——key 为止损阈值（None 表对照组）
- 变量 BEST_FREQ：从 spec-01 沿用，spec-03 仍以此为基准频率
- spec-03 将基于本表选出 BEST_SL = max(STOP_THRESHOLDS \ {None}, key=...)
```

让接续型 spec 链的"接口边界"在每一份 spec 的结尾显式可见——这是 spec-01/spec-02/spec-03 共有的薄弱点，但 spec-02 改起来最有示范价值。

### 改进 4：要求第 2 条的"挂单机制"补一个零基础对照【可学习】

**现状**（要求 第 2 条）：
```
- 当持有仓位时，StopLossRule 会以 avg_cost x (1 - threshold) 为止损价，提交一个 stop SELL 挂单给 SimBroker
- SimBroker 在后续每个 bar 检查：如果收盘价跌到止损价以下，自动触发卖出
```

**改写**：
```
- 当持有仓位时，StopLossRule 会以 avg_cost x (1 - threshold) 为止损价，
  提交一个 stop SELL 挂单给 SimBroker
  （类比：你在券商 app 上设了"跌到 X 元自动卖出"的触发单，挂着等条件满足）
- SimBroker 在后续每个 bar 检查：如果收盘价跌到止损价以下，自动触发卖出
- 这是一个挂单机制——不是立即卖，而是"预设一个底线，跌破了自动执行"
```

加一句"券商 app 触发单"的类比，让零基础学员把 oxq 概念锚定到他们可能在网上看过的实际功能上。

### 改进 5：分析段补"阈值太紧/太松的具体表现"【动机】【同步】

**现状**（要求 第 9 条）：
```
- 阈值太紧（5%）和太松（15%）的各自问题
```

**改写**：
```
- 阈值太紧（5%）的问题：
  · 在慢跌行情中反复触发——卖出后市场没真正反弹，下次调仓买回又跌下来，
    形成"锯齿效应"。每轮"卖→买"付两次手续费，但没真躲开下跌
- 阈值太松（15%）的问题：
  · 已经跌了 15% 才触发，损失已不可逆；保护意义打折
- 注意 5% 止损的最大回撤可能比无止损还略大——这正是锯齿效应的证据
```

把"锯齿效应"这个核心解读从 book markdown 前移一部分到 spec，确保 notebook 代码生成的分析就包含这条洞见——不依赖 book 单方面补救。

## 五、可沉淀的方法点

> 标签体系：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【可复现】动态选基准而非硬编码**：要求第 3 条 `BEST_FREQ = max(FREQUENCIES, key=...)` + 明确指令"不要硬编码数字"——这是接续型 spec 链防漂移的最重要一招。可作为方法章节"接续型 spec 之间用代码而非数字传递最优结果"的原型示范。
- **【动机】对比性概念用对照法解释**：要求第 2 条最后"为什么 Signal 做不到这件事——Signal 只看市场，不知道你的买入价"——通过和已学概念（Signal）对比，让新概念（Rule）的角色立刻清晰。可作为"新概念用对照已学概念解释"的示范。
- **【结构】"先解释机制 → 再做对比 → 再分析"三段式**：要求第 2 条解释 StopLossRule 原理 → 第 3-7 条做对比实验 → 第 9 条分析。这种"机制→实验→洞见"的小三段结构在大四段（上下文/任务/要求/结果呈现）内部嵌套，可作为"复杂概念性 spec 的内部结构"示范。
- **【精确】图表关键元素全列出**：figsize、虚线 vs 实线、归一化基准、阴影 alpha——给 AI 的视觉契约非常清楚，结果可复现。可作为"图表细节显式化"的示范。
- **【教学场景】把 book 的核心概念前移到 spec**：Signal vs Rule 的区分本可以只放在 book 的解读中，但 spec 把它前移到要求第 2 条——目的是让 AI 在 notebook 里也输出这段解释，给学员第二次曝光。可作为"重要概念应跨 spec/notebook/book 三处复述"的示范。

### B. 修补后可示范的原则

- **【动机】参数列表必须解释来源**：当前 STOP_THRESHOLDS 三个数没说为什么是这三个；修补后可作为"每个魔法数字背后都要给出反推逻辑"的示范——和 spec-01 的 FREQUENCIES 改进互为印证（同一标签出现 N 次 = 这条方法在书中要重点教）。
- **【验证】机械 assert 替代描述性对比**：当前只有验证段描述；修补后可作为"对比实验的核心规律应该写成 assert"的示范。
- **【结构】接续型 spec 链的"对外接口"显式声明**：当前隐式约定 results_sl 给 spec-03 用；修补后可作为"接续型 spec 必须在结尾声明 outgoing 接口"的示范。
- **【可学习】技术术语配生活类比**：当前"挂单机制"对零基础略硬；修补后用"券商 app 触发单"类比可作为"金融领域 spec 里硬术语必须配生活化锚点"的示范。

### C. 领域 / 教学场景特殊考量

- **【教学场景】金融"黑话"的零基础友好处理**：止损这个词在零售圈早被滥用得失去精度。spec 标题"亏了就跑"四个字、分析段"活着才能赚钱"、"亏到一定程度就跑"——这种把术语翻译成"行为描述 + 口语化箴言"是本课程独有的处理方式。可作为方法章节"金融 spec 必须双语化（术语 + 大白话）"的核心示范。
- **【教学场景】oxq 框架"挂单机制"是领域知识**：StopLossRule 不是"立即卖"而是"提交 stop SELL 挂单"——这个细节是 oxq 设计选择，更贴近真实券商。spec 把这层间接性显式说出来，是"领域知识必须显式化"的延续（与 q1/insight-spec-01 关于"实现细节就是知识"互证）。
- **【教学场景】"锯齿效应"是回测常识但非通识**：5% 止损在慢跌中反复触发，最大回撤反而加深——这是回测领域的"反直觉常识"。spec-02 留出了这个反例（数据真的会显示出来），book 用"锯齿效应"概念兜底解读。这种"实验先发现反直觉、文字后给概念名"的设计是本课程的优势——传统教材通常先讲概念再举例，会让学员相信权威而失去发现的乐趣。
- **【教学场景】Signal vs Rule 是 oxq 架构的认知锚**：q5 起会引入更多 Rule（仓位规则、风控规则等），都依赖学员先建立"Signal 看市场、Rule 看持仓"的角色区分。spec-02 是这个认知锚的奠基，应在方法章节作为"概念引入 spec"的样板（和"实验对比 spec""机制说明 spec"形成 spec 类型分类）。
