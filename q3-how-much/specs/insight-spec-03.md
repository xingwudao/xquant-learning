## 一、设计意图复盘

这是 q3 三联 spec 的收尾，承担**概念跃迁**的角色——从"加权（每只都买，比例不同）"跃迁到"排序（动量为负的不买）"。spec 引入两个新概念：**动量（Momentum）**和**风险调整动量（ram = mom / vol）**，并通过 `TopNRankingOptimizer` 把"按分数排名 + 过滤负值"机械化。这是 q3 中最复杂的一份 spec——也最贴近真实量化因子选股的工程范式。【结构】【教学场景：渐进难度】

**为什么 ram = mom / vol**：spec 没在主体解释，但合理推断是"风险调整后的动量"——单看动量会偏向高波动标的（涨跌都猛），除以波动率后才公平。这条设计动机如果写进 spec，学员才能学到"为什么不直接用 mom 排名"的领域思考。【动机】缺位是这份 spec 的薄弱处。

**为什么 filter_negative=True**：spec 第 3 条说"过滤 ram 为负的标的"，这是动量信号的灵魂——**正在跌的不买**（与等权/风险平价的"永远全持有"形成质变）。spec 把这条写成 `TopNRankingOptimizer(score_col="ram", n=3, filter_negative=True)` 一个参数，简洁但教学含量极高。【精确】

**为什么 n=3 而非 n=2 / n=1**：候选池只有 3 只 ETF，n=3 等于"全选 + 过滤负动量"。如果 n=2 就是"选最强的 2 只"，n=1 就是"全押最强的 1 只"。spec 没解释为什么选 n=3，留下了思考空间——但对 0 基础学员，这条空间可能让他们困惑。【动机】

**末尾分析话术的优雅**：spec 第 5 条明确写「根据实际数据动态描述方向，不要硬写"更大"或"更深"」。这是 q3 三 spec 中**最优秀的一处**——承认了"市场不可预测"，spec 不应假设结果方向。notebook 也实现得很好（`vol_dir = "更大" if tn_vol > rp_vol else "更小"`）。这条写法应该反向传播到 spec-01/02。【正向】【验证】

**结果呈现的三方对比**：spec 要求三条净值曲线（灰虚 + 蓝实 + 橙实）+ 三方指标表 + 动量权重历史图。三方对比是 q3 章的认知收口——把三种思路并排展示。spec 没要求夏普比率（夏普比率在 notebook 第 20 个 cell 才作为「综合评判」补上），这条留白是合理的：spec-03 聚焦"动量排名"这一种方法，夏普比率是更上位的综合评判工具。【简洁】

## 四、可改进点（带改写示例）

### 改进 1：解释 ram = mom / vol 的"为什么"【动机】

**现状**（spec 第 2 条）：
```
为每只 ETF 预计算 20 日动量，并用 Ratio 计算风险调整动量（ram = mom / vol）。
```

只说"风险调整动量"4 个字，0 基础学员不知道为什么要除以波动率。

**改写**：
```
2. 为每只 ETF 注册三个指标：
   - mom（20 日动量，Momentum）：衡量"近期涨多少"
   - vol（20 日波动率，RollingVolatility）：衡量"近期颠簸多少"
   - ram = mom / vol（风险调整动量，Ratio）：衡量"每承受一份颠簸换回的涨幅"
   为什么不直接用 mom 排名？因为高波动标的天然涨跌都更猛，单看 mom 会偏向"运气好"的高波动标的。除以 vol 后才公平——这是因子选股的常用技巧。
```

把"为什么"写进 spec，让学员单独阅读 spec 就能学到这种领域思考。

### 改进 2：解释 n=3 的"为什么"【动机】

**现状**（spec 第 3 条）：
```
用 TopNRankingOptimizer 构建组合优化器（score 使用 ram 列，选 Top 3，过滤 ram 为负的标的）
```

候选池就 3 只，n=3 等于全选——为什么不写 n=2？

**改写**（加一句解释）：
```
3. 用 TopNRankingOptimizer 构建组合优化器：
   - score_col="ram"：按风险调整动量排序
   - n=3：候选池就 3 只，选 Top 3 等于"全选 + 后续过滤"。如果候选池是 100 只，n=3 才有真正的选拔意义。本章为了对比方法差异保留候选池小，n=k 的实战意义在 q5+ 的因子选股章会发挥。
   - filter_negative=True：动量为负的标的直接踢出（不持仓）。这是动量排名与等权/风险平价的本质区别——前者会"放弃跌的"，后两者永远全持有。
```

让 spec 同时教"参数怎么写"和"参数为什么这样写"。

### 改进 3：补权重和的机械 assert（与 spec-01/02 同款模板）【验证】

**现状**：验证段三条全是描述（"权重不相等"/"可能有 0"/"权重图可见空白"）。

**改写**（加到结果呈现 第 0 条）：
```
0. 权重合法性检查：
   tn_w = tn_weights_df.iloc[-1]
   active_w = tn_w[tn_w > 1e-6]  # 排除被过滤的标的
   assert abs(active_w.sum() - 1.0) < 1e-6, f"动量排名活跃权重和应为 1.0"
   assert (tn_w >= -1e-9).all(), "权重不应为负"
   # 软断言：动量为负的时段应有标的被过滤
   has_zero_weight_period = (tn_weights_df.fillna(0) < 1e-6).any().any()
   assert has_zero_weight_period, "动量排名应至少有一个时段过滤掉某标的（否则退化为风险平价）"
```

注意"权重和=1.0"在动量排名下是**只对未被过滤的标的成立**——这是仓位分配 spec 的细节，必须在 assert 中体现。

### 改进 4：把"COMMON 字典"在三 spec 中要么全兑现要么全删【同步】

spec-02 改进 1 已讨论。spec-03 这里也要同步处理：要么三 spec 都给 COMMON 具体代码，要么三 spec 都删掉这条空头支票。

### 改进 5：继承 spec-01 的 end_date 修复【可复现】

同 spec-02 改进 2，q3 三联 spec 链中只要 spec-01 修了，spec-02/03 自动继承修复。

## 五、可沉淀的方法点（用于书中方法/模板章节）

> 标签体系（10 个）：【结构】【精确】【可复现】【动机】【验证】【正向】【简洁】【同步】【可学习】【教学场景】

### A. 这份 spec 已经示范的原则

- **【正向】「不要硬写"更大"或"更深"」是元正向**：spec 第 5 条明确要求 AI 根据实际数据动态判断方向 — 这是 q3 三 spec 中最优秀的反模式预防，可作为「分析话术不应预设结果方向」的范例
- **【精确】TopNRankingOptimizer 三参数全显式**：score_col / n / filter_negative 三个参数齐全 — 是 oxq 框架 spec 中"参数全显式"的良好示范
- **【教学场景：仓位分配】权重图特殊点显式说明**：「动量为负时权重为 0（面积会出现空白）」— spec 不仅要求画图，还预先说明视觉特征。这种"输出契约预设视觉点"是仓位分配章高级写法
- **【教学场景：渐进难度】概念跃迁的最后一份 spec**：从加权到排序的跃迁是 q3 三联 spec 的认知收口 — 可作为「连续 spec 链中最复杂一份的承担」范例

### B. 这份 spec 修补后可示范的原则

- **【动机】参数选择必须解释（ram 公式 + n=3）**：当前 spec 只给参数值，不给"为什么这样选" — 修复后作为「参数解释三层：怎么写 + 为什么这样选 + 候选池变化时怎么调」的范例
- **【验证】仓位分配灵魂断言的进阶版（动量排名特殊性）**：当前 spec 缺机械 assert — 修复后作为「权重断言要考虑被过滤标的」的进阶版（与等权/风险平价的简单 sum=1 不同）
- **【可复现】级联修复链**：q3 三 spec 共享一份 end_date 配置 — 修复后作为「连续 spec 链中复现性问题在源头修复，下游自动继承」范例
- **【同步】COMMON 字典：要么三 spec 全兑现要么全删**：当前三 spec 都说"复用 COMMON" 但 notebook 都没真提取 — 修复后作为「跨 spec 共享逻辑要么给具体代码要么不要写」范例

### C. 领域 / 教学场景特殊考量（不通用但必须教）

- **【教学场景：仓位分配】排序型 vs 加权型 spec 的本质区别**：等权/风险平价是"加权型"（永远全持有，比例不同），动量排名是"排序型"（可能放弃部分标的）。spec 必须显式区分：① 权重和断言对加权型是"全部和=1"，对排序型是"未被过滤的和=1"；② 排序型 spec 必须包含 filter_negative 或类似的"放弃机制"参数。这是仓位分配章的领域核心区分
- **【教学场景：因子选股前置】风险调整动量（ram）是因子选股的入门概念**：q5+ 的因子选股章会大量用 mom/vol、mom/turnover 等比值因子。q3 这里第一次出现 `Ratio(col_a, col_b)` 模式，spec 应在 ram 公式上多花笔墨——这是 q3 与 q5+ 的概念桥梁
- **【教学场景：oxq 框架】required_indicators 的多指标复合**：spec-02 注册 1 个指标（vol），spec-03 注册 3 个指标（mom/vol/ram）且 ram 依赖 mom 和 vol。这种"指标依赖链"是 oxq 信号系统的核心能力，spec-03 是首次完整示范——应作为 oxq 方法章节"指标依赖图"的标杆
